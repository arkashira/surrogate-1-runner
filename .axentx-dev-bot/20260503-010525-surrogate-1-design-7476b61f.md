## Final synthesized plan (highest correctness + concrete actionability)

**Goal (unchanged)**  
Eliminate HF API rate-limit failures and HF Space OOM during ingestion by replacing recursive `list_repo_files` and per-file API calls with **one per-folder `list_repo_tree` + CDN-only fetches**, and project to `{prompt,response}` at parse time.

**Why this wins (merged rationale)**  
- Replaces recursive pagination (100× calls) with a single `list_repo_tree` per folder → deterministic manifest, no 429s from listing.  
- CDN downloads (`/resolve/main/`) bypass `/api/` auth checks and rate limits; no Authorization header required for public datasets.  
- Projects to `{prompt,response}` only at parse time → prevents mixed-schema `pyarrow.CastError` and downstream type errors.  
- Keeps 16-shard deterministic split intact; no infra changes to the matrix strategy.  
- Adds explicit CDN retries/backoff and per-shard SQLite dedup to handle transient network issues and duplicates safely.

## Implementation plan (≤2h)

1. **Add manifest generator** (`bin/build-manifest.py`)  
   - Run on orchestrator (Mac/CI) before shards start.  
   - Inputs: repo, date folder (e.g. `2026-05-03`).  
   - Uses `huggingface_hub.list_repo_tree(path=date_folder, recursive=False)` to list top-level items in the date folder.  
   - Emits `manifest-{date}.json` with `{ "path": "...", "size": ..., "content_id": "..." }`.  
   - Uploads manifest to repo under `manifests/` (or commits to runner repo) when `COMMIT_MANIFEST` is set.

2. **Update ingestion script** (`bin/dataset-enrich.sh`)  
   - Accept `MANIFEST_URL` or local manifest; download once per shard run.  
   - Deterministic shard assignment: `hash(path) % 16 == SHARD_ID`.  
   - For each assigned file:  
     - Download via CDN: `curl -L --retry 3 --retry-delay 5 "https://huggingface.co/datasets/$REPO/resolve/main/$PATH" -o "$TMPDIR/file.parquet"`  
     - Parse with pyarrow; project to `{prompt,response}`; normalize strings; skip non-text entries.  
     - Dedup via central SQLite on HF Space (or local SQLite with WAL) using content hash.  
     - Append to `shard-<N>-<HHMMSS>.jsonl`.  
   - Upload shard output to `batches/public-merged/<date>/shard<N>-<HHMMSS>.jsonl`.

3. **Update dedup module** (`lib/dedup.py`)  
   - Keep SQLite-backed dedup; add content-hash indexing and batch upserts for throughput.  
   - Ensure idempotent inserts so retries do not duplicate rows.

4. **Update GitHub Actions matrix** (no change to 16-shard strategy)  
   - Optionally add a “manifest-builder” job that runs before the matrix and produces an artifact for shards to download.  
   - Pass `MANIFEST_URL` to shards via env or artifact.

5. **Add CDN retry/backoff and timeouts**  
   - Separate from API rate limits; use curl retries and exponential backoff in Python fetches if implemented.

#!/usr/bin/env python3
"""
Build a CDN-only manifest for a date folder in axentx/surrogate-1-training-pairs.
Usage:
  HF_TOKEN=<token> python bin/build-manifest.py --repo axentx/surrogate-1-training-pairs --date 2026-05-03
"""
import argparse
import json
import os
import sys
from typing import List, Dict

from huggingface_hub import HfApi

def build_manifest(repo: str, date_folder: str) -> List[Dict]:
    api = HfApi(token=os.environ.get("HF_TOKEN"))
    # List top-level items in the date folder (non-recursive)
    entries = api.list_repo_tree(repo=repo, path=date_folder, recursive=False)

    manifest = []
    for entry in entries:
        if getattr(entry, "type", None) != "file":
            continue
        # entry.path is like "2026-05-03/file1.parquet"
        manifest.append({
            "path": entry.path,
            "size": getattr(entry, "size", None),
            # Prefer stable content id; fallback to path
            "content_id": getattr(entry, "oid", None) or entry.path,
        })
    return manifest
