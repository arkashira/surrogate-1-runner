## Implementation Plan (≤2h)

**Highest-value improvement**: Add a pre-flight snapshot generator (`bin/snapshot.sh`) that lists all dataset files once, embeds the file manifest into training, and enables CDN-only downloads during parallel shard processing. This eliminates HF API rate-limit risk during ingestion and aligns with the "pre-list file paths once, embed in training script" pattern.

### Steps (all ≤2h total)

1. **Create `bin/snapshot.sh`**  
   - Uses `huggingface_hub` to call `list_repo_tree(path, recursive=False)` per date folder (non-recursive to avoid 100× pagination).  
   - Saves `snapshot-<date>.json` containing `{"repo": "...", "files": ["path1", "path2", ...]}`.  
   - Exits 0 only if snapshot created; prints file count.

2. **Update `bin/dataset-enrich.sh`**  
   - Accept optional `SNAPSHOT_FILE` env var.  
   - If provided, reads file list from snapshot instead of calling `list_repo_files`/`load_dataset`.  
   - Each shard computes its deterministic slice from the snapshot list (same as before: `shard_id % 16`).  
   - Downloads via CDN URL (`https://huggingface.co/datasets/{repo}/resolve/main/{path}`) with no Authorization header.

3. **Update GitHub Actions matrix**  
   - Add a "snapshot" job that runs `bin/snapshot.sh` first and uploads artifact.  
   - Make the 16-shard matrix job depend on snapshot job and download the artifact.  
   - Pass `SNAPSHOT_FILE` to each shard via env.

4. **Add lightweight fallback**  
   - If snapshot missing, `dataset-enrich.sh` falls back to current behavior but logs warning (so cron keeps working during transition).

5. **Validation**  
   - Run snapshot locally, verify JSON.  
   - Run one shard locally with snapshot, verify CDN download and output.  
   - Commit.

### 1. `bin/snapshot.sh`
```bash
#!/usr/bin/env bash
# bin/snapshot.sh
# Generate a deterministic file-list snapshot for a date folder.
# Usage: HF_TOKEN=... ./bin/snapshot.sh <repo> <date_folder> [out.json]
set -euo pipefail

REPO="${1:-axentx/surrogate-1-training-pairs}"
DATE_FOLDER="${2:-$(date +%Y-%m-%d)}"
OUT="${3:-snapshot-${DATE_FOLDER}.json}"

python3 - "$REPO" "$DATE_FOLDER" "$OUT" <<'PY'
import os, json, sys
from huggingface_hub import HfApi

repo, date_folder, out = sys.argv[1], sys.argv[2], sys.argv[3]
api = HfApi(token=os.environ.get("HF_TOKEN"))

# Non-recursive to avoid pagination explosion
tree = api.list_repo_tree(repo=repo, path=date_folder, recursive=False)
files = [item.rfilename for item in tree if item.rfilename.endswith((".parquet", ".jsonl", ".csv"))]
files.sort()

manifest = {"repo": repo, "date_folder": date_folder, "files": files}
with open(out, "w") as f:
    json.dump(manifest, f, indent=2)

print(f"Snapshot: {len(files)} files -> {out}")
PY

echo "Snapshot written to $OUT"
```

Make executable:
```bash
chmod +x bin/snapshot.sh
```
