# README.md
# surrogate-1-runner

Parallel public-dataset ingest workers for the
[axentx/surrogate-1-training-pairs](https://huggingface.co/datasets/axentx/surrogate-1-training-pairs)
HuggingFace dataset.

## What this does

Every 30 minutes (or on `workflow_dispatch`), GitHub Actions launches **16 parallel runners**.
Each runner takes a deterministic 1/16 slice (`slug-hash bucket = SHARD_ID`)
of the public dataset list defined in `bin/dataset-enrich.sh`, streams,
normalizes per-schema, dedups via the central md5 hash store, and uploads
its output to a unique path on the dataset repo:

## GitHub Action Usage

This repository provides a reusable GitHub Action `axentx/dependency-checker` that enforces dependency hygiene in CI.

### Basic Usage

```yaml
name: Dependency Check
on: [push, pull_request]

jobs:
  check-dependencies:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Run dependency checker
        uses: axentx/dependency-checker@v1
        with:
          ci: true
```

### With Custom Configuration

```yaml
name: Dependency Check
on: [push, pull_request]

jobs:
  check-dependencies:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Run dependency checker
        uses: axentx/dependency-checker@v1
        with:
          ci: true
          config-file: .dependency-checker.yaml
```

### Action Inputs

| Input           | Description                              | Required | Default |
|----------------|------------------------------------------|----------|---------|
| `ci`           | Run in CI mode (exits with non-zero on failure) | false    | `false` |
| `config-file`  | Path to configuration file               | false    | `.dependency-checker.yaml` |

### Action Outputs

| Output         | Description                             |
|----------------|-----------------------------------------|
| `missing-imports` | Comma-separated list of missing imports |

The action will fail the build if any missing imports are detected when `ci` is set to `true`.
A summary comment will be posted on the pull request listing the missing imports.
```
```
