## Unified, Production‑Ready LLM SDK

Below is a single, self‑contained SDK that pulls together the strongest ideas from the snippets you posted:

* **Clear, typed exception hierarchy** – all provider‑specific errors inherit from a single base class.
* **Consistent response schema** – `LLMResponse` and `Usage` are dataclasses that can be serialised to JSON.
* **Abstract provider interface** – `BaseProvider` defines the contract that every provider must fulfil.
* **Concrete provider implementations** – `OpenAIProvider`, `ClaudeProvider`, and `MinimaxProvider` are ready to be swapped in.
* **API‑key validation** – each provider checks its key on construction and exposes a `validate_api_key` method for lazy validation.
* **Extensibility** – new providers can be added by inheriting from `BaseProvider` and implementing `generate`.

> **Contradiction resolution**  
> *The original snippets had a mix of `ValueError` and custom exception classes for missing keys. We keep the custom `AuthenticationError` for consistency and raise it instead of `ValueError`.*

            f"Supported providers: {', '.join(supported_providers)}"
        )

        self.message = message
        self.provider = provider
        self.status_code = status_code
        super().__init__(f"[{provider}] {message}")


class AuthenticationError(Surrogate1Error):
    """Raised when authentication with the LLM provider fails."""

def test_generate_with_supported_providers():
    providers = [OpenAIProvider, ClaudeProvider, MinimaxProvider]
    for provider in providers:
        response = provider().generate("Test prompt")
        assert response is not None

def test_generate_with_unsupported_provider():
    with pytest.raises(ProviderNotSupportedError):
        class UnsupportedProvider(BaseProvider):

        UnsupportedProvider().generate("Test prompt")

def test_api_key_validation():
    class InvalidAPIKeyProvider(BaseProvider):
        def __init__(self, api_key):

    assert provider.api_key is None
    provider.validate_api_key()
