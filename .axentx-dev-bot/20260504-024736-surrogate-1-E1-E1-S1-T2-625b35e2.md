## 1. Acceptance Criteria
- **AC1 – Successful migration**: When a deployment is triggered, all PersistentVolumeClaims (PVCs) referenced by the new ReplicaSet are detached from the old Pods and attached to the new Pods within **5 minutes** of deployment start.
- **AC2 – Data consistency**: After migration, a checksum (e.g., SHA‑256) of a predefined test file stored on the volume before migration must match the checksum of the same file after migration (tolerance = 0 bytes).
- **AC3 – No data loss**: The number of files and total size on the volume after migration must be identical to the pre‑migration state (±0 bytes, ±0 files).
- **AC4 – Deployment time reduction**: End‑to‑end deployment time (from `kubectl apply` to all Pods Ready) must be at least **30 % faster** than the baseline measured with the legacy manual‑copy script (baseline recorded in CI).
- **AC5 – Rollback safety**: If migration fails, the system must automatically roll back to the previous PVC attachment within **2 minutes**, leaving the cluster in a ready state.
- **AC6 – Observability**: The `volume_manager` must emit structured logs (`INFO` level) for each migration step and expose a Prometheus metric `pv_migration_duration_seconds` with a label `status` (`success|failure|rollback`).
