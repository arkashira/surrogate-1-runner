## Final Implementation Plan (≤2 h)

**Highest-value improvement**  
Deterministic pre-flight file listing + CDN-only ingestion to eliminate HF API 429s during training and make shard workers fully resilient.

---

### Concrete steps (ordered for fastest safe rollout)

1. **Generate deterministic file list once per date**  
   - Add `bin/list_files.py` (Mac/Linux) that calls `list_repo_tree(..., recursive=False)` for `batches/public-raw/<YYYY-MM-DD>/` and writes `file-list-<date>.json` containing `{date, path, sha, size}`.  
   - Run it manually after any rate-limit window clears and commit the JSON to the repo (or upload as workflow artifact).  
   - This is the single source of truth for the run; no worker will call `list_repo_tree` or `list_files` during ingestion.

2. **Add robust CDN fetch utility**  
   - Add `bin/lib/cdn_fetch.py` with:  
     - `download_cdn(path, dest, timeout=30, retries=3, backoff=1.5)`  
     - Streamed download, integrity check (size if available), retries with exponential backoff.  
     - Raises on permanent failure so fallback can trigger.

3. **Make workers CDN-first with safe HF API fallback**  
   - Update `bin/dataset-enrich.sh` to accept `FILE_LIST_JSON`.  
     - If provided and valid, workers shard over that list (deterministic modulo assignment).  
     - For each file: try CDN; on CDN 404/timeout/exhausted retries, fallback to `hf_hub_download`.  
   - Keep matrix strategy (16 shards) unchanged.  
   - Remove any use of `list_repo_tree`/`list_files` inside workers when a file list is present.

4. **Update GitHub Actions to inject file list**  
   - In `.github/workflows/ingest.yml`:  
     - Add optional `file_list` input for `workflow_dispatch`.  
     - Add a setup step that either uses the committed JSON or fails fast with a clear message if not found.  
     - Pass `file_list` path to each shard via environment variable.  
   - Keep matrix `shard: [0..15]`.

5. **Training-side guidance (one-time)**  
   - Update `train.py` (outside this repo) to load the same `file-list-<date>.json` and construct CDN URLs directly (`https://huggingface.co/datasets/axentx/surrogate-1-training-pairs/resolve/main/<path>`) during `DataLoader` prefetch.  
   - This eliminates all HF API calls during training data loading.

6. **Dedup remains unchanged**  
   - `lib/dedup.py` continues to use the central SQLite store on the HF Space as the source of truth.

### Resolutions to contradictions

- Listing source: Candidate 1 included `sha` and `size`; Candidate 2 mentioned `sha256`. Use `sha` and `size` from `list_repo_tree` (fast, available) and add `sha256` only if needed later (avoid extra compute during listing).  
- Fallback behavior: Both candidates allow fallback, but Candidate 1 triggered fallback on any exception. Prefer explicit triggers: fallback only on CDN 404 or exhausted retries to avoid masking real CDN outages.  
- File-list distribution: Candidate 1 passed via `workflow_dispatch` input; Candidate 1 also mentioned artifact. Simpler and safer: commit JSON to repo and pass path via input/env; avoids artifact complexity and keeps runs deterministic.  
- Training-side change: Both agree. Embed file list in `train.py` and use CDN-only URLs to guarantee zero API calls during training.
