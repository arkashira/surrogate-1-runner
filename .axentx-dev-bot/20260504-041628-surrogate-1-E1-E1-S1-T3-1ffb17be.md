# Dataset Ingest Process

## Overview

The surrogate-1 project runs parallel public-dataset ingest workers every 30 minutes via GitHub Actions. Each run launches 16 parallel runners that process a deterministic 1/16 slice of the dataset list.

## Automated Ingestion

### Schedule
- **Frequency**: Every 30 minutes
- **Trigger**: GitHub Actions scheduler

### Workflow
1. GitHub Actions triggers `dataset-ingest.yml`
2. Workflow launches 16 parallel jobs (one per shard)
3. Each runner:
   - Loads its deterministic bucket from `SHARD_ID`
   - Streams dataset from `bin/dataset-enrich.sh`
   - Normalizes data per schema
   - Deduplicates via central md5 hash store
   - Uploads to unique path in dataset repo

## Manual Trigger

To trigger a manual ingestion:

```bash
# Via GitHub CLI
gh workflow run dataset-ingest.yml

# With custom parameters
gh workflow run dataset-ingest.yml -f dataset=custom-dataset-name
```

## Monitoring

- Check workflow status at: Actions > dataset-ingest
- View runner logs in the workflow run artifacts
- Dataset updates appear at: https://huggingface.co/datasets/axentx/surrogate-1-training-pairs

## Troubleshooting

See [Debugging Failed Shards](./debugging.md) for common issues.

---
*Last reviewed: 2025-06-03*
