# Quick Start Guide for surrogate-1

This guide provides a step-by-step process to set up and run the surrogate-1 sidecar alongside your test suite, with a focus on debugging flaky CI runs.

## Prerequisites

- A GitHub account
- A repository containing your project
- Docker installed on your machine

## Setup

1. Navigate to your repository on GitHub
2. Click on `Actions` in the left sidebar
3. Click on `Set up a workflow yourself`
4. Paste the following YAML workflow file into the text area:

```yaml
# .github/workflows/ci.yml

name: CI with Sidecar and FD List

on:
  push:
    branches:
      - main
  pull_request:
    branches:
      - main

jobs:
  build-and-test:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Build and push sidecar image
      run: |
        docker build -t surrogate-1-sidecar .
        docker push surrogate-1-sidecar

    - name: Run tests with sidecar
      uses: actions/setup-docker@v2
      with:
        image: surrogate-1-sidecar
      env:
        TEST_SUITE: your-test-suite
      run: |
        docker run --rm -it --entrypoint /bin/bash surrogate-1-sidecar /fd > fd_list.txt
        ./your-test-suite
        cat fd_list.txt
```

5. Click on `Start commit` and commit the changes

## Testing

After committing the workflow file, a new CI run will be triggered. If the tests fail, the `fd_list.txt` file will be generated and printed, allowing you to quickly debug the issue.
