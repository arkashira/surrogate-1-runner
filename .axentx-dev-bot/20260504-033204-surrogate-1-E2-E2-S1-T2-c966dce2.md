**Acceptance criteria:**

1. **Complete TTLManager class** - The `__init__` method is truncated (`TTLCo...`). Need the full implementation including the heap, locks, and background thread.

2. **Fix heap reordering for sliding TTL** - The `touch()` method updates `_expiry_ts` but won't reorder the entry in the heap. When using `heapq`, touching an entry invalidates its position. Either:
   - Re-heapify after touch (expensive but correct), or
   - Use a different data structure (e.g., sortedcontainers, or accept O(n) scan in cleanup)

3. **Add thread safety primitives** - The design claims "lock-based design" but no `threading.Lock` or `RLock` appears in the shown code. Add locks around:
   - `ContextEntry` modifications (`touch()`)
   - Heap operations in TTLManager
   - Dictionary access

4. **Clarify INHERIT policy handling** - The code returns `float("inf")` for INHERIT with comment "resolved during lookup". Ensure TTLManager implements this parent-chain resolution, or document that callers must handle it.

5. **Unused field** - `TTLConfig.inherit_from_parent` is defined but never used in the shown code.

6. **Minor**: Consider making `ContextEntry._expiry_ts` comparison more robust - heapq relies on consistent ordering, and mutable objects in heaps can cause subtle bugs if not handled carefully.

The core abstractions (TTLPolicy, TTLConfig, ContextEntry) are sound. The incomplete TTLManager is the main gap.
