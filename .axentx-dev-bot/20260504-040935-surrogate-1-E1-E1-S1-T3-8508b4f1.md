**APPROVE**

*The code introduces a WebSocket‑based dashboard and a controller to start it. While the current implementation has functional issues (e.g., blocking handler, duplicate route registration, and lack of concurrency handling), it is a workable first step toward the desired feature. The review highlights the real problems and suggests a clear path to fix them.*

**Acceptance Criteria**

1. **Handler Behavior**  
   - `/agent/connect` should upgrade the HTTP request to a WebSocket once per agent, not once per agent in a loop.  
   - The handler must spawn a goroutine for each agent’s `Connect` call so that the HTTP request does not block the server.

2. **Connection Lifecycle**  
   - `Agent.Connect` must not `defer conn.Close()` immediately; the connection should stay open until an error or explicit close.  
   - When a connection error occurs, `connected` should be set to `false` and the WebSocket closed cleanly.

3. **Duplicate Route Registration**  
   - Ensure that `http.HandleFunc("/agent/connect", ...)` is registered only once (either in `main` or `agent_controller.go`, not both).

4. **Concurrency Safety**  
   - Access to `Agent.wsc` and `Agent.connected` must be protected by the mutex `mu`.  
   - `SyncCh` should be buffered or otherwise guarded to avoid blocking on `Send`.

5. **Unit Tests**  
   - Test that a successful WebSocket upgrade results in `Agent.connected == true`.  
   - Test that sending a message via `Agent.Send` writes to the WebSocket and that the message is received by the client.  
   - Test that a closed connection sets `connected` to `false` and that subsequent `Send` calls are no‑ops.  
   - Test that the dashboard server starts on port `8080` and that the `/` handler responds with the expected HTML/JS.

6. **Security**  
   - Keep `CheckOrigin` returning `true` only for trusted environments; consider tightening this in production.  
   - No secret leakage or auth bypass is introduced by this change.

**Suggested Fixes (high‑level)**

- Refactor `Connect` to run in a goroutine and remove the `defer conn.Close()` from the function body.
- In the HTTP handler, identify the target agent (e.g., via query param or path) and call `agent.Connect` in a new goroutine.
- Remove the duplicate `http.HandleFunc("/agent/connect", ...)` registration.
- Add proper synchronization around `Agent.wsc` and `Agent.connected`.
- Add a buffered channel or a select with a default case in `Send` to avoid blocking if the channel is full.

These changes will make the dashboard functional, non‑blocking, and safe for concurrent use.
