* **Ingests** per‑service retry metrics (attempts, successes, failures, latency, back‑off, HTTP codes …).

* **Ranks** each finding by severity and attaches **actionable recommendations**.  
* Is **configurable**, **thread‑safe**, **testable**, and **easy to hook into existing observability stacks** (Prometheus, OpenTelemetry, alert‑manager, etc.).  

Below is the **single, unified solution** that merges the strongest ideas from the multiple drafts you received, removes contradictions, and adds concrete steps you can run today.

## 2️⃣ High‑level Architecture  

```
+-------------------+      +-------------------+      +-------------------+
|   Metric Exporter | ---> |   In‑Memory Store | ---> |   Detector Engine |
+-------------------+      +-------------------+      +-------------------+
                                   |                         |
                                   v                         v
                         +-------------------+      +-------------------+
                         |   Prometheus /    |      |   Alerting /      |
                         |   OpenTelemetry   |      |   Notification    |
                         +-------------------+      +-------------------+
```

1. **Metric Exporter** – any service that already emits retry‑related counters (e.g., via Prometheus client, StatsD, or OpenTelemetry) pushes a `RetryMetrics` struct into the detector via the public `AddMetrics` API.  
2. **In‑Memory Store** – a per‑service sliding‑window (`metricWindow`) holds the last *N* samples (or the last *T* minutes). The window is **lock‑protected** and automatically evicts stale data.  
3. **Detector Engine** – runs every `DetectInterval` (default = 1 min) and evaluates **all six anomaly rules** against each service’s window.  
4. **Observability Hooks** – the engine emits:
   * **Prometheus gauges/counters** (`detector_anomalies_total{type,service,severity}`)  
   * **OpenTelemetry spans/events** (so traces can be correlated)  
   * **Webhook / Alertmanager** payloads (JSON) that contain the full `DetectedAnomaly` object.
