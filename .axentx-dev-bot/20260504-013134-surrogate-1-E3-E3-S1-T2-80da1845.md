### 2. GitHub Actions вЂ“ `.github/workflows/secure_execution.yml`

```yaml
name: SurrogateвЂ‘1 Secure Execution

on:
  workflow_dispatch:          # Manual trigger
  schedule:
    - cron: '*/30 * * * *'    # Every 30 minutes

jobs:
  secure-execution:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Set up Python 3.10
        uses: actions/setup-python@v4
        with:
          python-version: '3.10'

      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install -r requirements.txt

      - name: Run SurrogateвЂ‘1 in secure mode
        run: |
          python -m surrogate1 --secure
        env:
          SURROGATE1_API_KEY: ${{ secrets.SURROGATE1_API_KEY }}

      - name: Upload logs
        uses: actions/upload-artifact@v3
        if: always()
        with:
          name: surrogate1-logs
          path: logs/
```

* **Why this is best**  
  * `workflow_dispatch` + `schedule` gives full control.  
  * `if: always()` guarantees logs are captured even on failure.  
  * Secrets are injected via environment variables, not hardвЂ‘coded.
