## Connector contract (surrogate-1)

All connectors expose synchronous methods that accept only documented parameter types and return a `ConnectorResult` (never raise `ConnectorError` to the caller).

- `BaseConnector` public API: `fetch_data(context)`, `send_order(order)`, `check_risk(quote)`.
- `ConnectorResult` is returned by all connector methods and provides `ok()` / `fail()` factory methods. `to_dict()` includes `ok` (bool), `latency_ms` (float), `value` (any), and `error` (dict or null). Round-trip via `from_dict` preserves equality.
- `ConnectorError` carries `code` (str), `context` (dict), `retryable` (bool), and UTC `timestamp` (ISO-8601). `to_dict()` serializes all fields; `from_dict` reconstructs the error.
- Implementations must report `latency_ms` (non-negative) and must not raise `ConnectorError` to callers. Internal helpers (e.g., `_time`) catch errors and convert them to `ConnectorResult.fail` while recording elapsed time.
- `SampleVendorAdapter` implements `fetch_data` and returns snapshots with keys `symbol`, `bid`, `ask`, `ts` (ISO-8601) when ok. If `symbol` is empty, it returns `ConnectorResult.fail` with `ConnectorError.code="DATA_UNAVAILABLE"`.
