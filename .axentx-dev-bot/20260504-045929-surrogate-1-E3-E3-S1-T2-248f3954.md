# Frequently Asked Questions (FAQ)

## General Questions

### What is surrogate-1?
surrogate-1 is a parallel public-dataset ingest tool that processes datasets for the axentx/surrogate-1-training-pairs HuggingFace dataset. It uses 16 parallel GitHub Actions runners to process data efficiently.

### How does sharding work?
Each runner gets a deterministic shard (SHARD_ID 0-15) based on hash(dataset_name) % 16, ensuring even distribution with no duplicates.

### How often does it run?
- Every 30 minutes (scheduled)
- On push to main branch
- Manual trigger via workflow_dispatch

## Installation & Setup

### How do I get started?
1. Clone: `git clone https://github.com/axentx/surrogate-1.git`
2. Configure secrets: `HF_TOKEN` and `GH_TOKEN`
3. Workflow runs automatically

### Where do I get tokens?
- **GitHub**: Settings → Developer settings → Personal access tokens (repo scope)
- **HuggingFace**: https://huggingface.co/settings/tokens (write permission)

## Troubleshooting

### Workflow won't start
**Check:**
1. GitHub Actions enabled in repo settings
2. Secrets properly configured
3. Branch matches trigger conditions

### Authentication failures
**Solution:**
1. Verify tokens haven't expired
2. Check token permissions
3. Regenerate tokens if needed

### Rate limiting
**Solution:** The tool includes backoff. If issues persist, reduce workflow frequency in `.github/workflows/ingest.yml`.

### Duplicate processing
**Solution:** The md5 hash store prevents duplicates. Verify central hash store is accessible and network connectivity is stable.
