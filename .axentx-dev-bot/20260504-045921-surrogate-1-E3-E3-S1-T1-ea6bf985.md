# surrogate-1

Parallel public-dataset ingest workers for the [axentx/surrogate-1-training-pairs](https://huggingface.co/datasets/axentx/surrogate-1-training-pairs) HuggingFace dataset.

## What This Does

Every 30 minutes (or on `workflow_dispatch`), GitHub Actions launches **16 parallel runners**. Each runner takes a deterministic 1/16 slice (`slug-hash bucket = SHARD_ID`) of the public dataset list defined in `bin/dataset-enrich.sh`, streams, normalizes per-schema, dedups via the central md5 hash store, and uploads its output to a unique path on the dataset repo.

## Quickstart

### Prerequisites

- Python 3.8+ (for local development and sample app)
- GitHub CLI (`gh`) authenticated
- HuggingFace token with write access to `axentx/surrogate-1-training-pairs`
- Standard Unix tools: `bash`, `curl`, `jq`

### Installation

1. **Clone the repository:**
   ```bash
   git clone https://github.com/axentx/surrogate-1.git
   cd surrogate-1
   ```

2. **Install dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Set required environment variables:**
   ```bash
   export HF_TOKEN="your_huggingface_token"
   export GH_TOKEN="your_github_token"
   ```

4. **Verify authentication:**
   ```bash
   gh auth status
   huggingface-cli whoami
   ```

### Configuration

#### Environment Variables

| Variable | Description |
|----------|-------------|
| `HF_TOKEN` | HuggingFace write token |
| `GH_TOKEN` | GitHub personal access token |
| `SHARD_ID` | Override shard ID (0-15) |
| `DATASET_REPO` | Target dataset (default: `axentx/surrogate-1-training-pairs`) |

#### Configuration File (Optional)

Alternatively, configure via `config.json`:
```json
{
  "dataset_repo": "axentx/surrogate-1-training-pairs",
  "shard_count": 16
}
```

### Running Locally

#### Single Shard Run

Process a specific shard (0-15):
```bash
./bin/ingest.sh 0  # Process shard 0
./bin/ingest.sh 7  # Process shard 7
```

#### Full Dataset Refresh

Trigger all 16 shards via GitHub Actions:
```bash
gh workflow run ingest.yml
```

Or push changes to trigger the workflow:
```bash
git add .
git commit -m "Configure surrogate-1"
git push
```

### Project Structure

```
surrogate-1/
├── bin/
│   ├── dataset-enrich.sh    # Dataset list and enrichment logic
│   ├── ingest.sh            # Per-shard ingestion script
│   └── normalize.sh         # Schema normalization
├── src/
│   ├── dedup.go             # MD5 deduplication client
│   ├── stream.go            # Dataset streaming
│   └── upload.go            # HuggingFace upload
├── examples/
│   └── sample_app.py        # Sample integration app
├── config.json              # Optional configuration file
├── requirements.txt         # Python dependencies
├── .github/
│   └── workflows/
│       └── ingest.yml       # GitHub Actions workflow
└── README.md
```

**Usage:** `./bin/ingest.sh <shard_id>`
