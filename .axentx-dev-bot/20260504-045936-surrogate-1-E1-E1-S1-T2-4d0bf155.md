## 1.  High‑level design

| Layer | Responsibility | Key tech |
|-------|----------------|----------|
| **API / In‑flight capture** | Intercepts every outbound request from the async pipeline and builds a *signature* | FastAPI + `httpx` middleware |
| **Detection engine** | Compares live signatures against a stored *baseline* and emits a `DriftDetection` event | Async queue + `asyncio` workers |
| **Persistence** | Stores baselines, drift logs and notification state | PostgreSQL + SQLAlchemy (or async‑pg) |
| **Notification** | Sends alerts to webhook, email, Slack, etc. | `httpx`, `smtplib`, `slack_sdk` |
| **Observability** | Metrics, logs, health checks | Prometheus client, structured JSON logs |
| **Deployment** | Docker + Kubernetes + Helm | Helm chart, CI pipeline |

> **Why this stack?**  
> *FastAPI* gives us async request handling out of the box.  
> *Asyncio* keeps the detection engine lightweight and scalable.  
> *PostgreSQL* is ACID‑compliant and works well with async ORMs.  
> *Helm* lets us ship the whole stack in a single chart.
