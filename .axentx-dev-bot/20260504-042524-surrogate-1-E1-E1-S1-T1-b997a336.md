## 1. Acceptance Criteria  
- **Latency**: The `POST /api/v1/conversation` endpoint must return a JSON response with the AI reply **within 1 second** under normal load.  
- **Session Persistence**: A conversation session identified by a unique `session_id` must remain active (no forced disconnect) for **≥ 5 minutes** of inactivity, with the ability to resume.  
- **Rating Capability**: After each AI turn, the client can submit a rating (`1–5`) via `POST /api/v1/conversation/{session_id}/rate`. The rating must be stored and retrievable.  
- **Transcript Storage**: Every user message and AI reply must be appended to a **secure** (encrypted at rest) transcript stored in the user’s profile. Retrieval via `GET /api/v1/conversation/{session_id}` must return the full transcript in chronological order.  
- **Framework & Schema**: The backend uses **FastAPI** with a **PostgreSQL** database (SQLAlchemy ORM). Conversation records are stored in a `conversations` table with columns: `id`, `user_id`, `session_id`, `messages` (JSONB), `created_at`, `updated_at`.  
- **AI Service**: The endpoint forwards the user prompt to an external AI service (`https://ai.native-speak.com/v1/chat`) via HTTPS POST, passing the `session_id` and user message. The AI response is returned unchanged.  
- **Security**: All endpoints require OAuth2 JWT authentication; the `session_id` is bound to the authenticated `user_id`.
