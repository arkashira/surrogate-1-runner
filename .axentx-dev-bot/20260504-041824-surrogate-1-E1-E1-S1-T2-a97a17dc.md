## Unified Real‑Time Cloud‑Policy Detector  
Below is a **single, production‑ready design** that merges the strongest ideas from the two drafts:

| What we keep from **Candidate 1** | What we keep from **Candidate 2** | Why it matters |
|-----------------------------------|-----------------------------------|----------------|
| Concrete AWS SDK calls (`iam`, `s3`, `ec2`) – the detector actually talks to the cloud. | Typed `Severity` enum, generic `PolicyService`/`RemediationService` interfaces – clean contracts, easy testing, multi‑cloud extensibility. | Gives you a working scanner **and** a clean, testable architecture. |
| `Violation` struct with `Resource`, `Policy`, `Severity`, `Message`. | `Timestamp` on the violation and a separate `logViolation` stub. | Adds observability (when it happened) without breaking existing code. |
| `handleViolation` that logs, flags, and auto‑remediates high‑severity findings. | Separate `RemediationService` that can be swapped (Lambda, Step‑Function, Terraform, etc.). | Keeps the detector thin and lets you plug any remediation engine. |
| Simple `PolicyService` implementation that returns hard‑coded violations. | `ListPolicies` / `EvaluatePolicy` pattern – future‑proof for a policy‑as‑code store (OPA, Rego, Guardrails, etc.). | Allows you to move from “hard‑coded checks” to a real policy engine without touching the detector. |

- **Real‑time detection**: The detector must process incoming resource events and emit a `Violation` within **≤ 50 ms** (measured end‑to‑end latency).
- **Policy coverage**: `awsPolicyService.ListPolicies` must return **all** policies for IAM, S3, EC2, and Azure Storage (verified against the AWS/Azure SDK catalogs).
- **Concrete API call**: At least one `EvaluatePolicy` implementation must invoke an actual AWS API (e.g., `ListAttachedUserPolicies`) and return a non‑empty result.
- **Remediation wiring**: The detector loop must trigger a `RemediationService` remediation action for any violation with severity **≥ HIGH**, and the action must be logged.
- **CLI entry point**: `cmd/detector/main.go` must start the detector, initialize all services, and run an infinite detection loop that can be gracefully stopped with `SIGINT`.
- **Test coverage**: Unit tests achieve **≥ 80 %** coverage for `detector.go` and `policy_service.go`; integration tests cover happy paths and at least three edge cases.
- **Portability**: `Violation.Timestamp` must be stored as an ISO‑8601 UTC string (`time.RFC3339`) to ensure cross‑system compatibility.
