## One‑Stop, Production‑Ready “Compliance Report” Generator  

Below is a **single, coherent implementation** that merges the strongest ideas from the two proposals you posted:

| What we keep from each proposal | Why it stays |
|--------------------------------|--------------|
| **Jinja2 inheritance (`base.html` → `report_template.html`)** (Candidate 1) | Guarantees a clean separation of layout and content and makes future branding changes trivial. |
| **Rich, printable CSS (page‑size, header/footer, colour scheme, status badges)** (Candidate 2) | Gives the PDF a professional look, automatic page numbers and a clear visual hierarchy. |
| **CLI that accepts a report‑id and optional JSON input** (Candidate 1) | Makes the tool scriptable and easy to embed in CI/CD pipelines. |
| **Real PDF rendering with WeasyPrint** (instead of the placeholder) | Produces a true PDF (fonts, pagination, vector graphics) in < 5 s for typical data‑sets. |
| **Cryptographic RSA‑2048 signing using `cryptography`** (Candidate 1, but now fully implemented) | Guarantees integrity and non‑repudiation; the signature is embedded as a detached file (standard practice for PDFs). |
| **Error handling, logging and configuration via a tiny `config.yaml`** | Turns the prototype into a maintainable production tool. |

## 1. Directory Layout  

```
surrogate-report/
├── config.yaml                # paths, key locations, defaults
├── requirements.txt
├── src/
│   ├── __init__.py
│   ├── cli.py                 # entry‑point for the command line
