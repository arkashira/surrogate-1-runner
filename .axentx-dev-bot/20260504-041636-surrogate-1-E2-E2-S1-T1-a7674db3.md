## 📌 Project Overview

**Project Name:** `surrogate-1`  
**Dataset:** `axentx/surrogate-1-training-pairs` (HuggingFace)  
**Purpose:** Parallel ingestion and processing of public datasets in 16 shards.

---

## 🔁 Workflow: Dataset Ingest Pipeline

### ✅ Description
A parallelized ingestion pipeline that splits the dataset into 16 shards for concurrent processing. Each shard is processed independently, normalized, deduplicated, and uploaded to the HuggingFace dataset repository.

### ⏰ Triggers
- **Scheduled:** Every 30 minutes (`*/30 * * * *`)
- **Manual:** Trigger via `workflow_dispatch` (on-demand)

### 🧠 Runners & Parallelism
- **Runner Count:** 16
- **Parallelism:** 16 shards running simultaneously
- **Shard Assignment Strategy:** Deterministic slicing using slug-hash bucket algorithm:
  ```
  SHARD_ID = hash(slug) % 16
  ```

---

## 🛠️ Steps

| Step ID | Name                  | Description                                                                 | Action / Command                                  |
|--------|-----------------------|-----------------------------------------------------------------------------|---------------------------------------------------|
| 1      | Slice Assignment       | Assign each runner a unique shard based on hash of dataset slug             | Runtime via GitHub Actions matrix strategy        |
| 2      | Data Streaming         | Fetch raw data from the dataset list defined in `bin/dataset-enrich.sh`     | `bin/dataset-enrich.sh`                           |
| 3      | Schema Normalization   | Normalize schema-specific fields to ensure consistency                      | Schema-specific normalization logic               |
| 4      | Deduplication          | Remove duplicates using centralized MD5 hash store                          | MD5-based deduplication logic                     |
| 5      | Upload to HuggingFace  | Upload processed shard to unique path in the HuggingFace dataset            | `huggingface.co/datasets/axentx/surrogate-1-training-pairs` |

---

## ⚙️ Configuration Requirements

### Environment Variables
| Variable              | Purpose                                |
|----------------------|----------------------------------------|
| `HF_TOKEN`           | Required for writing to HuggingFace    |
| `MD5_STORE_ENDPOINT` | Centralized deduplication service URL  |

### Resources
- `bin/dataset-enrich.sh`: Script used to stream dataset content
- GitHub Actions workflow files (`.github/workflows/...`)

---

## 🛡️ Error Handling

- Failed shards can be retried manually via `workflow_dispatch`.
- The MD5 store prevents data corruption or duplication across runs.
- All errors are logged and visible in GitHub Actions interface.

---

## 🧹 Cleanup & Maintenance

### Scheduled Tasks
- **Cleanup:** Daily at 01:00 UTC – removes temporary files and archives logs.
- **Validation:** Daily at 02:00 UTC – validates data integrity and generates reports.

### Maintenance Notes
- Monitor GitHub Actions logs for failed jobs.
- Update `bin/dataset-enrich.sh` when upstream dataset sources change.
- Scale runner count in workflow definition if necessary.
