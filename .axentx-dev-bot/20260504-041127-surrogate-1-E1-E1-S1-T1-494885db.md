| **Retry strategy** | Tenacity exponential back‑off | Keep the same `@retry` decorator but broaden the exception filter to include network‑related errors (`google.api_core.exceptions.GoogleAPIError`) while still limiting to 3 attempts. |
| **Query** | GAQL from proposal 1 | Use the same query, but add `segments.date` to the `SELECT` list so we can store the exact date of each spend row (helps with up‑serts). |
| **Currency handling** | Simple micros → USD conversion | Keep the conversion but expose a tiny hook (`_currency_converter`) that can be swapped out later if the account isn’t USD. |
| **Database up‑sert** | Delete‑then‑insert from proposal 1 | Replace with a true *up‑sert* using `ON CONFLICT`‑style logic (SQLAlchemy `merge`) – avoids race conditions and unnecessary deletes. |
| **Logging** | Structured logs from both drafts | Add `extra` fields (`campaign_id`, `date`) for easier log‑aggregation, and make the module’s logger configurable from the application. |
| **CLI / entry‑point** | `if __name__ == "__main__"` block from proposal 1 | Keep it, but also expose a tiny `main()` that can be called by external orchestration tools (e.g., `python -m axentx.integrations.google_ads`). |
| **Type safety** | Minimal typing in proposal 2 | Add full type hints, `TypedDict` for raw rows, and a small `@dataclass` for the ORM payload. |
| **Error handling** | Propagate non‑GoogleAds errors | All unexpected exceptions are logged and re‑raised so the scheduler can mark the run as failed. |
