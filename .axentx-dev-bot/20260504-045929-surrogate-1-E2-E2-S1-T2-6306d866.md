## 1. Acceptance Criteria  

1. **Status‑bar display** – When `ErrorDisplayService.displayError(errorCode, Throwable)` is invoked, the corresponding human‑readable message is shown in KiCAD’s status bar within **2 seconds**.  
2. **Log file creation** – On first use, the service creates the directory `/var/log/axentx/` (if missing) and a file named `kicad‑freerouter‑integration.log`. The file must be **writable** by the KiCAD process user.  
3. **Log entry format** – Each logged error line must contain: ISO‑8601 timestamp, error code, short description, and full stack trace, separated by a pipe (`|`). Example: `2026-05-20T14:32:10.123Z|MISSING_JAVA|Java runtime not found|java.lang.ClassNotFoundException…`.  
4. **Message mapping** – All error codes defined in `error_messages.properties` are loaded at startup; requesting an undefined code falls back to a generic “Unexpected error” message both in the status bar and the log.  
5. **Dependency‑failure handling** – Specific scenarios (missing Java, missing Freerouter binary, permission denied on log directory) trigger dedicated handler methods that produce the exact pre‑defined status‑bar text (e.g., “Freerouter cannot start: Java not installed”).  
6. **Graceful degradation** – If the log file cannot be opened (e.g., disk full, permission error), the service writes the same message to `System.err` and still updates the status bar.  
7. **Thread‑safety** – Concurrent calls to `displayError` from multiple threads must not corrupt the log file; the log file size after 100 concurrent errors must equal **100 × (single‑line length)** plus newline characters.
