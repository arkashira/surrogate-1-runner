## 1. Acceptance Criteria

- **Provider Selection**  
  - When the user selects a provider (`AWS`, `GCP`, or `Azure`) from the dropdown, the form renders the exact set of credential fields required for that provider (e.g., `Access Key ID` & `Secret Access Key` for AWS, `Service Account JSON` for GCP, `Client ID` & `Client Secret` for Azure).
- **Field Validation**  
  - Submitting the form with any required field empty triggers an error message (`errorMessage`) and prevents the `credential-added` event from being emitted.
- **Successful Submission**  
  - When all required fields are populated, submitting the form emits a `credential-added` event with a payload that contains only the fields relevant to the selected provider, e.g.:
    ```json
    { provider: "AWS", accessKeyId: "...", secretAccessKey: "..." }
    ```
- **Form Reset**  
  - After a successful emit, the form fields are cleared and the provider dropdown resets to its placeholder value.
- **Submit Button State**  
  - The submit button is disabled when no provider is selected or any required field is empty; it becomes enabled only when the form is fully valid.
