## 1. Acceptance Criteria (measurable)

1. **Checkbox Functionality**  
   - Selecting or deselecting any investment card toggles its `aria-checked` attribute between `"true"` and `"false"`.  
   - The JavaScript model `selectedIds` contains the ID of every card that is currently checked and excludes unchecked IDs.  
   - No full page reload occurs on any checkbox interaction.

2. **Weighted‑Average Calculation**  
   - Clicking the **Calculate** button triggers a function that reads the `weight` input of each selected card.  
   - The sum of all entered weights equals `100 ± 0.01`.  
   - The displayed expected return equals  
     \[
     \sum_{i \in selected} \text{return}_i \times \frac{\text{weight}_i}{100}
     \]  
     rounded to two decimal places.  
   - If weights do not sum to 100, an error message “Weights must total 100%” is shown and calculation is aborted.

3. **Responsive Design**  
   - At viewport widths ≥ 768 px the investment cards display in a 3‑column grid.  
   - At widths < 768 px the grid collapses to 2 columns, and at < 480 px to 1 column.  
   - All text remains ≥ 16 px and no horizontal scroll appears.  
   - The results dashboard remains fully visible and scrollable on mobile.

4. **Accessibility**  
   - Each custom checkbox is focusable (`tabindex="0"`).  
   - Pressing Space or Enter while focused toggles the checkbox and updates `aria-checked`.  
   - Screen readers announce the checkbox state (“checked” / “unchecked”).  
   - All interactive elements have visible focus outlines.

5. **Complete Markup**  
   - Every opening tag has a corresponding closing tag.  
   - The page passes W3C HTML5 validation.
