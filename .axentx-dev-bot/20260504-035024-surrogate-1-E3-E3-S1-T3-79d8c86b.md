# /opt/axentx/surrogate-1/README.md  
# surrogate-1-runner  

Parallel public-dataset ingest workers for the  
[axentx/surrogate-1-training-pairs](https://huggingface.co/datasets/axentx/surrogate-1-training-pairs)  
HuggingFace dataset.  

## What This Does  
Every 30 minutes (or on `workflow_dispatch`), GitHub Actions launches **16 parallel runners**. Each runner:  
1. Takes a deterministic 1/16 slice (`slug-hash bucket = SHARD_ID`) of the public dataset list from `bin/dataset-enrich.sh`  
2. Streams, normalizes per-schema, and dedups via the central md5 hash store  
3. Uploads output to a unique path on the dataset repo  

---

## OpenTelemetry Configuration  
Configure telemetry by setting these environment variables:  

```bash
# Set the OTLP endpoint for metrics
export OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:4317

# Set the interval (seconds) for emitting the `fd.scout.open_total` gauge
export FD_SCOUT_INTERVAL=30
```

**Example Workflow**:  
```bash
# Configure OpenTelemetry to send metrics every 30 seconds
export OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:4317
export FD_SCOUT_INTERVAL=30
```

This setup ensures metrics are emitted to the specified OTLP endpoint at the configured interval.  

---

## Key Features  
- **Parallel ingestion**: 16 shards processed concurrently  
- **Schema normalization**: Ensures consistent data
