## TDD Test Plan: Provider-Investor Matching

### 1. Acceptance Criteria

- **On-demand execution**: Matching algorithm triggers only when explicitly invoked via API endpoint or scheduled job, not on every provider data change
- **Top-5 results**: Response contains exactly 5 investor records sorted by match score descending (or fewer if total pool < 5)
- **Audit persistence**: Each match operation creates a record in `match_audit_log` table with provider_id, investor_id, score, timestamp
- **Idempotency**: Re-running matching for same provider within same session returns consistent top-5 (deterministic)
- **Response latency**: Matching completes within 500ms for provider pools up to 10,000 investors

```python
# /opt/axentx/surrogate-1/tests/test_matching.py

import pytest
from unittest.mock import Mock, patch
from matching.engine import MatchingEngine

class TestMatchingEngine:
    
    def test_returns_exactly_five_matches(self):
        """Matching returns top 5 investors sorted by score"""
        engine = MatchingEngine()
        provider = {"id": "p1", "criteria": {"sector": "tech", "min_investment": 10000}}
        investors = generate_investors(20)
        
        results = engine.match(provider, investors)
        
        assert len(results) == 5
        scores = [r["score"] for r in results]
        assert scores == sorted(scores, reverse=True)
    
    def test_persists_audit_record(self):
        """Each match creates audit log entry"""
        engine = MatchingEngine()
        provider = {"id": "p1", "criteria": {}}
        investors = generate_investors(10)
        
        with patch("matching.engine.db") as mock_db:
            engine.match(provider, investors)
            assert mock_db.insert.call_count == 5
    
    def test_empty_investor_pool_returns_empty(self):
        """Zero investors returns empty list, not error"""
        engine = MatchingEngine()
        results = engine.match({"id": "p1"}, [])
        
        assert results == []
    
    def test_fewer_than_five_investors_returns_all(self):
        """If only 3 investors exist, return all 3"""
        engine = MatchingEngine()
        results = engine.match({"id": "p1"}, generate_investors(3))
        
        assert len(results) == 3
    
    def test_on_demand_requires_explicit_call(self):
        """Matching does not auto-trigger on provider update"""
        engine = MatchingEngine()
        with patch.object(engine, "match") as mock_match:
            engine.on_provider_updated({"id": "p1"})
            mock_match.assert_not_called()
```

### 3. Integration Tests

**Happy Path (3-5):**
1. **Full workflow**: POST `/api/match` with provider_id → returns 5 investors → verify audit table has 5 new rows
2. **Concurrent requests**: Two simultaneous match requests for different providers complete without race conditions
3. **Score calculation**: Provider seeking "healthcare" + $50k minimum matches investors with healthcare focus and capacity ≥ $50k ranked higher
4. **API contract**: Response JSON matches schema `{provider_id, matches: [{investor_id, score, rationale}]}`
5. **Pagination**: Second call with `offset=5` returns next 5 matches (if available)

**Edge Cases (2-3):**
1. **Provider not found**: POST with invalid provider_id returns 404 with error message
2. **Database failure during audit**: If audit insert fails, matching still returns results but logs error
3. **Provider with no criteria**: Matching uses default scoring (e.g., random or first N)
