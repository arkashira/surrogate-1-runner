# surrogate-1-runner: Parallel Public-Dataset Ingest Workers

## Overview
Every 30 minutes (or on `workflow_dispatch`), GitHub Actions launches **16 parallel runners**. Each runner processes a deterministic 1/16 slice (`SHARD_ID`) of the public dataset list defined in `bin/dataset-enrich.sh`. The workflow includes:
- Streaming and schema normalization
- Deduplication via central MD5 hash store
- Uploading results to unique paths in the dataset repository

## Quick Start

### Installation
Install the gateway package with pip:
```bash
pip install surrogate1-llm-gateway
```

### Usage Examples

#### Using Claude
```python
from surrogate1_llm_gateway import ClaudeGateway

# Initialize with your API key
gateway = ClaudeGateway(api_key="your_api_key")

# Send a prompt
response = gateway.send_prompt(
    prompt="Explain quantum entanglement in simple terms."
)
print(response)
```

#### Using Minimax
```python
from surrogate1_llm_gateway import MinimaxGateway

# Initialize with your API key

# Send a prompt
response = gateway.send_prompt(
    prompt="Summarize 'Pride and Prejudice' in 50 words."
)
print(response)
```

Both gateways provide:
- Simple `send_prompt` interface
- Built-in authentication
- Rate-limiting and error handling
