| **Two‑stage copy (requirements → code)** | Guarantees Docker layer caching for fast rebuilds in CI/CD. |
| **`pip install --no‑cache‑dir`** | Prevents the image from bloating with pip caches. |
| **Explicit `ENV LOG_FORMAT=json`** | Guarantees structured JSON logs for downstream parsers. |
| **`ENTRYPOINT ["python","bin/run_runner.py"]`** (instead of `CMD`) | Guarantees the runner script is always executed and that the container exits with the script’s exit code – essential for CI pipelines. |
| **Optional Datadog side‑car** (kept as a separate service, not hard‑wired) | Gives observability when the user wants it, but does not force an API key on every deployment. |
| **Resource limits (2 CPU / 2 GiB)** | Matches typical CI‑runner constraints and protects the host. |
| **Log volume mount** | Lets the CI system or an external log‑collector read the JSON logs without needing to `docker exec`. |
| **No exposed ports by default** | The surrogate runner is a batch job, not a web server; exposing port 80 is unnecessary and would confuse users. |
| **`restart: "no"`** | Guarantees the container stops after the run – a CI job should not be auto‑restarted. |

### 1️⃣ Dockerfile – `/opt/axentx/surrogate-1/Dockerfile`

```dockerfile
# --------------------------------------------------------------
#  Surrogate‑1 – CI‑friendly container
# --------------------------------------------------------------
# Base image: latest stable slim Python with security updates
FROM python:3.11-slim

# ---- System utilities -------------------------------------------------
# Install only what we need (ca‑certificates for HTTPS pip installs)
RUN apt-get update && \
    apt-get install -y --no-install-recommends ca-certificates && \
    rm -rf /var/lib/apt/lists/*

# ---- Working directory -------------------------------------------------
WORKDIR /app

# ---- Python dependencies (cached layer) -------------------------------
# Copy just the lock/requirements file first so Docker can reuse the layer
COPY requirements.txt .

# Install the exact versions the project needs
RUN pip install --no-cache-dir -r requirements.txt

# ---- Application code -------------------------------------------------
COPY . .

# ---- Runtime configuration --------------------------------------------
# Structured JSON logging expected by downstream parsers
ENV LOG_FORMAT=json

# ---- Entrypoint --------------------------------------------------------
# The runner script is the *only* thing this container does.
# Using ENTRYPOINT (instead of CMD) makes the exit status of the script
# become the container exit status – perfect for CI/CD.
ENTRYPOINT ["python", "bin/run_runner.py"]
```

**Key points**

* **No `EXPOSE`** – the container is a short‑lived job, not a web service.  
* **No `CMD`** – the `ENTRYPOINT` guarantees the correct command and exit‑code handling.  
* **`ca-certificates`** are the only extra OS package needed; everything else stays in the Python layer.
