## Final Implementation Plan (≤2 h)

Replace `bin/dataset-enrich.sh` with a **manifest-driven, CDN-bypass ingestion worker** (`bin/dataset-enrich.py`) that:

- Accepts `SHARD_ID` and `SHARD_TOTAL` (16) from the GitHub Actions matrix.
- Uses a precomputed `manifest.json` (generated once per date folder on a Mac orchestrator) listing **only immediate file paths** for that folder.  
  - This eliminates recursive `list_repo_files` and avoids HF API 429s during ingestion.
- Downloads each assigned file via **HF CDN** (`https://huggingface.co/datasets/{repo}/resolve/main/{path}`) with no Authorization header (bypasses `/api/` rate limits).
- Streams each file with `datasets` but **never uses `streaming=True` on heterogeneous repos**; instead downloads raw bytes and projects to `{prompt, response}` at parse time to avoid `pyarrow.CastError`.
- Deduplicates via the existing `lib/dedup.py` central md5 store.
- Writes output to `batches/public-merged/{date}/shard{N}-{HHMMSS}.jsonl` containing **only** `{prompt, response}` (no extra metadata columns) with deterministic filenames to prevent cross-shard collisions.
- Exits with standard success/failure codes so GitHub Actions can retry cleanly.

Workflow changes:
- Add a one-time “prepare manifest” job (or run locally on Mac) before the 16-shard matrix to produce `manifest.json` for the target date folder.
- Keep the 16-shard matrix but pass `MANIFEST_URL` (CDN) and `DATE` to each runner.
- Ensure `SHELL=/bin/bash` in any cron or wrapper usage (per past lessons).
