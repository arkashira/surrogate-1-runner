## 1. High‑Level Architecture  

```
+-------------------+      +-------------------+      +-------------------+
|   Collector Agent | ---> |   Storage Layer   | <--- |   Scheduler/Job   |
| (psutil / Cloud‑ |      | (SQLite / PG)     |      | (APScheduler)    |
|  Watcher)         |      +-------------------+      +-------------------+
|                     ^                                 |
|                     |                                 v
|               +-------------------+        +-------------------+
|               |   Rightsizer      | <---- |   Decision Engine |
|               | (AWS/Azure/GCP)   |        | (threshold eval) |
+-------------------+-------------------+-------------------+
          ^                     |
          |                     v
   +-------------------+   +-------------------+
   |   REST API (FastAPI)  |   |   CLI (click)   |
   +-------------------+   +-------------------+
```

* **Collector** – runs on each VM/instance, gathers CPU, memory, disk, network stats (via `psutil` or cloud‑watch APIs) and pushes them to the storage layer (local DB or remote API).  
* **Storage** – abstracted via `BaseStorage`; default implementation `SQLiteStorage` for single‑node, `PostgresStorage` for HA.  
* **Decision Engine** – reads recent metrics, applies **hysteresis‑aware thresholds**, and decides whether a resize is needed.  
* **Rightsizer** – translates a resize request into a cloud‑provider API call (or a dry‑run).  
* **Scheduler** – runs the decision engine every `N` seconds (configurable).  
* **API / CLI** – expose all operations for automation, monito
