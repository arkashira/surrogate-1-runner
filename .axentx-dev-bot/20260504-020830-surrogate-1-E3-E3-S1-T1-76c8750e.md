## 1. Acceptance Criteria  

1. **File Upload** – The form must accept only files with extensions `.tf`, `.tf.json`, `.yaml`, `.yml`, `.tgz`, or `.tar.gz` and reject any other file types.  
2. **Audit Log Upload** – A separate file input must accept only files with extensions `.log`, `.txt`, or `.json`.  
3. **CSRF Protection** – A hidden input named `csrf_token` must be present in the form, populated by the server, and validated on submission.  
4. **JavaScript UI Feedback** – When a file is selected, the UI must display the file name(s) and enable the “Generate PDF” button only after at least one config file is chosen.  
5. **Graceful Degradation** – Without JavaScript, the form must still submit, the server must process the files, and the response page must contain a direct link to the generated PDF.  
6. **Download Link Availability** – After successful processing, a link to the generated PDF must be visible and functional (HTTP 200).  
7. **Accessibility** – All form controls must have associated `<label>` elements and appropriate ARIA attributes for screen readers.
