## 1. Acceptance Criteria  

- **AC1** – The audit configuration must expose an `enabled` boolean flag that defaults to `false`.  
- **AC2** – The configuration must include a `target_table` string that defaults to `"audit_logs"`.  
- **AC3** – The configuration must allow a list of `loggable_fields` (e.g., `["request", "response", "timestamp", "user", "pattern_version"]`) that defaults to the full set.  
- **AC4** – All fields must be overridable via environment variables prefixed with `AUDIT_` (e.g., `AUDIT_ENABLED`, `AUDIT_TARGET_TABLE`, `AUDIT_LOGGABLE_FIELDS`).  
- **AC5** – The YAML schema must be syntactically valid and loadable by the existing config loader without errors.  
- **AC6** – When the `enabled` flag is `false`, no audit logs should be emitted (verified via integration test).  
- **AC7** – When the `enabled` flag is `true`, audit logs must be written to the configured `target_table` with only the specified `loggable_fields`.
