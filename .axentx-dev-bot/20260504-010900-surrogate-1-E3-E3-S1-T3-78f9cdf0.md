#### Overview

It ensures correct behavior through structured logging, graceful degradation, and clear state tracking — all while maintaining safety by avoiding unsafe operations or assumptions.

---

### Key Features

1. **Safe Execution**
   - Uses `subprocess.run(..., check=True)` to ensure failures are raised and handled explicitly.
   - Avoids unsafe shell commands or direct manipulation of critical system components without validation.

2. **Structured Logging**
   - Logs both success and failure events with descriptive messages.
   - Tracks service state transitions (before/after) for debugging and audit trails.

3. **Graceful Degradation**
   - Falls back to disabling non-critical services when primary actions fail.
   - Ensures continued operation even under partial failure conditions.

4. **Modular Design**
   - Encapsulates logic within a `TelemetryAgent` class for reusability and testability.
   - Separates concerns between service control, logging, and error handling.

5. **Actionable Output**
   - Provides actionable feedback via logs that can be used for monitoring, alerting, or automated recovery systems.
   - Clear distinction between expected outcomes (`start`, `stop`) and fallback behaviors (`graceful_degradation`).
