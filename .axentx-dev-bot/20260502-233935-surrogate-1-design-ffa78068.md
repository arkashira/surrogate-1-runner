### 1. Acceptance criteria
- `bin/snapshot.sh` exists, is executable (`chmod +x`), and produces `snapshot.txt` containing one file path per line with no empty lines; exit code 0 on success.
- `bin/snapshot.sh` logs repo, path, and dataset version/tag/commit (or latest timestamp) to stdout/stderr in parseable key=value format.
- `dataset-enrich.sh` reads `snapshot.txt` when present and processes every listed file; if `snapshot.txt` is missing, it falls back to live query and logs `fallback=live`.
- `dataset-enrich.sh` exit code is 0 when all listed files are processed successfully; non-zero if any listed file cannot be processed (and logs the failing file).
- GitHub Actions workflow references `snapshot.txt` (committed or downloaded) and runs `dataset-enrich.sh` without invoking live dataset queries (detectable by absence of `huggingface_hub` download calls in job logs).
- Smoke test: for a pinned dataset revision, enriched output from snapshot run matches live-query run within 0% drift (exact match) for file list and metadata counts; tolerance ≤1% for non-deterministic fields.
- Snapshot and enrichment logs include `repo=` and `path=` and `dataset_version=` (or commit hash/tag) so failures are traceable.
