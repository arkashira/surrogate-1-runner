import time
from datetime import timedelta
from threading import Thread
import logging

class SessionManager:
    """
    A robust session management system with idle timeout functionality.
    Automatically terminates inactive sessions while allowing activity tracking.
    """

    def __init__(self, idle_timeout=300):
        """
        Initialize the session manager with configurable idle timeout.

        Args:
            idle_timeout (int): Maximum allowed inactivity time in seconds (default: 300)
        """
        self.sessions = {}
        self.idle_timeout = idle_timeout
        self.logger = logging.getLogger(__name__)
        logging.basicConfig(level=logging.INFO)

    def start_session(self, session_id):
        """
        Start a new session with the given ID.

        Args:
            session_id (str): Unique identifier for the session
        """
        if session_id in self.sessions:
            self.logger.warning(f"Session {session_id} already exists")
            return

        self.sessions[session_id] = {
            'last_activity': time.time(),
            'active': True
        }

        # Start monitoring thread
        Thread(
            target=self._monitor_idle_time,
            args=(session_id,),
            daemon=True
        ).start()
        self.logger.info(f"Started session {session_id}")

    def _monitor_idle_time(self, session_id):
        """
        Internal method to monitor session activity and terminate if idle.

        Args:
            session_id (str): Session ID to monitor
        """
        while True:
            if session_id not in self.sessions or not self.sessions[session_id]['active']:
                break

            current_time = time.time()
            last_activity = self.sessions[session_id]['last_activity']

            if current_time - last_activity > self.idle_timeout:
                self.terminate_session(session_id)
                break

            time.sleep(60)  # Check every minute

        """

APPROVE: The implementation demonstrates a clear, pragmatic approach to session lifecycle management with idle timeout handling. It introduces a per‑session monitoring thread, activity tracking, and a cleanup hook, which are all reasonable first steps toward a production‑ready system.

**Acceptance Criteria**

- **Thread‑sleep interval**: The `_monitor_idle_time` method sleeps for 60 s, which is unsuitable for short idle timeouts used in tests. The interval should be configurable (e.g., via an `check_interval` parameter) or set to a small default (e.g., 1 s) to make unit tests deterministic.
- **Graceful thread termination**: The loop currently relies on the session being removed or marked inactive. Adding a `threading.Event` to signal shutdown would make the thread exit cleanly even if the session dictionary is altered elsewhere.
- **Logging level**: `logging.basicConfig(level=logging.INFO)` is called in `__init__`, which may override application‑wide logging configuration. Consider removing this call or allowing the level to be passed in.
- **Duplicate session handling**: The current implementation logs a warning but silently ignores the request. Tests should verify that a duplicate `start_session` call does not create a second monitoring thread or overwrite the existing session data.
- **Unit test completeness**: The `test_duplicate_session` method is truncated. It should be completed to assert that the session count remains unchanged after attempting to start a duplicate session.
- **Resource cleanup placeholder**: `_cleanup_resources` currently only logs. In a real deployment, this should invoke actual cleanup logic (e.g., deleting Kubernetes pods, closing DB connections). The placeholder is acceptable for a discovery‑stage prototype but should be documented as a TODO.

With these adjustments, the code will be more robust, testable, and ready for integration into a larger system.
