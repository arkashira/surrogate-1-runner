### 3. **Integration Tests**

#### Happy Paths:
1. **User selects a recorded stream → playback starts → progress updates → ends naturally**
   - Load real `RecordedStream` from storage
   - Call `play()` → verify audio output via mock speaker
   - Assert progress increases over time
   - Confirm completion callback is fired

2. **User plays → pauses → resumes → completes playback**
   - Simulate UI interaction
   - Verify state transitions: playing → paused → playing
   - Ensure no audio during pause

3. **Multiple recordings: playback one after another**
   - Switch between two recorded streams
   - Ensure correct file is played each time
   - No cross-talk or buffering issues

4. **Playback of large file (30+ mins)**
   - Validate memory usage remains stable
   - No stuttering or crashes

5. **Playback after app restart**
   - Reopen app, select existing recording
   - Confirm playback works as expected

#### Edge Cases:
1. **Playback of corrupted or incomplete recording**
   - Expected: Show error toast, do not crash
2. **No audio output device available (e.g., Bluetooth disconnected mid-playback)**
   - Expected: Pause playback, notify user
3. **File deleted between selection and playback**
   - Expected: Handle `FileNotFoundException`, show alert
