## 1. Acceptance Criteria  
* **CSV size limit** – The endpoint rejects any multipart upload larger than **10 MB** with a `413 Payload Too Large` response.  
* **Job creation** – A successful POST returns **HTTP 202** and a JSON payload containing a non‑empty `job_id` (UUID‑v4).  
* **Raw storage path** – The uploaded file is saved under `/data/finance/raw/<YYYY‑MM‑DD>.csv` where the date is derived from the request header `X-Upload-Date` (or defaults to server date). The directory must exist after the first successful upload.  
* **Row validation** – For each CSV row, `validate_row` returns `(True, "")` when the row has **≥ 3** non‑empty columns; otherwise it returns `(False, <reason>)`. Invalid rows are **logged** and listed in the response payload under `invalid_rows`.  
* **Config correctness** – `config.py` loads all required constants (`BASE_RAW_DIR`, `LOG_DIR`, broker/backend URLs) from environment variables, creates `BASE_RAW_DIR` and `LOG_DIR` if missing, and raises a clear error if mandatory vars are absent.  
* **Celery readiness** – Importing `celery_app` does **not** raise exceptions; a connection test (`celery_app.control.ping()`) succeeds within **5 s** when the broker is reachable.  
* **Logging** – The dedicated logger writes entries to `logs/finance_ingest.log`; the file is created on first log write and contains a timestamped entry for each processed row (valid or invalid).
