## Signal generator

The pipeline includes an example signal generator (`exampleSignalGenerator`) that accepts `{ symbol: string, horizon: string }` and returns a deterministic `SignalOutput`:

- `signal_id` — non-empty string
- `side` — `"long" | "short" | "neutral"`
- `price` — finite number > 0
- `confidence` — number in [0,1]

The generator is side-effect-free and replaceable; swapping implementations must preserve the `SignalOutput` contract.

## Pipeline runner

- Stages can be async; the runner awaits resolution and validation before proceeding.
- Input references of the form `${stageName}.output.field}` are resolved before stage execution. If resolution fails, the runner throws a `PipelineError` with code `RESOLUTION_FAILED`.
- If a signal stage output is missing required fields or violates constraints, the pipeline aborts with code `SIGNAL_INVALID` (or `SIGNAL_MISSING` when absent).
- The runner aborts on the first stage failure and propagates a `PipelineError` with a stage-specific code (`STAGE_FAILED` when a stage throws).
