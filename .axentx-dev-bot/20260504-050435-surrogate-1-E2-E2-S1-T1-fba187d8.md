Welcome to Axentx Surrogate‑1 Onboarding Guide
================================================

1. Introduction
   • Surrogate‑1 is a lightweight, high‑throughput data ingestion platform that normalizes, deduplicates, and pushes datasets to HuggingFace.
   • Key benefits: zero‑config parallel ingestion, schema‑agnostic, audit‑ready logs, and instant HF dataset publishing.

2. Getting Started
   a. Create an Axentx account at https://app.axentx.com/signup
   b. Verify your email and log in.
   c. In the dashboard, click **Create Project** → name it *MyFirstProject*.
   d. Note the Project ID (e.g., `proj_12345`) – you’ll need it for API calls.

3. Core Features
   • **Dataset Ingestion** – Upload a CSV/JSON/Parquet file via the UI or `axentx ingest` CLI.
   • **Parallel Runner** – Surrogate‑1 automatically shards the file and runs 8 workers; you can tweak `--workers` if needed.
   • **Schema Normalization** – The engine auto‑detects field types and casts to a canonical schema; review the preview before commit.
   • **Deduplication** – Uses a SHA‑256 hash of each row; duplicates are dropped and logged.
   • **Upload to HuggingFace** – After ingestion, click **Publish → HuggingFace**; supply your HF token and target repo.

4. Advanced Configuration
   a. **Custom Ingestion Parameters** – Edit `config.yaml` in the project folder:
      ```
      delimiter: ","
      header: true
      batch_size: 5000
      ```
   b. **Webhooks** – In the dashboard → *Settings → Webhooks*, add a URL to receive `ingestion.completed` events.
   c. **Monitoring & Logging** – View real‑time logs in the dashboard or tail the log file:
      ```
      tail -f /var/log/axentx/surrogate.log
      ```

5. Troubleshooting
   • **Common Issues**  
     - *“File too large”* – split the file or increase `--max_file_size` in `config.yaml`.  
     - *“Duplicate rows detected”* – review the `dedupe.log` for offending keys.  
     - *“HF upload fails”* – ensure your HF token has `write` scope and the repo exists.
   • **Support** – Email support@axentx.com or open a ticket in the dashboard.

6. Next Steps
   • Explore the **Analytics** tab to see ingestion metrics.  
   • Join the Axentx community Slack (#surrogate-1) for tips.  
   • Schedule a quarterly review to update the ingestion pipeline as data grows.

Resources
---------
• Documentation: https://docs.axentx.com/surrogate-1  
• API Reference: https://api.axentx.com/docs  
• Community Forum: https://forum.axentx.com
```
