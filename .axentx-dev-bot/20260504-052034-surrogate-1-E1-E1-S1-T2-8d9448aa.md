## 1. Acceptance Criteria
- **AC‑1**: The wizard UI displays exactly three sequential steps with headings “1. Select Data Source”, “2. Configure Ingestion Options”, and “3. Review & Launch”.
- **AC‑2**: Navigation controls (`Next`, `Back`, `Finish`) are only enabled when all required fields on the current step are valid.
- **AC‑3**: Step 1 requires the user to select a data source from a dropdown; the list must contain at least the options “Reddit”, “Twitter”, and “Custom API”.
- **AC‑4**: Step 2 contains fields for `batchSize` (integer ≥ 1), `schedule` (cron expression), and an optional `filterExpression`; validation errors are shown inline.
- **AC‑5**: Step 3 renders a read‑only summary table that mirrors the selections from steps 1‑2 and displays an **estimated monthly cost** calculated by the backend endpoint `/api/wizard/estimate`.
- **AC‑6**: Clicking **Finish** sends a POST to `/api/wizard/launch` with the aggregated payload and shows a success toast; the wizard then redirects to the pipeline list page.
- **AC‑7**: All HTML templates are served with a `Content‑Security‑Policy` header that includes `script-src 'self'` and `style-src 'self'`.
