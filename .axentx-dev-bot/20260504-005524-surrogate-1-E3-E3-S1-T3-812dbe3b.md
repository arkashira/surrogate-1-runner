| **`match_commit_to_cost_records`** that filters by **repo, branch and a configurable time window** (the 1‑hour window from candidate 1 is kept as the default, but the function is generic enough to be reused with any window) | Gives a clear, testable contract and lets callers decide the granularity. |

| **Comprehensive unit‑tests** covering (1) a successful match, (2) no match, (3) aggregation, and (4) DB insertion via a mock | Guarantees correctness and gives a template for future test‑cases. |
| **Error handling** (`DatabaseError`) that implements `std::error::Error` and `Debug/Display` | Makes the API ergonomic for callers. |
| **No contradictory logic** – the time‑window is a single, well‑documented parameter, and the “store” function always works on the *aggregated* `CommitCost` rather than a list of raw `CostRecord`s. | Resolves the clash between “return a vector of matching records” and “return a summed cost”.
