## 1. Acceptance Criteria
- **Endpoint Availability** – GET `/v1/datadog/metrics` must exist and respond within 200 ms under normal load.
- **Query Parameters** – The endpoint accepts `type` (string) and `volume` (integer) as query parameters; missing or invalid values return **400 Bad Request**.
- **Successful Response** – Valid requests return **HTTP 200** with a JSON body containing a `metrics` array of synthetic datapoints matching the requested `volume`.
- **Rate Limiting** – Exceeding **100 requests per minute** from the same IP returns **429 Too Many Requests** with a `Retry-After` header.
- **Content-Type** – All successful responses include `Content-Type: application/json`.
- **Statelessness** – The endpoint does not persist state; each request generates fresh synthetic data.
- **Error Handling** – Any unexpected server error returns **500 Internal Server Error** with a JSON error message.
