## 1.  One‑file, one‑purpose `.gitlab-ci.yml`

The goal is to keep the pipeline **simple, readable, and fully reproducible** while still exposing all the advanced features that the two proposals offered:

* **Parallel test stages** – run the full test matrix and a “filtering” test that only exercises the filtering logic.
* **Coverage** – generate a Cobertura XML file, upload it to GitLab, and set a pipeline variable that can be used by downstream jobs.
* **Artifacts** – keep a short‑lived filtering result JSON, a coverage report, and a tiny “API endpoint” file that can be used by a downstream service.
* **Linting** – run Ruff in a separate job that is *allowed to fail* so that the pipeline never blocks a merge.
* **Conditional execution** – only run the coverage job on the default branch or when a merge‑request is present.
* **Explicit environment variables** – keep the CI variables in one place so that the adapter can read them reliably.

```yaml
# .gitlab-ci.yml
# -------------------------------------------------------------
# 1️⃣  Global configuration
# -------------------------------------------------------------
stages:
  - lint
  - test
  - coverage
  - artifacts

variables:
  PYTHON_VERSION: "3.11"
  COVERAGE_THRESHOLD: "80"          # percent
  ARTIFACT_RETENTION_DAYS: "30"

# -------------------------------------------------------------
# 2️⃣  Shared before_script
# -------------------------------------------------------------
before_script:
  - python --version
  - pip install -e .[dev]   # installs pytest, coverage, ruff, etc.

# -------------------------------------------------------------
# 3️⃣  Linting – can fail without blocking the pipeline
# -------------------------------------------------------------
lint:
  stage: lint
  image: python:${PYTHON_VERSION}-slim
  script:
    - pip install ruff
    - ruff check src/ tests/
  allow_failure: true

# -------------------------------------------------------------
# 4️⃣  Full test matrix
# -------------------------------------------------------------
test:full:
  stage: test
  image: python:${PYTHON_VERSION}-slim
  script:
    - pytest tests/ -v --tb=short
  artifacts:
    when: always
    paths:
      - test-results.xml
    reports:
      junit: test-results.xml
  coverage: '/TOTAL.*\s+(\d+%)$/'

# -------------------------------------------------------------
# 5️⃣  Filtering test – only the filtering logic
# -------------------------------------------------------------
test:filtering:
  stage: test
  image: python:${PYTHON_VERSION}-slim
  script:
    - pytest tests/test_filtering.py -v
    - python -c "
        from src.gitlab_adapter import GitLabAdapter
        adapter = GitLabAdapter()
        adapter.report_filtering_results('filtering-results.json')
      "
  artifacts:
    paths:
      - filtering-results.json
    reports:
      junit: filtering-test-results.xml

# -------------------------------------------------------------
# 6️⃣  Coverage – only on default branch or MR
# -------------------------------------------------------------
coverage:report:
  stage: coverage
  image: python:${PYTHON_VERSION}-slim
  script:
    - pytest tests/ --cov=src --cov-report=term --cov-report=xml:coverage.xml
    - python -c "
        from src.gitlab_adapter import GitLabAdapter
        adapter = GitLabAdapter()
        adapter.upload_coverage_report('coverage.xml')
        adapter.set_pipeline_metadata('coverage', 'passed')
      "
  coverage: '/TOTAL.*\s+(\d+%)$/'
  artifacts:
    reports:
      coverage_report:
        coverage_format: cobertura
        path: coverage.xml
    paths:
      - coverage.xml
      - coverage.html
  rules:
    - if: '$CI_MERGE_REQUEST_IID'
    - if: '$CI_COMMIT_BRANCH == $CI_DEFAULT_BRANCH'

# -------------------------------------------------------------
# 7️⃣  Artifacts – filtering results + a tiny API endpoint
# -------------------------------------------------------------
artifacts:filtering:
  stage: artifacts
  image: python:${PYTHON_VERSIO
