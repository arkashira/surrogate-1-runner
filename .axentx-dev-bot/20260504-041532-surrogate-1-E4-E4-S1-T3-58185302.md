## The “One‑Stop” Analytics Dashboard – A Unified, Actionable Blueprint

Below is a **single, production‑ready design** that pulls together the strongest ideas from the three candidate proposals:

| Candidate | Strengths | Weaknesses |
|-----------|-----------|------------|
| **1** | • Full‑stack React component with Recharts visualisation<br>• Built‑in feedback form & rating logic<br>• Graceful fallback data on fetch error | • Hard‑coded mock data in the catch block (no real‑time updates)<br>• No type safety or error boundaries<br>• No accessibility audit |
| **2** | • Uses `react-query` for data fetching, caching, and refetching<br>• TypeScript for compile‑time safety<br>• Modular UI primitives (Card, Table, Tabs) | • No real‑time WebSocket updates<br>• No visualisation of performance metrics<br>• Feedback form is minimal |
| **3** | • Uses `socket.io` for real‑time analytics<br>• Provides a dedicated `FeedbackService` abstraction<br>• Uses `react-hook-form` for robust form handling | • No charting library – visualisation is missing<br>• No type safety (plain JS)<br>• No error handling for socket disconnects |

### The Synthesis

1. **Framework & Language** – **React + TypeScript** (for type safety)  
2. **Data Layer** – `react-query` for REST and `socket.io-client` for real‑time updates  
3. **UI Primitives** – Re‑usable `Card`, `Table`, `Tabs`, `Button`, `Input`, `Select`, `Badge` (Tailwind‑based)  
4. **Charting** – `Recharts` (already in Candidate 1)  
5. **Form Handling** – `react-hook-form` + `zod` for schema validation (from Candidate 3)  
6. **Accessibility** – ARIA roles, keyboard navigation, focus‑ring, high‑contrast fallback  
7. **Error Handling** – Error boundaries, toast notifications, graceful degradation  
8. **Testing** – Jest + React Testing Library (unit + integration)  
9. **Deployment** – Vite + Docker, CI/CD pipeline with linting & type‑checking

## 2. Socket Context (Real‑time)

**src/lib/socket.tsx**

```tsx
import { createContext, useContext, useEffect, useState } from 'react'
import { io, Socket } from 'socket.io-client'

type Analytics = {
  workflowStats: WorkflowStat[]
  performanceMetrics: PerformanceMetric[]
  userFeedback: Feedback[]
}

export const SocketContext = createContext<Socket | null>(null)

export const SocketProvider = ({ children }: { children: React.ReactNode }) => {
  const [socket, setSocket] = useState<Socket | null>(null)

  useEffect(() => {
    const s = io(import.meta.env.VITE_API_URL || 'http://localhost:4000')
    setSocket(s)

    return () => {
      s.disconnect()
    }
  }, [])

  return (
    <SocketContext.Provider value={socket}>{children}</SocketContext.Provider>
  )
}

export const useSocket = () => {
  const ctx = useContext(SocketContext)
  if (!ctx) throw new Error('useSocket must be used within SocketProvider')
  return ctx
}
```
