        }

## 1. Acceptance Criteria  
- **Rule Definition**: The system must load task assignment rules from a configuration dictionary at engine initialization.  
- **Role Mapping**: A task can only be assigned to a team member whose role matches the rule’s required role.  
- **Workload Threshold**: A team member’s current workload (number of active tasks) must be below a configurable threshold before assignment.  
- **Automatic Assignment**: When a task is created via `TaskManager.create_task`, the workflow engine automatically assigns it to an eligible member.  
- **Logging**: Each assignment is logged with a message containing the task ID and assignee’s name.  
- **Visibility**: The assigned member’s name appears in the task’s history record.  
- **Deterministic Selection**: If multiple eligible members exist, the engine selects the one with the lowest current workload.

    ]

def test_load_rules(engine):
