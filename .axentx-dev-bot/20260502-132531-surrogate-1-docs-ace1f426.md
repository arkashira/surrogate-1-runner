## Deduplication index (CDN)

- A per-shard, append-only deduplication index is published to the CDN for public reads (no token required).
- Listing processed slugs: `python3 bin/cross_run_dedup.py --list` exits 0 and prints newline-separated slugs when the CDN file exists; exits 0 and prints nothing on 404; exits non-zero on non-404 HTTP errors.
- Appending slugs: `python3 bin/cross_run_dedup.py --append --slug owner/repo --shard 1 --ts 20240101-000000` appends the slug to `index/processed_slugs_shard1-20240101-000000.txt` and uploads it to the `main` branch of the dataset (requires `HF_TOKEN`).
- CDN files are publicly readable; `HF_TOKEN` is only used for uploads.
- Per-shard append files are created atomically (write-then-close) and contain only the slugs processed by that shard+timestamp run.
- `bin/dataset-enrich.sh` skips any slug present in the preloaded PROCESSED set (exact match) before enrichment steps.
- Intra-run/source SQLite dedup behavior is unchanged: for a given shard run, the unique slug count in SQLite equals the number of distinct slugs processed (no duplicates within the run).
