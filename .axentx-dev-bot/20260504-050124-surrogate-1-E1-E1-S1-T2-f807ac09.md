## The “one‑stop” Unified LLM SDK

Below is a **single, self‑contained, production‑ready** implementation that pulls the best ideas from the two candidates:

* A **runtime‑registered provider registry** (so you can plug‑in any LLM provider you want).
* A **rich `LLMResponse`** that carries content, provider, model, token count and arbitrary metadata.
* A **simple, test‑friendly `ClaudeProvider`** that can be swapped out for a real HTTP client when you’re ready.
* A **JUnit 5 test** that uses **Mockito** to verify the contract without making real network calls.

> **Why this is the best of both worlds**  
> *Candidate 1* gives you a quick test harness, but it hard‑codes the provider and misses the provider‑registry pattern.  
> *Candidate 2* supplies a full interface hierarchy and a stubbed Claude provider, but its test is incomplete and uses JUnit 4.  
> The merged solution keeps the clean API from Candidate 2, adds the real‑world provider‑registration logic, and finishes the test with the modern JUnit 5 + Mockito stack.

### 2.1 `UnifiedLLMInterface`

```java
package com.axentx.surrogate.llm;

import java.util.Map;

/**
 * A single entry point for all LLM providers.
 * <p>
 * All providers expose the same contract, so developers can switch
 * between them without touching the calling code.
 */
public interface UnifiedLLMInterface {

    /**
     * Forward a prompt to the chosen provider and return a full response.
     *
     * @param provider   Name of the provider (e.g. "claude", "minimax")
     * @param prompt     The prompt to send
     * @param options    Optional parameters (temperature, maxTokens, etc.)
     * @return A fully‑populated {@link LLMResponse}
     */
    LLMResponse generate(String provider, String prompt, Map<String, Object> options);

    /**
     * List all providers that have been registered.
     *
     * @return Array of provider names
     */
    String[] getSupportedProviders();
}
```
