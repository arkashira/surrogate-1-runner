**Deterministic date-partitioning + CDN-bypass ingestion**  
Fix the two root causes in the last diagnosis:  
1. Workers re-stream the entire public dataset list every run → redundant HF API calls + non-deterministic outputs.  
2. No date-partitioned output paths → re-runs overwrite history and training sees unstable file lists.  
3. Workers hit HF API during data load → rate-limit risk and quota waste.

Ship in <2h by:
- Adding a single pre-flight script on the Mac orchestrator that lists today’s folder once via HF API (after rate-limit window) and writes `file-list.json`.  
- Embedding that file list in the GitHub Actions matrix so each shard only processes its deterministic slice.  
- Switching workers to download via raw CDN URLs (no auth, no API calls) and write to `batches/public-merged/YYYY/MM/DD/shard<N>-<HHMMSS>.jsonl`.  
- Reusing existing dedup logic unchanged.

### 1) Mac orchestrator: pre-flight file list (run once per cron tick)
```bash
#!/usr/bin/env bash
# bin/list-today.sh
# Run on Mac after rate-limit window clears (or via cron with retry/backoff).
set -euo pipefail

REPO="axentx/surrogate-1-training-pairs"
DATE=$(date -u +%Y-%m-%d)
OUT="file-list-${DATE}.json"

# List only today's folder (non-recursive) to avoid pagination explosion.
# If folder doesn't exist yet, produce empty list.
if curl -sf "https://huggingface.co/api/datasets/${REPO}/tree?path=public/${DATE}&recursive=false" > "${OUT}.tmp"; then
  # Keep only files (not dirs) and their raw CDN paths
  jq -r '[.[] | select(.type=="file") | "https://huggingface.co/datasets/'${REPO}'/resolve/main/public/'${DATE}'/" + .path] | sort' "${OUT}.tmp" > "${OUT}"
  rm "${OUT}.tmp"
else
  echo '[]' > "${OUT}"
fi

echo "Wrote ${OUT} with $(jq length ${OUT}) files"
```
