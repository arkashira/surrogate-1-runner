## Final Consolidated Implementation Plan (≤2h)

**Highest-value improvement:** Eliminate Hugging Face API rate-limit exposure during ingestion and training by pre-computing deterministic file lists on the Mac orchestrator and switching all data movement to CDN-only fetches. Embed the file list as a small JSON artifact so Lightning training uses zero HF API calls. Reuse running Studio instances and guard against idle stops to preserve quota and avoid cold-start delays.

### Concrete, ordered steps

1. **Add Mac-side pre-list script** (`bin/list-date-folder.sh`)  
   - Runs on orchestrator (Mac) after any rate-limit window clears.  
   - Calls `list_repo_tree(path, recursive=False)` for one date folder (e.g., `public-raw/2026-05-03`).  
   - Emits `file-list-<date>.json` containing `{ "date": "...", "files": [...], "base": "public-raw/2026-05-03" }`.

2. **Embed file list into repo and propagate via CI**  
   - Commit `file-list-<date>.json` under `file-lists/` (small JSON, negligible commit load).  
   - In GitHub Actions, have the Mac script optionally upload the JSON as a workflow artifact.  
   - Each shard runner downloads the artifact and uses its deterministic 1/16 slice (hash-based shard assignment).

3. **Update ingestion worker to use CDN-only downloads**  
   - Replace any `hf_hub_download`/`load_dataset` calls for raw files with direct `requests.get` to:  
     `https://huggingface.co/datasets/axentx/surrogate-1-training-pairs/resolve/main/{path}`  
   - Keep `lib/dedup.py` unchanged (central md5 store still used).  
   - Project to `{prompt, response}` at parse time; do not add extra columns.

4. **Add small CDN utility** (`lib/hf_cdn.py`)  
   - Expose `stream_parquet_from_cdn(repo, path)` that yields rows without HF API calls.  
   - Use `requests` + `pyarrow.parquet` and minimal schema handling.

5. **Update Lightning training to accept file list and use CDN**  
   - Accept `file_list_path` argument (CLI or DataModule init).  
   - During `DataLoader` init, stream from CDN URLs (no HF API).  
   - Cache downloaded shards locally in `/tmp/surrogate_cache` to avoid re-download across epochs.  
   - Use an `IterableDataset` that yields only `{prompt, response}` rows.

6. **Add Studio reuse + idle-resume guard**  
   - Before `Studio(create_ok=True)`, enumerate `Teamspace.studios` and reuse a running one with matching name.  
   - Before each `.run()`, check `studio.status`; if stopped, restart with `target.start(machine=Machine.L40S)`.

### 1. Mac orchestrator: list-date-folder.sh
```bash
#!/usr/bin/env bash
set -euo pipefail

REPO="axentx/surrogate-1-training-pairs"
DATE="${1:-$(date +%F)}"          # e.g. 2026-05-03
OUTDIR="file-lists"
OUTFILE="${OUTDIR}/file-list-${DATE}.json"

mkdir -p "${OUTDIR}"

python3 - <<PY
import json, os
from huggingface_hub import HfApi

api = HfApi()
repo = os.environ.get("REPO", "axentx/surrogate-1-training-pairs")
date = os.environ.get("DATE")
base = f"public-raw/{date}"

tree = api.list_repo_tree(repo=repo, path=base, recursive=False)
files = [item.path for item in tree if not item.path.endswith("/")]

payload = {
    "date": date,
    "base": base,
    "files": sorted(files)
}

out = os.environ.get("OUTFILE")
with open(out, "w") as f:
    json.dump(payload, f, indent=2)
print(f"Wrote {len(files)} files to {out}")
PY
```
