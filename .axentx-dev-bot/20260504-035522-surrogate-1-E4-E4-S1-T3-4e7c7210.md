## 1. Acceptance Criteria
- **AC1** – `BridgeModule.invoke(modelId, prompt, options)` returns a Promise that resolves to the exact JSON payload returned by the native bridge when the call succeeds.  
- **AC2** – When the native bridge throws an error (e.g., network failure, rate‑limit exceeded, or backend validation error), the Promise rejects with an instance of `RuntimeException` that contains the original error message and a `code` property matching the backend error code.  
- **AC3** – The method forwards `options` (e.g., `temperature`, `maxTokens`) unchanged to the native bridge and receives them back in the response payload.  
- **AC4** – Rate‑limit logic is respected: if the bridge reports a `429` error, the wrapper should throw a `RuntimeException` with a `code` of `RATE_LIMIT_EXCEEDED`.  
- **AC5** – Cost‑tracking metadata (`cost`, `tokensUsed`) returned by the bridge is present in the resolved payload and can be queried by the workflow engine.  
- **AC6** – The wrapper does not swallow any unexpected errors; any unhandled exception propagates as a `RuntimeException`.
