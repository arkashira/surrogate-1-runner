Replace `bin/dataset-enrich.sh` with a **manifest-driven, CDN-only ingestion worker** (`bin/dataset-enrich.py`) that:

- Accepts `SHARD_ID`, `SHARD_TOTAL=16`, `DATE`, `HF_TOKEN`, `REPO_ID` (default: `axentx/surrogate-1-training-pairs`)
- Uses a **single pre-generated `manifest.json`** (produced by the Mac orchestrator via `list_repo_tree`) to eliminate recursive API calls and 429s
- Downloads exclusively via **HF CDN** (`https://huggingface.co/datasets/{repo}/resolve/main/{path}`) with **no Authorization header** during ingestion to bypass API rate limits entirely
- Projects heterogeneous file schemas to `{prompt, response}` only at parse time (avoids `pyarrow.CastError`)
- Deduplicates via central md5 store (`lib/dedup.py`) and writes to deterministic shard outputs:
  ```
  batches/public-merged/<date>/shard<N>-<HHMMSS>.jsonl
  ```
- Handles HF 429/128-commit limits: spreads writes across 5 sibling repos when needed (hash-slug → repo selection)
- Reuses running Lightning Studio when present (saves quota) and respects idle-stop behavior
- Uses Bearer auth for Kaggle KGAT pushes (bypasses CLI 2.1.0 issues) with `isPrivate: true` to skip phone verification

    """Deterministic shard assignment by slug hash."""
