## 1. Acceptance Criteria  

- **CLI Parameters**  
  - `--type <datatype>` must accept a predefined list (`metrics`, `traces`, `logs`) and default to `metrics`.  
  - `--volume <number>` must be a positive integer; default is `1000`.  
  - `--format <format>` must accept `json` or `csv`; default is `json`.  

- **Data Generation**  
  - When invoked, the CLI must produce exactly `<volume>` records of the specified `<datatype>` within 5 seconds for volumes ≤ 10,000.  
  - Each record must contain a timestamp, a unique ID, and a payload appropriate to the datatype.  

- **Batch vs Real‑Time**  
  - If `--batch` flag is present, all records are generated before any are output.  
  - Without `--batch`, records are streamed to stdout as they are generated.  

- **HTTP Serving**  
  - When `--serve <port>` is provided, the CLI starts an HTTP server on the given port.  
  - A GET request to `/data` returns the generated data in the requested format.  
  - The server must respond within 200 ms for requests when data is already generated.  

- **Error Handling**  
  - Invalid parameters produce a non‑zero exit code and a clear error message.  
  - Server port conflicts return a descriptive error and exit.
