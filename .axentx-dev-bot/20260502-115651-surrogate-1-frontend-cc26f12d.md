## dev — claude/llm-fallback-chain @ 2026-05-02T11:56:51.589599Z

### 1. Diagnosis (integrated)
- **No persisted manifest** (frontend + training): current flows re-enumerate HF repos on every run via authenticated API, causing rate-limit exposure (429s), quota burn, and non-deterministic training inputs.
- **Authenticated API on hot path**: training uses `load_dataset(streaming=True)` or repeated `list_repo_files` calls; frontend may do similar enumeration. This burns quota and increases latency.
- **Schema hazards**: `load_dataset(streaming=True)` with heterogeneous parquet schemas causes `pyarrow` CastErrors.
- **No CDN bypass**: authenticated calls are used for file downloads that could use public CDN, slowing fetches and consuming quota.
- **Missing UI feedback**: operators lack lightweight visibility into dataset readiness (folders, last sync, file counts) before training starts.
- **No commit-cap mitigation**: ingestion can hit HF’s 128-commit/hr repo limit; no deterministic repo-to-sibling routing to spread commits.
- **No studio reuse** (training infra): creating new Lightning Studios wastes quota and increases costs.

### 2. Proposed change (integrated)
Create a single, deterministic pipeline that:
- Persists a manifest of HF repo files (date-scoped) once, then reuses it for both frontend listing and training fetches.
- Replaces `load_dataset(streaming=True)` with CDN-only `hf_hub_download`/direct CDN URLs enumerated from the manifest.
- Adds lightweight UI feedback showing manifest status and file counts.
- Spreads HF commits across 6 sibling repos via deterministic routing to avoid 128-commit/hr caps.
- Reuses existing Running Lightning Studios instead of creating new ones.

Files to modify/create:
- `/opt/axentx/surrogate-1/frontend/src/lib/dataSync.js` (new): manifest generator + CDN router.
- `/opt/axentx/surrogate-1/train.py`: switch to CDN-only fetches via manifest; add studio reuse.
- `/opt/axentx/surrogate-1/frontend/src/App.svelte` (or equivalent): add dataset-ready indicator.
- `/opt/axentx/surrogate-1/manifest-{date}.json` (generated): persisted file list.

// dataSync.js
// Manifest generator + CDN router for surrogate-1
// Run on Mac/CI before training. Training uses manifest for CDN-only fetches.

import { writeFileSync, readFileSync } from 'fs';
import { list_repo_tree } from 'huggingface-hub'; // or use fetch + token if preferred

const REPO = 'datasets/your-dataset'; // or org/repo
const DATE = new Date().toISOString().slice(0, 10); // YYYY-MM-DD
const OUT_PATH = `/opt/axentx/surrogate-1/manifest-${DATE}.json`;

// Deterministic repo routing to spread commits (avoid HF 128-commit/hr limit)
function siblingRepoIndex(slug) {
  let hash = 0;
  for (let i = 0; i < slug.length; i++) {
    hash = ((hash << 5) - hash) + slug.charCodeAt(i);
    hash |= 0;
  }
  return Math.abs(hash) % 6; // 0..5
}

export async function generateManifest({ path = DATE, recursive = false } = {}) {
  const tree = await list_repo_tree({
    repo: REPO,
    path,
    recursive
  });

  const files = tree
    .filter((t) => t.type === 'file' && t.path.endsWith('.parquet'))
    .map((f) => {
      const repoIndex = siblingRepoIndex(REPO);
      // Primary repo for manifest; siblings can be used for writes if needed
      const cdnUrl = `https://huggingface.co/datasets/${REPO}/resolve/main/${encodeURIComponent(f.path)}`;
      return {
        repo: REPO,
        path: f.path,
        sha: f.oid || f.commit,
        size: f.size,
        cdnUrl,
        repoIndex
      };
    });

  const manifest = {
    generatedAt: new Date().toISOString(),
    repo: REPO,
    date: DATE,
    path,
    recursive,
    fileCount: files.length,
    files
  };

  writeFileSync(OUT_PATH, JSON.stringify(manifest, null, 2));
  console.log(`Manifest written to ${OUT_PATH} (${files.length} files)`);
  return manifest;
}
