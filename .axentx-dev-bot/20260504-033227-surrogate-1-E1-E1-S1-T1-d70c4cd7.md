- **Metric completeness**: Returned dict contains keys `percent` (float 0–100), `secs_left` (Optional[int]), `temperature` (Optional[float]), `charge_cycles` (Optional[int]), `capacity` (Optional[int]).  
- **Async API**: `async for metrics in monitor()` yields a new dict each cycle.  
- **Sync API**: `BatteryMonitor().start()` spawns a background thread; `get_latest()` returns the most recent snapshot; `stop()` terminates the thread without leaving dangling resources.  
- **Cross‑platform behavior**:  
  - On Windows/macOS: `percent` and `secs_left` are provided via `psutil.sensors_battery()`.  
  - On Linux: if `/sys/class/power_supply/BAT0/` files exist, `temperature`, `charge_cycles`, and `capacity` are read; missing files result in `None` values.  
- **Graceful degradation**: Absence of any sysfs file does **not** raise an exception; instead, the corresponding key is `None` and a debug log is emitted.  
- **Type safety**: All public functions are annotated with correct types (`AsyncGenerator[Dict[str, Any], None]`, `Dict[str, Any]`, etc.).
