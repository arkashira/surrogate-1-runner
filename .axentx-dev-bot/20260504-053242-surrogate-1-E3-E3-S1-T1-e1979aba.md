## 1. Acceptance Criteria
- **AC‑1:** Every AI request (prompt, inference, model selection) generates a log entry that includes:
  - UTC timestamp (ISO‑8601 with millisecond precision)
  - Authenticated user ID and role
  - Service/endpoint identifier
  - Request payload hash (SHA‑256) and response status
- **AC‑2:** Log entries are written to the configured immutable store (e.g., append‑only file, WORM‑enabled S3 bucket, or tamper‑evident DB table) within 100 ms of request completion.
- **AC‑3:** The storage layer enforces write‑once semantics – attempts to modify or delete existing entries are rejected and raise an audit alert.
- **AC‑4:** An export API (`GET /audit/logs`) returns logs in CSV and JSON formats, supporting:
  - Date‑range filtering
  - User‑ID filtering
  - Pagination (max 10 000 records per page)
- **AC‑5:** Exported files are signed with a server‑side RSA‑2048 private key; the signature is included in the response headers.
- **AC‑6:** Access to the export endpoint is restricted to users with the `audit_viewer` role; unauthorized attempts return HTTP 403.
- **AC‑7:** The logging component does not degrade AI service latency by more than 5 % under a sustained load of 500 concurrent requests.
