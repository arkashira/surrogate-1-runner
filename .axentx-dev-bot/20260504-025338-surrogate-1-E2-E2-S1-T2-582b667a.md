## 2.  Final design

* **`ChangeTracker`** – a thread‑safe, file‑backed change‑history manager.  
  * All public methods acquire a re‑entrant lock (`threading.RLock`) so that nested calls (e.g. a notification that records another change) are safe.  
  * The JSON file is written atomically (`tempfile` + `os.replace`) to avoid corruption.  
  * Errors while loading or saving are logged but never crash the application.  
  * The class accepts an optional *notify* callback that receives `action_id` and the new record.

* **`dashboard`** – a thin wrapper that reads the pinned‑action list and asks the tracker for history.  
  * The file paths are **parameters** of the public functions, so tests can inject temporary files without monkey‑patching globals.  
  * The API returns a plain `dict` that can be serialised to JSON or fed into a template.

* **Tests** – use `pytest` fixtures to create isolated temporary files.  
  * No global state is mutated; the tracker is instantiated with the temporary history file.  
  * The dashboard functions are called with the temporary pinned‑action file.
