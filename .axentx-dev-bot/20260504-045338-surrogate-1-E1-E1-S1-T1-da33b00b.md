## 1. Acceptance Criteria
- **AC1** – The application can load AWS access key ID and secret access key from `application.properties` (or environment variables) without throwing exceptions.  
- **AC2** – When valid credentials are supplied, the `AwsConfig` bean creates an `AWSStaticCredentialsProvider` that can successfully call `sts:GetCallerIdentity` and returns the expected AWS account ID.  
- **AC3** – When invalid or missing credentials are supplied, the startup fails with a clear error message (`MissingAWSConfigurationException` or similar) and the process exits with a non‑zero status.  
- **AC4** – Credential values can be overridden at runtime via standard Spring Boot property sources (e.g., `SPRING_APPLICATION_JSON`, command‑line `--aws.accessKeyId=…`).  
- **AC5** – Sensitive values are never logged in plain text; logs contain only masked versions (e.g., `AKIA****`).  
- **AC6** – The configuration supports optional use of an IAM role (i.e., when `aws.useIamRole=true`, the static credentials are ignored and the default provider chain is used).
