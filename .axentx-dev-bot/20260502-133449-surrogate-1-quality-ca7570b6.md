        path="$DATE_FOLDER",

    )

paths = [f.rfilename for f in tree if f.rfilename.endswith(('.parquet', '.jsonl'))]

APPROVE: This is a workable, incremental improvement that directly addresses rate-limit and CastError risks by replacing recursive/expensive HF API usage with a manifest-driven, CDN-bypass ingestion path. It is deterministic, shardable, and testable end-to-end.

Acceptance criteria (downstream tester can verify):
- `bin/list-date-folder.sh <date-folder>` exits 0 and emits valid JSON with `{"date": "...", "files": [...]}` containing only `.parquet`/`.jsonl` filenames from the top-level of the date folder.
- `bin/dataset-enrich.sh <shard-id> <total-shards> [file-list.json]` completes without unhandled exceptions and writes a Parquet file under `batches/public-merged/<date>/shard<shard-id>-*.parquet` containing only rows with `prompt` and `response` string columns.
- Shard assignment is deterministic: running the same manifest through two workers with identical `SHARD_ID`/`TOTAL_SHARDS` produces identical output files (same rows, same order or dedup-stable order).
- No `load_dataset` or recursive `list_repo_tree` calls are made during per-file ingestion; only `hf_hub_download` and a single optional `list_repo_tree(..., recursive=False)` (or provided manifest) are used.
- Failures on individual files are non-fatal: malformed or unreadable files are logged to stderr and skipped, and the job continues processing remaining files.
