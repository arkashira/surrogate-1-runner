## 1. Acceptance Criteria
- **AC‑1:** Every change to a dependency (add, update, remove) triggers a log entry written to `/var/log/axentx/dependency-shield/audit.log`.
- **AC‑2:** Each audit log entry contains the fields: timestamp (ISO‑8601 UTC), userId, operation (ADD/UPDATE/REMOVE), dependencyId, previousVersion (if applicable), newVersion (if applicable), and a unique auditId (UUID).
- **AC‑3:** The audit log file is created with file permissions `rw-r-----` (owner `axentx`, group `axentx`).
- **AC‑4:** The Surrogate‑1 dashboard can retrieve the last *N* audit entries via `GET /api/audit?limit=N` and returns a JSON array with the same fields as the file entries.
- **AC‑5:** Retrieval via the dashboard respects the configured maximum limit of 1000 entries; requests above that return HTTP 400 with an explanatory error.
- **AC‑6:** Audit logs are rotated daily and retained for 30 days; older files are compressed and removed automatically.
- **AC‑7:** Access to the audit endpoint is restricted to users with the role `COMPLIANCE_OFFICER`; unauthorized requests receive HTTP 403.
