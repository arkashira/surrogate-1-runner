## TDD Test Plan: NightlyIngestionWorker

### 1. Acceptance Criteria

- [ ] Worker executes successfully when triggered by cron-scout at 02:00 UTC (verifiable via scheduled job logs)
- [ ] Each successful fetch inserts one row into `rat_account_snapshot` table with `account_id`, `snapshot_date`, `balance`, `transactions_json`, and `created_at` populated
- [ ] Provider errors (HTTP 5xx, timeout, malformed response) are logged to `/var/log/axentx/cron-scout-audit/daily_scan.log` with timestamp and error details
- [ ] On provider error, worker exits with non-zero status and does NOT insert partial data; next cron run retries
- [ ] Upsert logic ensures `INSERT ... ON CONFLICT (account_id, snapshot_date) DO NOTHING` prevents duplicate snapshots for same account/date combination

void execute_logsErrorAndThrowsOnProviderFailure() {

#### Happy Path (3-5 tests)

| # | Scenario | Expected Result |
|---|----------|-----------------|
| 1 | Full fetch with 10 accounts returns success | 10 rows inserted into `rat_account_snapshot`, exit code 0 |
| 2 | Empty provider response (no accounts) | 0 rows inserted, exit code 0 (valid state) |
| 3 | Worker runs twice same day for same accounts | Second run performs upsert-do-nothing, no duplicates |
| 4 | Database connection healthy | Transaction commits, snapshot visible to subsequent reads |
| 5 | Audit log file created on first run | `/var/log/axentx/cron-scout-audit/daily_scan.log` exists with header |

#### Edge Cases (2-3 tests)

| # | Scenario | Expected Result |
|---|----------|-----------------|
| 1 | Provider returns HTTP 500 | Error logged, exit code 1, no partial inserts |
| 2 | Database constraint violation (unexpected duplicate) | Worker catches exception, logs warning, continues |
| 3 | Disk full / audit log unwritable | Worker fails gracefully, logs to stderr, exits with code 1 |
