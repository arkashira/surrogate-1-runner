Replace `bin/dataset-enrich.sh` with a **manifest-driven, CDN-bypass ingestion worker** (`bin/dataset-enrich.py`) that:

- Accepts `SHARD_ID`, `SHARD_TOTAL=16` (matrix) and optional `DATE_FOLDER` (defaults to today `YYYY-MM-DD`).
- Uses a **single HF API call** from the runner (before the 5min window closes) to list one date folder via `list_repo_tree(recursive=False)` and saves `file-list.json`.
- Embeds that list in the runner; worker loads it and **downloads only assigned shard files via CDN** (`https://huggingface.co/datasets/.../resolve/main/...`) with **no Authorization header** → bypasses `/api/` rate limits.
- Projects each file to `{prompt, response}` at parse time (avoids `load_dataset(streaming=True)` on mixed schemas).
- Deduplicates via central md5 store (`lib/dedup.py`) and uploads shard output to:
  ```
  batches/public-merged/<date>/shard<SHARD_ID>-<HHMMSS>.jsonl
  ```
- Adds retry/backoff for CDN 429s and HF API 429s (360s wait), and falls back to `hf_hub_download` per file if CDN fails.
