## Test Plan – Real‑Time Shell Connection

### 1. Acceptance Criteria
- **AC1**: A shell connection is established within **1 second** of the user initiating the request.  
- **AC2**: The connection stays **active** for at least **30 minutes** without dropping or timing out.  
- **AC3**: When the user logs out, the connection is **terminated** within **2 seconds** of the logout event.  
- **AC4**: The connection does **not** accept any commands after termination.  
- **AC5**: No resource leaks (e.g., orphaned sockets) remain after termination.

### 2. Unit Tests (pseudo‑code)

```java
// RealtimeConnectionTest.java
class RealtimeConnectionTest {

  @Test
  void testConnectionEstablishmentWithinOneSecond() {
    long start = System.currentTimeMillis();
    ShellConnection conn = ShellConnectionFactory.create();
    long elapsed = System.currentTimeMillis() - start;
    assertTrue(elapsed <= 1000, "Connection took longer than 1s");
    assertTrue(conn.isActive(), "Connection should be active");
  }

  @Test
  void testConnectionStabilityOverThirtyMinutes() {
    ShellConnection conn = ShellConnectionFactory.create();
    // Simulate 30 minutes of idle time
    Thread.sleep(30 * 60 * 1000);
    assertTrue(conn.isActive(), "Connection should remain active after 30 minutes");
  }

  @Test
  void testTerminationOnLogout() {
    ShellConnection conn = ShellConnectionFactory.create();
    UserSession session = new UserSession("admin");
    session.logout(); // triggers termination
    long start = System.currentTimeMillis();
    while (conn.isActive() && System.currentTimeMillis() - start < 2000) {
      Thread.sleep(100);
    }
    assertFalse(conn.isActive(), "Connection should be terminated within 2s");
  }

  @Test
  void testNoCommandAfterTermination() {
    ShellConnection conn = ShellConnectionFactory.create();
    UserSession session = new UserSession("admin");
    session.logout();
    assertThrows(ConnectionClosedException.class, () -> conn.execute("ls"));
  }

  @Test
  void testNoResourceLeakAfterTermination() {
    ShellConnection conn = ShellConnectionFactory.create();
    UserSession session = new UserSession("admin");
    session.logout();
    // Verify no open sockets remain
    assertEquals(0, SocketPool.getOpenSocketCount(), "No sockets should remain open");
  }
}
```

### 3. Integration Tests

| Test # | Description | Expected Outcome |
|--------|-------------|------------------|
| 1 | **Happy Path** – User requests shell, stays active for 5 min, then logs out. | Connection established < 1 s, remains active, terminates within 2 s. |
| 2 | **Happy Path** – Multiple concurrent users each establish shell connections. | All connections < 1 s, no interference, each terminates properly on logout. |
| 3 | **Happy Path** – User sends a command after establishing connection. | Command executes successfully, output returned. |
| 4 | **Edge Case** – User logs out immediately after connection request. | Connection terminates within 2 s, no hanging sockets. |
| 5 | **Edge Case** – Network latency spikes to 2 s during connection. | Connection still established within 1 s (timeout handled), or fails gracefully with clear error. |
| 6 | **Edge Case** – Server restarts while a connection is active. | Connection is closed cleanly, user receives termination notice. |
| 7 | **Edge Case** – Attempt to reuse a terminated connection object. | Throws `ConnectionClosedException`. |
