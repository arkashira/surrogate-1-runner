## Unified Cloud‑Cost‑Optimization Engine  
*(“OptiCore” – the single, production‑ready module that pulls the best ideas from the three proposals and removes all contradictions.)*

### 1. High‑Level Design

| Layer | Responsibility | Key Features |
|-------|----------------|--------------|
| **Data Ingestion** | Pulls raw resource metadata from cloud APIs or log streams | • Queue‑based, idempotent ingestion <br>• Immediate acknowledgement (≤ 200 ms) <br>• Auto‑scales with worker pool |
| **Cost Analyzer** | Normalises raw data into a unified cost model | • Supports multi‑cloud (AWS, Azure, GCP, OCI) <br>• Handles spot/spot‑like pricing, reserved‑instance pricing, on‑demand, savings plans <br>• Generates a *Cost Snapshot* (resource‑level cost, utilization, trend) |
| **Recommendation Engine** | Applies a rule‑based + ML‑driven policy set | • Rule‑based core (right‑size, reserved‑instances, idle‑resource deletion, tiering, spot‑utilisation, etc.) <br>• Optional ML model for anomaly detection & “what‑if” scenarios <br>• Prioritises recommendations by *potential savings × confidence* |
| **Preference Manager** | Stores user‑specific filters, thresholds, and action‑plans | • “Do‑not‑terminate” tags <br>• Minimum acceptable utilization <br>• SLA‑based cost‑cap |
| **Event Bus** | Decouples components and enables real‑time dashboards | • Emits `ingestion:queued`, `ingestion:completed`, `recommendation:generated`, `error` events |
| **API Layer** | Exposes REST/GraphQL endpoints for ingestion, querying, and preference updates | • Idempotent `POST /ingest` <br>• `GET /recommendations/:id` <br>• `PATCH /preferences` |
| **Dashboard** | Real‑time UI (React/Next.js) that visualises cost, trends, and recommendations | • Auto‑refresh every 30 s <br>• Drill‑down to resource level <br>• Export CSV/JSON |

> **Contradiction resolution**  
> *Candidate 1* promised 1‑minute processing; *Candidate 2* focused on batch; *Candidate 3* on real‑time.  
> OptiCore uses a **worker pool** that processes each ingestion in < 30 s on average, while a background job can re‑run the full analysis nightly for deeper insights. This satisfies both real‑time and batch use‑cases.

### 2. Core Algorithms

#### 2.1 Cost Snapshot

```ts
type Resource = {
  id: string;
  type: 'instance' | 'storage' | 'database' | 'network';
  provider: 'aws' | 'azure' | 'gcp' | 'oci';
  region: string;
  usage: number;          // e.g. CPU‑hrs, GB‑hrs
  cost: number;           // current cost
  utilization: number;    // 0‑100%
  lastUsed?: number;      // epoch ms
  tags?: Record<string, string>;
};

function costSnapshot(resources: Resource[]): CostSnapshot {
  const totalCost = resources.reduce((s, r) => s + r.cost, 0);
  const avgUtil = resources.reduce((s, r) => s + r.utilization, 0) / resources.length;
  return { resources, totalCost, avgUtil, timestamp: Date.now() };
}
```
