### What this template does

1. **Semantic structure** – Uses `<header>`, `<nav>`, `<section>`, `<dl>`, and `<footer>` for clear document outline.

3. **Accessibility** – `aria-label` on the nav, proper heading levels, and a `lang` attribute.
4. **Data safety** – The `|safe` filter is applied only to the diagnostics blocks where raw HTML is expected; all other fields are auto‑escaped by the template engine.
5. **Navigation** – A single, consistent “Back to Diagnostics” link that matches the UI pattern used elsewhere in the app.
