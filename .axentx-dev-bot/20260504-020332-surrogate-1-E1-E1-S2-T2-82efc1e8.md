| Area | What the final design does | Why it’s the best choice |
|------|---------------------------|--------------------------|
| **Data model** | Pydantic‑based models that capture every field you need (severity, change‑type, resource‑type, status, timestamps, tags, etc.) | Clear, type‑safe, auto‑documented by FastAPI; ready for ORM mapping later. |
| **Persistence** | Starts with an in‑memory list for rapid prototyping, but a **SQLAlchemy + PostgreSQL** layer is wired in and can be swapped in with a single import change. | Gives you a production‑ready DB without rewriting the API logic. |
| **Filtering / Sorting / Pagination** | A single `POST /dashboard/query` endpoint that accepts a `DashboardQuery` payload (filters, sort, page, page_size). Internally the query is turned into an SQLAlchemy filter chain. | One endpoint, no combinatorial explosion of query‑string parameters; easy to extend. |
| **Real‑time push** | WebSocket endpoint (`/ws/updates`) with a `ConnectionManager` that broadcasts `RealTimeUpdate` messages whenever a change or misconfiguration is created/updated/deleted. | Keeps the UI in sync without polling; the manager gracefully drops dead connections. |
| **CRUD** | Full `GET /changes`, `GET /misconfigs`, `POST /changes`, `POST /misconfigs`, `PATCH /changes/{id}`, `PATCH /misconfigs/{id}`, `DELETE …` | Allows the UI (or other services) to manage data, not just read it. |
| **Security** | JWT‑based authentication (FastAPI `Depends`), role‑based scopes (`viewer`, `editor`, `admin`), CORS locked to your front‑end origin, rate‑limiting via `slowapi`. | Protects sensitive infrastructure data and prevents abuse. |
| **Observability** | Structured logging (loguru), request tracing (OpenTelemetry), health‑check endpoint (`/health`), Prometheus metrics (`/metrics`). | Makes the service operable in production. |
| **Testing & CI** | Pytest suite covering model validation, query logic, WebSocket broadcast, auth, and DB migrations; GitHub Actions pipeline that runs tests, lints, and builds a Docker image. | Guarantees correctness and fast feedback. |
| **Deployment** | Dockerfile + `docker-compose.yml` (API + Postgres), Helm chart for Kubernetes, environment‑variable driven config (`DATABASE_URL`, `JWT_SECRET`, etc.). | One‑click local dev, cloud‑native production. |
| **Documentation** | Automatic OpenAPI UI (`/docs`) plus a static Markdown “Developer Guide” that explains the data model, query language, and real‑time contract. | Lowers onboarding friction. |
