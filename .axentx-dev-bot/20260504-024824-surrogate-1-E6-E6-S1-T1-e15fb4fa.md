- **AC1 – Prometheus format**: A `GET /metrics` request returns a `200 OK` response with `Content‑Type: text/plain; version=0.0.4` and body containing the two counters `surrogate_checks_total` and `surrogate_checks_failed` in valid Prometheus exposition format.  
- **AC2 – Service label**: Each counter includes a `service="<service_name>"` label for every service that has reported checks; the label set matches the services registered in the collector.  
- **AC3 – Real‑time refresh**: After an internal update to a counter, a subsequent `GET /metrics` (performed ≤ 5 seconds later) reflects the new value.  
- **AC4 – Bearer token auth**: Requests that include the exact static token defined in `metrics.yaml` succeed (200). Requests with missing, malformed, or incorrect token receive `401 Unauthorized`.  
- **AC5 – Config loading errors**: If `metrics.yaml` is missing or malformed, the service fails to start with a clear error log and exits with a non‑zero status.  
- **AC6 – Thread safety**: Concurrent updates from ≥ 10 parallel threads do not cause lost increments or corrupted metric values; final counter values equal the sum of all increments performed.  
- **AC7 – Graceful error handling**: Any unexpected exception inside the exporter returns a `500 Internal Server Error` with a JSON payload `{ "error": "<message>" }` and logs the stack trace.
