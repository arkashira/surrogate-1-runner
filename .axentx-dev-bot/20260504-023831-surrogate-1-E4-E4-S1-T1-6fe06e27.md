# surrogate-1-runner

Parallel public-dataset ingest workers for the
[axentx/surrogate-1-training-pairs](https://huggingface.co/datasets/axentx/surrogate-1-training-pairs)
HuggingFace dataset.

## What this does

Every 30 minutes (or on `workflow_dispatch`), GitHub Actions launches **16 parallel runners**.
Each runner takes a deterministic 1/16 slice (`slug-hash bucket = SHARD_ID`)
of the public dataset list defined in `bin/dataset-enrich.sh`, streams,
normalizes per-schema, dedups via the central md5 hash store, and uploads
its output to a unique path on the dataset repo.

## Usage

To validate your dataset, use the following command:

```bash
./validate my_dataset.json
```

## Creating and Running a Validation Plan

1. Create a validation plan file (e.g., `plan.json`) following the schema below:

```json
{
  "datasets": [
    "path/to/dataset1.json",
    "path/to/dataset2.json"
  ],
  "validators": [
    "validate_schema",
    "validate_normalization"
  ]
}
```

2. Run the validation plan using the following command:

```bash
./run_plan plan.json
```

## Example Template

For an example validation plan template, see [example_plan.json](example_plan.json).

## Summary

- Updated README with usage example of the `validate` command.
