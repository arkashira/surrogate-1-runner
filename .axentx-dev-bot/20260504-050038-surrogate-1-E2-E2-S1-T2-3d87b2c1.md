## 1.  What We’re Building

A **single, production‑ready, cloud‑agnostic cost‑reporting engine** that:

| Cloud | Feature | Why it matters |
|-------|---------|----------------|
| **AWS** | Cost Explorer integration (service, region, tag, usage) | Native source, no extra cost |
| **Azure** | Cost Management API | Same for Azure customers |
| **GCP** | Cloud Billing API | Same for GCP customers |
| **Unified API** | `CostReport.get_report()` | One call, one result, regardless of provider |
| **Extensible** | Plug‑in architecture | Add new clouds or metrics without touching core |
| **CLI & Web UI** | Simple command‑line tool + lightweight Flask API | Quick adoption, no heavy front‑end |

The goal is to give a **single, consistent view** of spend across all clouds, with the ability to drill down by service, region, department tag, and usage quantity.

---

## 2.  Core Design Principles

| Principle | What it means | Implementation |
|-----------|---------------|----------------|
| **Single source of truth** | All cost data is fetched directly from the provider’s native API, never cached locally unless explicitly requested. | Each provider module implements a `CostProvider` interface. |
| **Idempotent & retry‑safe** | Calls are wrapped in a retry decorator that respects provider rate limits. | `@retry_on_exception` decorator with exponential back‑off. |
| **Minimal dependencies** | Only the official SDKs and `pydantic` for validation. | `requirements.txt` contains `boto3`, `azure-mgmt-costmanagement`, `google-cloud-billing`, `pydantic`. |
| **Clear data model** | All providers return the same `CostEntry` dataclass. | `CostEntry` defined once in `core/models.py`. |
| **Extensibility** | New providers are added by implementing the interface and registering in `core/__init__.py`. | `ProviderRegistry.register('aws', AWSCostProvider)` |
