Below is a **single, self‑contained web‑shell UI** that pulls the strongest ideas from both candidates:

* **Full command line UI** – a text input + “Run” button (Candidate 1).
* **Real‑time streaming** – WebSocket that pushes line‑delimited output (Candidate 2).
* **Efficient paging** – keep a buffer, show only the last *N* lines, and a “Load Older” button that pulls earlier lines into view (both candidates).
* **Auto‑scroll** – when new data arrives and the user is already at the bottom, scroll automatically.
* **Graceful cleanup** – close the WebSocket when the command finishes.

```html
<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<title>Web Shell Interface</title>
<style>
  body { font-family: monospace; margin: 0; padding: 0; }
  #controls {
    padding: 10px;
    background:#333;
    display:flex;
    gap:10px;
  }
  #command { flex:1; padding:5px; }
  #run, #loadOlder { padding:5px 10px; cursor:pointer; }
  #output {
    height:80vh;
    overflow-y:auto;
    background:#1e1e1e;
    color:#dcdcdc;
    padding:10px;
    white-space:pre-wrap;
  }
</style>
</head>
<body>
<div id="controls">
  <input type="text" id="command" placeholder="Enter command" />
  <button id="run">Run</button>
  <button id="loadOlder">Load Older</button>
</div>
<pre id="output"></pre>

<script>
/* ---------- Configuration ---------- */
const VISIBLE_LINES = 200;          // lines shown at once
const WS_URL = `ws://${location.host}/ws/exec`; // adjust if needed

/* ---------- DOM Elements ---------- */
const outputEl   = document.getElementById('output');
const cmdInput   = document.getElementById('command');
const runBtn     = document.getElementById('run');
const loadBtn    = document.getElementById('loadOlder');

/* ---------- State ---------- */
let buffer = [];          // all lines received for the current command
let displayedStart = 0;   // index of first line currently rendered
let ws = null;            // current WebSocket

/* ---------- Helpers ---------- */
function renderPage(start, end) {
  const slice = buffer.slice(start, end).join('\n');
  outputEl.textContent = slice;
  // Keep at bottom if we were already at the bottom
  if (displayedStart === 0) {
    outputEl.scrollTop = outputEl.scrollHeight;
  }
}

function appendLines(newLines) {
  buffer.push(...newLines);
  // If we are at the bottom, show the newest lines
  if (displayedStart === 0) {
    const end = Math.min(VISIBLE_LINES, buffer.length);
    renderPage(0, end);
  }
}

/* ---------- Load Older ---------- */
loadBtn.addEventListener('click', () => {
  const newStart = Math.max(0, displayedStart - VISIBLE_LINES);
  const newEnd   = displayedStart;
  displayedStart = newStart;
  renderPage(newStart, newEnd);
});

/* ---------- Run Command ---------- */
runBtn.addEventListener('click', () => {
  const cmd = cmdInput.value.trim();
  if (!cmd) return;

  // Reset state
  buffer = [];
  displayedStart = 0;
  outputEl.textContent = '';

  // Close any existing socket
  if (ws) ws.close();

  // Open new WebSocket
  ws = new WebSocket(WS_URL);
  ws.onopen = () => ws.send(JSON.stringify({ cmd }));
  ws.onmessage = e => {
    const msg = JSON.parse(e.data);
    if (msg.type === 'output') {
      const lines = msg.lines.filter(Boolean); // strip empty
      appendLines(lines);
    } else if (msg.type === 'done') {
      ws.close();
    }
  };
  ws.onerror = err => console.error('WebSocket error:', err);
});
</script>
</body>
</html>
```

### How it works

1. **Command entry** – User types a shell command and clicks **Run**.  
2. **WebSocket** – The client opens a WebSocket to `/ws/exec`, sends `{cmd}`.  
3. **Streaming** – The server streams JSON objects of the form  
   ```json
   { "type":"output", "lines":[ "line1", "line2", ... ] }
   ```  
   The client appends these to the buffer.  
4. **Rendering** – If the user is already at the bottom (`displayedStart===0`), the newest lines are shown automatically.  
5. **Paging** – Clicking **Load Older** shifts the view
