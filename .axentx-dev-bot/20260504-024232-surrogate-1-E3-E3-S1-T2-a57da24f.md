## Unified, Production‑Ready Drift‑Alert Implementation  

Below is a single, coherent solution that merges the strongest ideas from the two proposals:

* **Rich `DriftEvent` model** – includes all fields needed for detection, alerting, and optional downstream processing (`RawDiff` for internal use).  
* **Self‑contained Slack payload builder** – uses Slack’s *attachments* format (colour, fields) for a readable, nicely‑styled message while still exposing a simple “text‑only” fallback.  
* **Configurable dispatcher** – reads the webhook URL and suppression window from environment variables, defaults to a 10‑minute deduplication window, and works as a no‑op when the webhook is missing (great for local development).  
* **Thread‑safe, low‑contention cache** – a `sync.Map` (lock‑free) for fast reads, plus a small `sync.Mutex`‑protected map for writes when we need to update the cache after a successful send.  
* **Robust HTTP handling** – 5‑second timeout, proper content‑type, explicit error on non‑2xx responses, and automatic cache cleanup on failure so retries are possible.  
* **Testability** – the dispatcher is exported as a struct with an injectable `http.Client`, making unit‑tests easy (you can pass a mock RoundTripper).

// /opt/axentx/surrogate-1/src/model/drift_event.go
package model

import "time"

// DriftEvent represents a detected configuration drift.
// It is used throughout the detector pipeline and also for alerting.
type DriftEvent struct {
	// ServiceName is the logical name of the service where drift was detected.
	ServiceName string `json:"service_name"`

	// DeploymentHash uniquely identifies the deployment version
	// (e.g., a git commit SHA or container image digest).
	DeploymentHash string `json:"deployment_hash"`

	// DetectedAt is the time the drift was observed.
	DetectedAt time.Time `json:"detected_at"`

	// DiffSummary provides a concise, human‑readable summary of the differences.
	// Keep it short enough to fit in a Slack field (≈200 chars).
	DiffSummary string `json:"diff_summary"`

	// RawDiff holds the full diff payload for internal consumers.
	// It is omitted from the JSON sent to Slack to keep the alert lightweight.
	RawDiff string `json:"-"` // internal use only
}

// Payload builds the JSON structure expected by Slack Incoming Webhooks.
// It returns a map that can be marshaled directly.
func (e *DriftEvent) Payload() map[string]interface{} {
	ts := e.DetectedAt.UTC().Format(time.RFC3339)

	// Slack “attachments” give us colour and nicely‑aligned fields.
	return map[string]interface{}{
		"text": "*Drift detected*",
		"attachments": []map[string]interface{}{
			{
				"color": "#ff0000", // red → attention
				"fields": []map[string]string{
					{"title": "Service", "value": e.ServiceName, "short": "true"},
					{"title": "Deployment", "value": e.DeploymentHash, "short": "true"},
					{"title": "Timestamp", "value": ts, "short": "true"},
					{"title": "Diff", "value": e.DiffSummary, "short": "false"},
				},
			},
		},
	}
}
```

*Why this version?*  
- Keeps the **concise struct** from Candidate 1 (adds `RawDiff` for internal pipelines).  
- Supplies a **payload helper** (from Candidate 2) that produces a richer Slack message with colour and fields, improving readability for ops teams.
