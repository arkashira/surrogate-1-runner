## TDD Test Plan: AI Request Logging

### 1. Acceptance Criteria

- [ ] **AC1**: Every AI request through the gateway produces a log entry containing: ISO8601 timestamp, user ID (string), model name (string), and request payload (JSON/dict)
- [ ] **AC2**: Log file exists at `/var/log/surrogate-1/ai_requests.log` with read/write permissions for the application user
- [ ] **AC3**: Log entries are append-only and written synchronously before response is returned to caller
- [ ] **AC4**: Admin UI endpoint `/admin/logs` returns last 100 log entries as JSON array
- [ ] **AC5**: Log format is parseable as JSON (one JSON object per line - JSON Lines format)
- [ ] **AC6**: Failed log writes (disk full, permission denied) raise an exception that propagates to caller
- [ ] **AC7**: Unit of work identifier (correlation ID) is included in each log entry for traceability
