**Highest-value improvement**: Replace fragile shell-based ingestion with a **manifest-driven, CDN-bypass Python worker** that eliminates HF API rate limits during training data loads and prevents mixed-schema CastErrors.

### Changes

1. **Add `bin/worker.py`** — single-file worker that:
   - Accepts `SHARD_ID` and `TOTAL_SHARDS` from the matrix.
   - Uses **one** HF API call (`list_repo_tree`) to list date folders (non-recursive), then writes `file-manifest.json` locally.
   - Downloads only files assigned to this shard via **CDN direct URLs** (`resolve/main/...`) — zero auth, zero API rate limit during training.
   - Projects each file to `{prompt, response}` at parse time (avoids `load_dataset` mixed-schema CastError).
   - Deduplicates via `lib/dedup.py` (central md5 store) and streams output as newline JSON to `batches/public-merged/<date>/shard<N>-<HHMMSS>.jsonl`.
   - Validates schema and emits small stats (rows, dups, bytes).

2. **Replace `bin/dataset-enrich.sh` with `bin/worker.py`**  
   - Deterministic sharding via `hash(path) % TOTAL_SHARDS`.  
   - Deterministic output filename: `shard{N}-{YYYYmmdd-HHMMSS}.jsonl`.  
   - `set -euo pipefail` wrapper retained for local testing; Actions invokes Python directly.

3. **Update `.github/workflows/ingest.yml`** — minor:
   - Ensure matrix provides `SHARD_ID: [0..15]` and `TOTAL_SHARDS: 16`.
   - Add `timeout-minutes: 30` and `concurrency: surrogate-1-ingest` to avoid queue pileups.
   - Keep `HF_TOKEN` secret usage.

4. **Add `requirements.txt` entries** (if missing):
