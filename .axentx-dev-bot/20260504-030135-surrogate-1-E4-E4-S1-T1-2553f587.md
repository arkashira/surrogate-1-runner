# /opt/axentx/surrogate-1/README.md
# surrogate-1-runner

Parallel public-dataset ingest workers for the
[axentx/surrogate-1-training-pairs](https://huggingface.co/datasets/axentx/surrogate-1-training-pairs)
HuggingFace dataset.

## What this does

Every 30 minutes (or on `workflow_dispatch`), GitHub Actions launches **16 parallel runners**.
Each runner takes a deterministic 1/16 slice (`slug-hash bucket = SHARD_ID`) of the public dataset list defined in `bin/dataset-enrich.sh`, streams, normalizes per-schema, dedups via the central md5 hash store, and uploads its output to a unique path on the dataset repo.

## CLI Usage

### Installation

You can install the surrogate-1 CLI tool using pip:

```bash
pip install surrogate-1
```

### Basic Usage

Run the formatter with default settings:

```bash
surrogate-1 format --input your_file.py
```

### Configuration

You can configure the formatter with a YAML configuration file:

```bash
surrogate-1 format --config config.yaml --input your_file.py
```

### Available Options

```bash
surrogate-1 [options]

Options:
  --input, -i FILE      Input file or directory
  --output, -o FILE    Output file or directory (default: stdout)
  --config, -c FILE    Configuration file (default: .surrogate-1.yaml)
  --help, -h           Show this help message and exit
  --version, -v        Show version and exit
```

### Example Configuration File

Create a `.surrogate-1.yaml` file in your project root:

```yaml
# .surrogate-1.yaml
input_dir: "src"
output_dir: "dist"
ignore_patterns:
  - "tests/*"
  - "node_modules/*"
  - "vendor/*"
```

### GitHub Action Integration

You can use the surrogate-1 CLI in GitHub Actions:

```yaml
# .github/workflows/format.yml
name: Format Code

on: [push, pull_request]

jobs:
  format:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: 3.11
      - name: Install surrogate-1
        run: pip install surrogate-1
      - name: Format code
        run: surrogate-1 format --config .surrogate-1.yaml
      - name: Check for errors
        run: surrogate-1 format --check --config .surrogate-1.yaml
```

## Development

To develop the surrogate-1 CLI:

```bash
# Clone the repository
git clone https://github.com/axentx/surrogate-1.git
cd surrogate-1

# Install dependencies
pip install -e .

# Run the development server
surrogate-1 dev
```

The development server will be available at http://localhost:8000.
