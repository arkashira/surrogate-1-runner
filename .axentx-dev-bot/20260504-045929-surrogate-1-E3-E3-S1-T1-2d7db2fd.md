# surrogate‑1‑runner

Parallel public‑dataset ingest workers for the
[axentx/surrogate‑1‑training‑pairs](https://huggingface.co/datasets/axentx/surrogate-1-training-pairs)
HuggingFace dataset.

## What it does

Every 30 minutes (or on `workflow_dispatch`) GitHub Actions launches **16 parallel runners**.  
Each runner processes a deterministic 1/16 slice (`SHARD_ID`) of the public dataset list defined in `bin/dataset‑enrich.sh`, streams, normalises per‑schema, dedups via the central MD5 store, and uploads its output to a unique path on the dataset repo.

## Quick start

```bash
# 1️⃣ Clone the repo
git clone https://github.com/axentx/surrogate-1.git
cd surrogate-1

# 2️⃣ (Optional) Build the Docker image
docker build -t axentx/surrogate-1:latest .

# 3️⃣ Run a single worker locally
export SHARD_ID=0
docker run --rm -e SHARD_ID=$SHARD_ID axentx/surrogate-1:latest python bin/dataset-enrich.sh
```

> **Tip** – If you prefer not to use Docker, install the Python dependencies (`pip install -r requirements.txt`) and run the script directly.

## Documentation

- **[Installation Guide](/docs/installation_guide.md)** – step‑by‑step setup for local or CI use.
- **[Usage Walkthrough Video](/docs/usage_walkthrough_video.md)** – visual demo of the full pipeline.
- **[FAQ](/docs/faq.md)** – common questions & troubleshooting.

## GitHub Actions example
