## Event schema and broker modes

- The system emits normalized `OrderEvent` objects (see `surrogate_1/src/connectors/broker/events.py`) for order lifecycle events: submitted, acked, filled, rejected, cancelled.
- `OrderEvent.to_dict()` produces UTC ISO timestamps (`emitted_at`). Deserialization via `OrderEvent.from_dict()` preserves all fields and requires timezone-aware datetimes.
- Choose broker implementation with `BROKER_MODE=simulated|live` (env or config). Both emit events conforming to the same normalized schema.
- Events are persisted durably in the event store and support replay via read-all and optional time-range queries. Ordering is deterministic (monotonic by `emitted_at` then `sequence`).
- Submit-order path measures ack latency; p95 ≤ 200 ms is reported by the metrics endpoint/logger for ≥1000 sequential local submissions.
- Exports and wiring are explicit (factory/entrypoint) so downstream services can start the system and produce/consume events without hidden defaults.
