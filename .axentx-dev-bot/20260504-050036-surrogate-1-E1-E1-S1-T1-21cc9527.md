- **AC2 – Baseline Benchmarking**: For each resource, the model computes a benchmark cost based on industry averages (provided in `benchmarks.json`) and stores it in `resource_benchmarks` dict.  
- **AC3 – Recommendation Generation**: The model returns a list of recommendation objects where each object contains:
  - `resource_id`
  - `action` (e.g., `rightsize`, `terminate`, `schedule_downscale`)
  - `justification` (text explaining the cost saving)
  - `estimated_monthly_savings` (numeric, ≥ 0)
- **AC4 – Real‑time Update**: When new usage rows are appended to `usage_patterns.csv` (or streamed via the `update_usage` API), the model recomputes recommendations within **5 seconds** of the change.  
- **AC5 – Accuracy Threshold**: At least **80 %** of generated recommendations must match a ground‑truth set derived from historical manual optimizations (tested on the provided validation dataset).  
- **AC6 – Fault Tolerance**: If the CSV is malformed or missing columns, the model raises a custom `CostOptimizerError` with a clear message and does not crash the service.  
- **AC7 – Logging**: Each recommendation generation run logs start/end timestamps, number of resources processed, and total estimated savings at INFO level.
