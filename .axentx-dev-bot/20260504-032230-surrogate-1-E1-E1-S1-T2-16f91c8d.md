Below is a **single, coherent implementation** that merges the strongest ideas from the two proposals:

* a **clean, extensible `PasteResolver`** that works with pluggable paste‑strategies (direct + fallback) – this gives you the test‑ability and future‑proofing of the first proposal;
* a **realistic `DirectPasteStrategy`** that contains the simple “non‑empty & no “FAIL”” rule from the second proposal, so the behaviour is easy to reason about in unit‑tests;
* a **fallback stub** that can be replaced later with a legacy implementation;
* **JUnit 5 + Mockito** tests that verify:
  1. Direct paste is tried first;
  2. The resolver returns the direct‑paste result when it succeeds;
  3. The fallback is only invoked when the direct paste fails;
  4. Direct paste never mutates the system clipboard (is isolated).

## 1. Production code

### 1.1. Core model – `PasteResult`

```java
package com.axentx.surrogate1.paste;

/**
 * Simple immutable DTO that tells the caller whether a paste succeeded
 * and which strategy performed it.
 */
public final class PasteResult {
    private final boolean successful;
    private final String strategyName;

    public PasteResult(boolean successful, String strategyName) {
        this.successful = successful;
        this.strategyName = strategyName;
    }

    public boolean isSuccessful() {
        return successful;
    }

    public String getStrategyName() {
        return strategyName;
    }
}
```

### 1.2. Strategy contract

```java
package com.axentx.surrogate1.paste;

/**
 * A paste strategy knows how to attempt a paste operation.
 * Implementations must **never** touch the system clipboard unless they
 * really intend to paste – the resolver guarantees isolation.
 */
public interface PasteStrategy {
    /**
     * Tries to paste the supplied content.
     *
     * @param content the text that should be pasted
     * @return a {@link PasteResult} describing success and the strategy name
     */
    PasteResult attemptPaste(String content);
}
```

### 1.3. Direct‑paste implementation (the “AX direct paste”)

```java
package com.axentx.surrogate1.paste;

/**
 * Simulated AX‑direct‑paste.  The real implementation would call the AX API;
 * for now we keep the simple rule from the second proposal:
 *
 *   – succeed if the content is non‑null, non‑empty and does **not** contain
 *     the literal word “FAIL”.
 */
public class DirectPasteStrategy implements PasteStrategy {

    private static final String NAME = "direct";

    @Override
    public PasteResult attemptPaste(String content) {
        boolean ok = content != null
                     && !content.isEmpty()
                     && !content.contains("FAIL");
        return new PasteResult(ok, NAME);
    }
}
```

### 1.4. Fallback implementation (placeholder)

```java

/**
 * Legacy fallback – at the moment it always fails, but the class exists so
 * the resolver can be wired with a real implementation later without
 * touching the resolver logic.
 */
public class FallbackPasteStrategy implements PasteStrategy {

    private static final String NAME = "fallback";

    @Override
    public PasteResult attemptPaste(String content) {
        // TODO: replace with real legacy paste logic.
        return new PasteResult(false, NAME);
    }
}
```

### 1.5. Resolver – the orchestrator

```java
package com.axentx.surrogate1.paste;

/**
 * Resolves a paste request by trying the strategies in order:
 *   1. Direct paste (AX)
 *   2. Fallback (legacy)
 *
 * The resolver itself is **stateless** – it only coordinates the strategies,
 * making it trivial to unit‑test.
 */
public class PasteResolver {

    private final PasteStrategy directStrategy;
    private final PasteStrategy fallbackStrategy;

    public PasteResolver(PasteStrategy directStrategy,
                         PasteStrategy fallbackStrategy) {
        this.directStrategy = directStrategy;
        this.fallbackStrategy = fallbackStrategy;
