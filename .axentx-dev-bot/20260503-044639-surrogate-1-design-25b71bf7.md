**Highest-value improvement**: Replace fragile shell-based ingestion with a **manifest-driven, CDN-bypass Python worker** that eliminates HF API rate limits during training data loads and prevents mixed-schema CastErrors.

### Changes

1. **Add `bin/worker.py`** — deterministic shard worker that:
   - Uses a single `list_repo_tree` call (per date folder) from the runner, embeds file list in a `manifest.json`
   - Downloads only assigned shard files via HF CDN (`resolve/main/...`) with **no Authorization header** (bypasses API rate limits)
   - Projects to `{prompt, response}` at parse time; never loads full heterogeneous schemas into a single `load_dataset`
   - Writes `batches/public-merged/<date>/shard<N>-<HHMMSS>.jsonl` (existing path)
   - Uses central `lib/dedup.py` for cross-source md5 dedup

2. **Update `bin/dataset-enrich.sh`** → thin wrapper that:
   - Invokes `python3 bin/worker.py` with `SHARD_ID`, `TOTAL_SHARDS`, `DATE`, `MANIFEST`
   - Keeps executable bit; Bash shebang retained for cron/workflow compatibility

3. **Update `.github/workflows/ingest.yml`** → matrix runner:
   - Pre-step: one lightweight job (or first matrix entry) generates `manifest.json` for the target date folder using HF API (rate-limit safe: single tree call) and uploads as artifact
   - Each shard downloads the artifact and runs `worker.py`
   - Sets `SHELL=/bin/bash` in job env (per pattern)

1. **Acceptance criteria**
- Deterministic shard assignment: for any filename and TOTAL_SHARDS=N, hash(filename) % N assigns the file to exactly one shard across runs.
- Manifest-driven file selection: worker reads manifest.json and processes only files listed for the specified DATE folder.
- CDN-bypass downloads: each file is fetched via `resolve/main/...` without Authorization header; HTTP 200 responses are accepted and non-2xx responses raise clear errors.
- Schema projection: only `prompt` and `response` columns are read from source files; heterogeneous schemas never trigger mixed-schema CastErrors.
- Deduplication: records with duplicate md5 (via lib/dedup) are excluded from output.
- Output contract: each shard writes `batches/public-merged/<DATE>/shard<N>-<HHMMSS>.jsonl` containing valid JSONL objects with `prompt` and `response` fields.
- Idempotency: running the same shard with the same manifest and dedup store produces identical output (no duplicate inserts).
