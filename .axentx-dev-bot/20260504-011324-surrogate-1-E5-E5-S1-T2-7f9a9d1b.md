## surrogate‑1‑runner – One‑stop guide  

**What you get**  

* A **quick‑start README** that tells a new user how to install the SDK, generate a sample PDF (if you don’t already have one), and run a **self‑contained Retrieval‑Augmented Generation (RAG) demo**.  
* A **single, well‑structured demo script** that:  
  1. Loads a PDF (generated on‑the‑fly if missing).  
  2. Extracts its text with the **Python SDK** (`DocumentLoader`).  
  3. Performs a naïve retrieval over the extracted chunks (`SimpleRetriever`).  
  4. Generates a placeholder answer (`SimpleGenerator`).  
* A **binary `sample.pdf`** (tiny “Hello, Axentx!” document) for the fastest possible first run, plus a **fallback generator** (`generate_sample_pdf.py`) that creates a richer PDF with ReportLab if you prefer a more realistic file.  
* Clear troubleshooting notes and a **GitHub‑Actions‑ready description** of the 16‑parallel ingest workers (the original “surrogate‑1‑training‑pairs” pipeline).

## 1️⃣ README – Quick‑start & overview  

```markdown
# surrogate-1-runner

Parallel public‑dataset ingest workers for the
[axentx/surrogate-1-training-pairs](https://huggingface.co/datasets/axentx/surrogate-1-training-pairs)
HuggingFace dataset.

## Quick‑start

```bash
# 1️⃣ Install the SDK (the package name is `axentx-surrogate-1`; adjust if it changes)
pip install axentx-surrogate-1

# 2️⃣ (Optional) Install the PDF‑generation helper – only needed if you want to
#    recreate `sample.pdf` with richer content.
pip install reportlab

# 3️⃣ Run the demo
python -m examples.simple_rag.demo
```

The demo will:

1. **Generate** `sample.pdf` if it does not exist (tiny “Hello, Axentx!” or a
   ReportLab‑generated page).  
2. **Extract** the text with the SDK.  
3. **Retrieve** the most relevant chunk(s) for a hard‑coded query.  
4. **Generate** a placeholder response and print it.

## What the repo does in production

Every 30 minutes (or on manual `workflow_dispatch`) a GitHub Actions workflow
spins up **16 parallel runners**. Each runner:

| Step | Action |
|------|--------|
| **Shard selection** | Deterministic 1/16 slice of the public‑dataset list (`slug‑hash % 16 == SHARD_ID`). |
| **Streaming & normalisation** | `bin/dataset‑enrich.sh` streams the slice, normalises to the schema, and dedupes via a central MD5 hash store. |
| **Upload** | Writes its output to a unique path in the dataset repo (`shard-{id}/…`). |

> The workflow file (`.github/workflows/ingest.yml`) is omitted here for brevity
> but follows the same pattern as the official Axentx example.

## Troubleshooting FAQ

| Symptom | Fix |
|---------|-----|
| `ImportError: No module named axentx_surrogate` | Ensure the SDK is installed: `pip install axentx-surrogate-1`. |
| PDF text extraction returns empty | Verify that `sample.pdf` exists and contains real text. Run `python -m examples.simple_rag.generate_sample_pdf` to create a fresh PDF. |
| MkDocs build fails | Run `python -m py_compile **/*.py` locally to locate syntax errors. |
| GitHub Actions runner times out | Increase the timeout in `.github/workflows/ingest.yml` or split the shard list into smaller batches. |
