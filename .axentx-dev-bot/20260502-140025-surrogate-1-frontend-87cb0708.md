### 1) Generate deterministic file list (run once per folder/date)

`scripts/build-file-list.sh`
```bash
#!/usr/bin/env bash
# Usage: HF_TOKEN=... ./scripts/build-file-list.sh axentx/surrogate-1-training-pairs [folder] > file-list-YYYY-MM-DD.json
set -euo pipefail
REPO="${1:?repo}"
FOLDER="${2:-}"

python3 - <<PY
import os, json, sys
from huggingface_hub import HfApi
api = HfApi(token=os.environ["HF_TOKEN"])
tree = api.list_repo_tree(repo_id="$REPO", path="$FOLDER", recursive=True)
paths = sorted(item.rfilename for item in tree if item.type == "file")
json.dump(paths, sys.stdout, indent=2)
PY
```

- Commit or upload `file-list-YYYY-MM-DD.json` to repo root or as a workflow artifact.  
- This is the **single source of truth** for both ingestion and frontend.

### 2) Update ingestion worker (`bin/dataset-enrich.sh`)

Key changes:
- Accept `FILE_LIST` (path to JSON). If absent, fail fast (do not fallback to runtime listing in production).  
- Download via **CDN-only URLs** (`resolve/main/...`).  
- Deterministic sharding by path hash.  
- Project only `{prompt,response}`; robust fallback for non-parquet files.  
- Retry/backoff for CDN downloads.  
- Emit clean JSONL.
