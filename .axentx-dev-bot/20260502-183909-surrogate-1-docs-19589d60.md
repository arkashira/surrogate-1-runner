## signal_fn usage

`signal_fn(payload)` evaluates a research payload and returns a trade signal.

- **Input**: `payload` (dict) — must include `symbol`, `edge_bps`, one of `target_price` or `last_close`, `model`, and `meta`.
- **Output** (dict): `signal_id`, `side`, `quantity`, `price`, `meta`.
  - `signal_id` is deterministic for identical payloads.
  - `side` is `"LONG"` if `edge_bps > 0`, otherwise `"SHORT"`.
  - `quantity` is non-negative numeric.
  - `price` is numeric (prefers `target_price`, falls back to `last_close`).
- **Side effects**: appends a JSON line to `/opt/axentx/surrogate-1/logs/events.log` and calls `os.fsync` for durability.
- **Errors**: raises `ValueError` if required fields (`symbol`, price fields) are missing; raises `TimeoutError` if execution exceeds 200ms.
