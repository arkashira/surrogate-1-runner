## Analytics Metrics Definition

### Overview
This document outlines the key metrics that will be tracked to measure user engagement with the surrogate-1 framework and system health. These metrics are designed to provide actionable insights into how users interact with the system, the effectiveness of the data ingestion pipeline, and overall system performance.

### Key Metrics

| Metric | Definition | Data Source | Frequency | Notes |
|--------|------------|-------------|-----------|-------|
| **Total Ingested Records** | Number of dataset records processed by the ingestion pipeline. | `bin/dataset-enrich.sh` logs | Hourly | Indicates overall throughput. |
| **Successful Ingests** | Count of records that passed validation and were uploaded successfully. | Pipeline success logs | Hourly | Measures data quality. |
| **Failed Ingests** | Count of records that failed validation or upload. | Pipeline error logs | Hourly | Helps identify data issues. |
| **Average Ingest Latency** | Time from record ingestion start to completion. | Timestamped logs | Hourly | Gauges pipeline performance. |
| **Shard Completion Rate** | Percentage of shards completed per run. | Shard status logs | Every run | Ensures parallelism is effective. |

#### User Engagement Metrics

| Metric | Definition | Data Source | Frequency | Notes |
|--------|------------|-------------|-----------|-------|
| **User Session Count** | Number of distinct user sessions interacting with the framework UI. | Front-end analytics (e.g., Google Analytics, Mixpanel) | Daily | Measures user engagement. |
| **Active Users (DAU/WAU/MAU)** | Daily/Weekly/Monthly active users. | Front-end analytics | Daily | Standard engagement metric. |
| **Session Duration** | Average time spent by users during each session. | Front-end analytics | Daily | Indicates how engaging the framework is. |
| **Page Views** | Total number of pages viewed by users. | Front-end analytics | Daily | Provides insight into which parts of the framework are most popular. |
| **Feature Usage Frequency** | Count of times each UI feature (e.g., dataset view, filter, export) is used. | Front-end event tracking | Daily | Identifies popular features. |
| **Bounce Rate** | Percentage of users who leave the framework after viewing only one page. | Front-end analytics | Daily | Indicates potential issues with user experience or content relevance. |
| **User Feedback Score** | Average score given by users through feedback forms or surveys. | Front-end analytics | Daily | Helps gauge user satisfaction and identify areas for improvement. |

#### System Performance Metrics

| Metric | Definition | Data Source | Frequency | Notes |
|--------|------------|-------------|-----------|-------|
| **Error Rate** | Number of errors per 1,000 operations. | Backend error logs | Hourly | Tracks system reliability. |
| **Data Quality Score** | Composite score based on validation success, duplicate rate, and schema compliance. | Validation logs | Daily | Provides a single health indicator. |
| **Report Generation Count** | Number of analytics reports generated. | Reporting service logs | Daily | Measures reporting usage. |
| **Report Delivery Success** | Percentage of reports delivered successfully to users. | Delivery logs | Daily | Ensures report reliability. |
| **Average Report Generation Time** | Time taken to generate a report. | Reporting service timestamps | Daily | Gauges reporting performance. |
| **Storage Utilization** | Amount of storage used by processed data. | Cloud storage metrics | Daily | Helps with capacity planning. |
| **Cost per Record** | Operational cost divided by number of records processed. | Billing data | Monthly | Cost efficiency metric. |

- **Real-time Dashboards**: Key metrics such as ingest counts, error rates, and latency will be displayed on a live dashboard (e.g., Grafana).
