# docs/api_endpoints.md
# Surrogate‑1 API – v1

## Overview
The Surrogate‑1 service exposes a REST API that lets Platform Engineers trigger and monitor dataset‑ingestion workflows.

- **Base URL**: `https://api.surrogate-1.axentx.dev/v1`
- **Auth**: `Authorization: Bearer <token>`
- **Content‑type**: `application/json`

---

## Endpoints

### 1️⃣ List Available Datasets
`GET /datasets`

```json
{
  "datasets": [
    {
      "id": "surrogate-1-training-pairs",
      "name": "axentx/surrogate-1-training-pairs",
      "description": "Training pairs for surrogate model",
      "shards": 16,
      "last_updated": "2024-05-03T12:00:00Z"
    }
  ]
}
```

### 2️⃣ Trigger Workflow Run
`POST /workflows/run`

```json
{
  "dataset_id": "surrogate-1-training-pairs",
  "shards": 16,
  "options": {
    "deduplicate": true,
    "normalize": true
  }
}
```

_Response_

```json
{
  "run_id": "run_20240503_120000_abc123",
  "status": "queued",
  "shards": 16,
  "created_at": "2024-05-03T12:00:00Z"
}
```

### 3️⃣ Get Run Status
`GET /workflows/{run_id}`

```json
{
  "run_id": "run_20240503_120000_abc123",
  "status": "running",
  "progress": {
    "completed_shards": 8,
    "total_shards": 16,
    "percentage": 50
  },
  "started_at": "2024-05-03T12:00:00Z",
  "estimated_completion": "2024-05-03T12:15:00Z"
}
```

**Status values** – `queued | running | completed | failed`

### 4️⃣ Get Run Results
`GET /workflows/{run_id}/results`

```json
{
  "run_id": "run_20240503_120000_abc123",
  "status": "completed",
  "output": {
    "dataset_path": "axentx/surrogate-1-training-pairs/run_abc123",
    "records_processed": 1000000,
    "unique_records": 950000,
    "duplicates_removed": 50000
  },
  "completed_at": "2024-05-03T12:15:00Z"
}
```

### 5️⃣ Cancel Running Workflow
`POST /workflows/{run_id}/cancel`

```json
{
  "run_id": "run_20240503_120000_abc123",
  "status": "cancelled",
  "cancelled_at": "2024-05-03T12:05:00Z"
}
```

### 6️⃣ Health Check
`GET /health`

```json
{
  "status": "healthy",
  "version": "1.0.0",
  "timestamp": "2024-05-03T12:00:00Z"
}
```

## Error Model (applies to every endpoint)

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid dataset_id provided",
    "details": {
      "field": "dataset_id",
      "constraint": "must be a valid dataset identifier"
    }
  }
}
```

|------|---------|
| `AUTHENTICATION_ERROR` | Missing/invalid token |
| `VALIDATION_ERROR` | Bad request payload |
