## Unified Context‑Inheritance System

Below is a single, production‑ready implementation that blends the strongest ideas from the three candidate proposals:

* **Scoped, inheritable context variables** – each variable knows its scope (`global`, `workflow`, `agent`) and can be inherited by child workflows.
* **Copy‑on‑write shadowing** – when a child workflow writes to a variable that exists in a parent, a shadow copy is created so the parent is never mutated.
* **Fine‑grained expiration** – five policies (`NEVER`, `TIMEOUT`, `ABSOLUTE_TIME`, `IDLE`, `WORKFLOW_COMPLETE`) with per‑variable configuration.
* **Thread‑safe, lock‑free reads** – reads are lock‑free when no mutation occurs; writes acquire a re‑entrant lock.
* **Explicit deletion** – deleting a variable in a child only removes the shadow; the parent remains untouched.
* **Automatic cleanup** – a background thread purges expired variables at a configurable interval.
* **Extensible metadata** – each context can carry arbitrary metadata (e.g., workflow name, owner, tags).
