## Final Synthesized Answer

### 1. Diagnosis (merged, de-duplicated)
- **No persisted, date-scoped file manifest**: every training run re-enumerates HF repos via authenticated API, risking 429 rate-limits and non-deterministic inputs.
- **No CDN bypass during data loading**: training uses authenticated API calls (`load_dataset`, `hf_hub_download`) instead of public CDN, consuming rate-limit quota unnecessarily.
- **No deterministic sibling-repo selection for writes**: ingestion writes are not sharded across sibling repos, risking HF’s 128 commits/hour/repo cap.
- **Lightning Studio reuse not enforced**: new Studio instances are likely created per run, burning quota instead of reusing running ones.
- **No idle-stop guard for Lightning training**: if a Studio stops, `.run()` fails instead of restarting on an available machine.
- **Schema projection missing**: raw files with mixed schemas can trigger pyarrow `CastError` during loading.

---

### 2. Proposed Change (merged, prioritized)
Create `/opt/axentx/surrogate-1/training/manifest.py` and update training code to:
- Generate and persist a date-scoped `file_list.json` via a **single** authenticated `list_repo_tree` call (run once per date folder).
- Embed that list in training and fetch files **exclusively via HF CDN** (`resolve/main/...`) with **no Authorization header** during training.
- Add deterministic sibling-repo selection for writes (hash-slug → repo index) to avoid commit caps.
- Add Studio reuse + idle-stop guard before each training run.
- Add schema projection at parse time (select only `prompt`, `response`) to avoid `CastError`.

```python
import json
import hashlib
import os
from datetime import datetime
from pathlib import Path
from typing import Dict, List, Optional
from huggingface_hub import HfApi

HF_REPO = "datasets/axentx/surrogate-1"  # adjust as needed
SIBLING_REPOS = [
    "datasets/axentx/surrogate-1",
    "datasets/axentx/surrogate-1-sib1",
    "datasets/axentx/surrogate-1-sib2",
    "datasets/axentx/surrogate-1-sib3",
    "datasets/axentx/surrogate-1-sib4",
    "datasets/axentx/surrogate-1-sib5",
]

def build_manifest(date_str: str, out_path: Path, repo: str = HF_REPO) -> Dict:
    """
    Run once per date folder (e.g., on Mac orchestration node).
    Single authenticated list_repo_tree call.
    """
    api = HfApi()
    folder_path = f"batches/mirror-merged/{date_str}"
    items = api.list_repo_tree(repo_id=repo, path=folder_path, recursive=True)
    file_names = [it.rfilename for it in items if it.rfilename.endswith(".parquet")]

    manifest = {
        "date": date_str,
        "repo": repo,
        "created_at": datetime.utcnow().isoformat() + "Z",
        "files": sorted(file_names),
        "cdn_prefix": f"https://huggingface.co/datasets/{repo}/resolve/main"
    }
    out_path.parent.mkdir(parents=True, exist_ok=True)
    with open(out_path, "w") as f:
        json.dump(manifest, f, indent=2)
    print(f"Manifest written to {out_path} with {len(file_names)} files")
    return manifest

def pick_sibling_repo(slug: str, repos: Optional[List[str]] = None) -> str:
    """Deterministic sibling repo selection to avoid HF commit cap."""
    if repos is None:
        repos = SIBLING_REPOS
    idx = int(hashlib.sha256(slug.encode()).hexdigest(), 16) % len(repos)
    return repos[idx]

def resolve_cdn_url(repo: str, file_path: str) -> str:
    return f"https://huggingface.co/datasets/{repo}/resolve/main/{file_path}"
```
