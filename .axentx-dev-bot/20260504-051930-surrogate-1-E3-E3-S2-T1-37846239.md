# Common Pitfalls for S3 Source Type

## Authentication Issues
- Ensure your AWS credentials have the correct permissions
- Verify the credentials are in the correct format (access key ID and secret access key)
- Check that the credentials are not expired

## Bucket Permissions
- Make sure the bucket policy allows access from your IP or VPC
- Verify that the bucket exists and the name is correct
- Check that the bucket is not suspended or locked

## Object Access
- Ensure the objects you are trying to access exist in the bucket
- Verify that the objects are not encrypted with a key you don't have access to
- Check that the objects are not versioned or locked

## Performance Issues
- Consider using multi-part uploads for large files
- Verify that your network connection is stable and fast enough
- Check that you are not hitting the S3 request rate limits
```
