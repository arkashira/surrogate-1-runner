# samples/home-assistant/README.md
# Axentx Surrogate‑1 Home Assistant Quick‑Start Sample

This repository contains a minimal Home Assistant project that shows how to
expose an **Axentx Surrogate‑1** inference result as a sensor entity.

## Prerequisites

1. **Home Assistant** (any recent version) installed and reachable.
2. An **Axentx API key** – you can create one in the Axentx console under
   *API Keys*.
3. (Optional) Python 3 and the Axentx SDK if you want to experiment with the
   SDK locally:  

   ```bash
   pip install axentx
   ```

## Step‑by‑Step Guide

### 1️⃣ Create an API key

1. Log in to the Axentx portal.
2. Navigate to **Developer → API Keys**.
3. Click **Create New Key**, give it a name (e.g., `home-assistant-demo`),
   and copy the generated secret.

### 2️⃣ Clone this sample

```bash
git clone https://github.com/axentx/surrogate-1.git
cd surrogate-1/samples/home-assistant
```

### 3️⃣ Add the configuration to Home Assistant

1. Copy `configuration.yaml` into your Home Assistant config directory
   (usually `/config/` or `~/.homeassistant/`).

   ```bash
   cp configuration.yaml /path/to/your/homeassistant/config/
   ```

2. Edit the file and replace `YOUR_AXENTX_API_KEY` with the key you created
   in step 1.

3. Restart Home Assistant (Configuration → Server Controls → Restart).

### 4️⃣ Verify the sensor

After the restart, open the Home Assistant UI:

- Go to **Developer Tools → States**.
- Search for `sensor.surrogate_1_inference`.

You should see the latest inference result and two attributes:

- `timestamp`
- `model_version`

### 5️⃣ (Optional) Use the SDK to push custom data

If you want to send custom prompts to Surrogate‑1 from a script, you can
use the SDK:

```python
from axentx import Surrogate1Client

client = Surrogate1Client(api_key="YOUR_AXENTX_API_KEY")
response = client.infer(prompt="Explain quantum entanglement in one sentence.")
print(response.result)
```
