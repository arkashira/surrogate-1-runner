**RefactorAI**

RefactorAI is an AI-driven command-line tool designed to provide automated refactoring suggestions for problematic code patterns in Java files. It offers both interactive and non-interactive modes, allowing developers to choose the best approach for their workflow.

**Key Features:**

1. **Problematic Pattern Detection:** RefactorAI scans Java files for common problematic patterns such as empty catch blocks, magic numbers, and long methods. It can be extended to support more patterns in the future.

2. **Refactoring Suggestions:** For each detected problematic pattern, RefactorAI generates a list of refactoring suggestions. These suggestions are presented along with a description of the issue and the location in the code.

3. **Interactive Mode:** In interactive mode, RefactorAI prompts the user to select a refactoring option for each issue. It then applies the chosen refactoring to the code and displays the result.

4. **Non-Interactive Mode:** In non-interactive mode, RefactorAI displays all available refactoring options for each issue but does not apply any changes to the code. This mode is useful for reviewing and understanding the suggested refactorings before applying them.

**Usage:**

To use RefactorAI, run the following command in your terminal:

```bash
java -jar refactorai.jar refactor --file <path_to_java_file> [--interactive]
```

Replace `<path_to_java_file>` with the path to the Java file you want to refactor. Add the `--interactive` flag to enable interactive mode.

**Example Output (Interactive Mode):**

```

Select an option (1-3) or 'n' to skip: 1

Applying refactoring:
try {
    // risky code
} catch (Exception e) {
    logger.error("An error occurred", e);
}
