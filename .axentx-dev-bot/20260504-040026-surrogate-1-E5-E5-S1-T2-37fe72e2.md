## 1. Acceptance Criteria
- **Policy Summary**: The PDF must contain a table listing each policy with its compliance status (e.g., *Compliant*, *Non‑Compliant*, *Pending*).  
- **Model Compliance**: For every model in the selected month, the report must show a row with model ID, name, and compliance status.  
- **Evidence Artifacts**: Each evidence item must be listed with a clickable URL that resolves to the artifact location.  
- **Digital Signature**: The PDF must be signed using the platform’s private key; the signature must be verifiable via the public key and include a SHA‑256 hash of the PDF content.  
- **Performance**: Report generation (data aggregation + PDF creation) must finish in ≤ 5 seconds when processing up to 200 models.  
- **Endpoint**: A GET request to `/api/report?month=YYYY‑MM` returns a `200 OK` with `Content-Type: application/pdf` and the signed report as the body.  
- **Error Handling**: Invalid month format or missing data returns appropriate HTTP status codes (e.g., 400 Bad Request, 404 Not Found).

func TestSignPDF(t *testing.T) {

    gen := NewReportGenerator(mockPolicyRepo{}, mockModelRepo{}, mockEvidenceRepo{}, key)

    signed, err := gen.signPDF(pdf)
    assert.NoError(t, err)
    assert.NotEmpty(t, signed)
    // Verify signature
    pubKey := &key.PublicKey
    err = rsa.VerifyPKCS1v15(pubKey, crypto.SHA256, sha256.Sum256(pdf)[:], signed)
