# Integration with Dokka/KDoc Pipeline

This guide will help you integrate the new overload linking tool with your existing Dokka/KDoc pipeline.

## Prerequisites

- Dokka 1.9.0 or later
- Kotlin 1.9.0 or later
- Gradle 8.0 or later

## Installation

1. Add the following dependency to your `build.gradle.kts` file:

```kotlin
plugins {
    id("org.jetbrains.dokka") version "1.9.0"
}

dependencies {
    dokkaPlugin("org.jetbrains.dokka:dokka-overload-link-plugin:1.0.0")
}
```

2. Apply the Dokka plugin to your Kotlin modules:

```kotlin
kotlin {
    sourceSets.forEach { sourceSet ->
        dokka {
            sourceSet.configureDokkaSourceSet()
        }
    }
}
```

## Configuration

Add the following configuration to your `dokkaConfiguration` block in `build.gradle.kts`:

```kotlin
dokka {
    configuration {
        dokkaSourceSets {
            create("main") {
                // Your existing Dokka configuration
            }
        }
    }
    plugins {
        overloadLink {
            // Overload linking tool configuration
            enabled.set(true)
            // Additional configuration options
        }
    }
}
```

## Usage

After configuring the plugin, you can generate your documentation as usual:

```bash
./gradlew dokkaHtml
```

The overload linking tool will automatically process your KDoc comments and generate appropriate links between overloaded functions.

## Troubleshooting

If you encounter any issues during the integration process, please check the following:

1. Ensure that you are using compatible versions of Dokka, Kotlin, and Gradle.
2. Verify that the plugin is correctly applied to your Kotlin modules.
3. Check the Dokka build output for any error messages related to the overload linking tool.

If you are still experiencing issues, please file an issue on the [project's GitHub repository](https://github.com/axentx/surrogate-1/issues).
```
