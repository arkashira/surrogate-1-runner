## CDN-backed file listing and parquet fetches

To reduce load on the Hugging Face API (429/slow-start), Surrogate now retrieves folder/file lists via a single CDN manifest and fetches Parquet files directly from CDN rather than per-file auth calls.

- **Performance**: Fewer API calls and lower latency for listing and bulk reads.
- **Fallback behavior**: If the CDN is unavailable, the client may fall back to cached/stale manifests. This can mask upstream failures and delay detection of stale data.
  - Mitigations: staleness is logged in client metrics; critical workflows can be gated on freshness (e.g., block writes if manifest age exceeds TTL and no live refresh is possible); CDN-vs-API success rates are emitted to detect upstream degradation.
- **Retries**: Transient failures use bounded retries (2 attempts) with exponential backoff and jitter to limit tail latency. The UI is surfaced fast-failure paths and may use stale-while-revalidate to remain responsive.

If your workflow requires strongly consistent listings, ensure live refresh is available or enforce freshness checks before writes.
