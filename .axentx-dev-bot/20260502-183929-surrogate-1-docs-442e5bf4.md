## Connectors and adapters

Surrogate-1 provides a connector abstraction and concrete adapters for trading and risk operations.

- **Connector contract** (`fetch_data`, `send_order`, `check_risk`) with documented SLAs:
  - `fetch_data` p95 < 200 ms
  - `check_risk` p95 ≤ 100 ms
- **SimulatedBrokerAdapter**: implements `send_order` with fill simulation, order tracking, and timing metadata; `fetch_data` delegates to other adapters and raises `DataError` on failure.
- **Cancellation**: `cancel_order` returns success/failure and preserves order state.
- **Risk**: `check_risk` returns a `RiskResult` with pass/fail, limit/position details, and enforces the 100 ms p95 target locally.

## Models

Dataclasses for core entities include UTC timestamps and sensible defaults and are serializable for logging/metrics:
- `Order`, `Fill`, `Quote`, `Snapshot`, `RiskResult`

Validation and safety checks prevent invalid runtime state:
- `fill_probability` must be in [0, 1]
- `order_id` must be non-empty
- `fill_delay` must be non-negative

## Errors

Custom exceptions carry optional codes/details for observability and routing:
- `ConnectorError`, `OrderError`, `DataError`, `RiskError`
