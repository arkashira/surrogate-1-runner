# surrogate-1-runner

Parallel public-dataset ingest workers for the
[axentx/surrogate-1-training-pairs](https://huggingface.co/datasets/axentx/surrogate-1-training-pairs)
HuggingFace dataset.

## What this does

Every 30 minutes (or on `workflow_dispatch`), GitHub Actions launches **16 parallel runners**.
Each runner takes a deterministic 1/16 slice (`slug-hash bucket = SHARD_ID`)
of the public dataset list defined in `bin/dataset-enrich.sh`, streams,
normalizes per-schema, dedups via the central md5 hash store, and uploads
its output to a unique path on the dataset repo:

## Quickstart Guide

### Setup

1. Clone the repository:

```bash
git clone https://github.com/axentx/surrogate-1-runner.git
```

2. Install dependencies:

```bash
cd surrogate-1-runner
pip install -r requirements.txt
```

### Example Usage

#### Claude

```python
from surrogate_1_runner.claude import Claude

claude = Claude()
claude.process_dataset("path/to/claude_dataset")
```

#### Minimax

```python
from surrogate_1_runner.minimax import Minimax

minimax = Minimax()
minimax.process_dataset("path/to/minimax_dataset")
```

### Adding New Providers

To add a new provider, create a new class in the `providers` directory and implement the `process_dataset` method.
