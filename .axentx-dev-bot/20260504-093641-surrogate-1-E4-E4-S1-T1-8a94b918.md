# Streaming API Documentation

## Overview

The Streaming API allows you to process data from the surrogate-1 dataset in a streaming fashion, which is particularly useful for large datasets that cannot be loaded into memory all at once.

## API Endpoints

### Stream Data

`GET /stream`

This endpoint streams data from the surrogate-1 dataset. You can specify the shard ID and other parameters to control the data stream.

#### Parameters

- `shard_id` (required): The ID of the shard to stream data from.
- `limit` (optional): The maximum number of records to stream. Default is 1000.
- `offset` (optional): The number of records to skip before starting to stream. Default is 0.

#### Example Request

```bash
curl -X GET "https://api.surrogate-1.axentx.com/stream?shard_id=1&limit=100" \
     -H "X-API-Key: your-api-key"
```

#### Response

The response is a stream of JSON objects, one per line. Each object represents a record from the dataset.

```json
{"id": "1", "data": "record1"}
{"id": "2", "data": "record2"}
...
```

## Back-Pressure and Memory Guarantees

The Streaming API is designed to handle back-pressure and ensure memory efficiency. Here are some key points:

- **Back-Pressure Handling**: The API uses a back-pressure mechanism to ensure that the client can keep up with the data stream. If the client falls behind, the API will automatically adjust the streaming rate.
- **Memory Efficiency**: The API streams data in chunks, so it does not load the entire dataset into memory. This makes it suitable for processing large datasets.

## Example Project

An example project demonstrating how to use the Streaming API can be found in the `examples` directory. The project includes a sample implementation of a stream parser and instructions on how to run it.

## Release Notes

The Streaming API is part of the breaking-fix improvements in the latest release. For more details, please refer to the [release notes](https://github.com/axentx/surrogate-1/releases).

## Blog Post Draft

A draft of the blog post highlighting the Streaming API and its features is ready for marketing review. The post includes use cases, performance metrics, and code snippets to help users get started quickly.

// /opt/axentx/surrogate-1/examples/streamParserExample.ts
import axios from 'axios';

class StreamParser {
  private apiKey: string;
  private baseUrl: string;

  constructor(apiKey: string, baseUrl: string = 'https://api.surrogate-1.axentx.com') {
    this.apiKey = apiKey;
    this.baseUrl = baseUrl;
  }

  async streamData(shardId: number, limit: number = 1000, offset: number = 0) {
    const url = `${this.baseUrl}/stream?shard_id=${shardId}&limit=${limit}&offset=${offset}`;
    const response = await axios.get(url, {
      headers: {
        'X-API-Key': this.apiKey,
      },
      responseType: 'stream',
    });

    return response.data;
  }

  async parseStream(shardId: number, limit: number = 1000, offset: number = 0) {
    const stream = await this.streamData(shardId, limit, offset);
    let data = '';

    stream.on('data', (chunk) => {
      data += chunk;
    });

    stream.on('end', () => {
      console.log('Stream ended');
      const records = data.split('\n').filter(Boolean).map(JSON.parse);
      console.log('Parsed records:', records);
    });

    stream.on('error', (err) => {
      console.error('Stream error:', err);
    });
  }
}

// Example usage
const apiKey = 'your-api-key';
const streamParser = new StreamParser(apiKey);

streamParser.parseStream(1, 100, 0)
  .then(() => console.log('Stream parsing completed'))
  .catch((err) => console.error('Error:', err));
