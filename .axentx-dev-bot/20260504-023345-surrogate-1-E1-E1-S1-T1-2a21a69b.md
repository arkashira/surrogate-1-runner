class LLMCallMiddleware:
    def __init__(self, get_response: Callable[[request], response]):
        self.get_response = get_response

    def __call__(self, request):
        response = self.get_response(request)

        if 'llm_call' in request.path:
            model = request.headers.get('X-LLM-Model')
            endpoint = request.path
            token_count = int(request.headers.get('X-Token-Count', 0))
            cost = float(request.headers.get('X-Cost', 0.0))

            record_usage(model, endpoint, token_count, cost, request.user.id, request.team.id)

        return response

def record_usage(model: str, endpoint: str, token_count: int, cost: float, user_id: int, team_id: int):

## 1. Acceptance Criteria  

1. **Middleware Interception** – Every outbound LLM request passes through the new middleware and the request payload contains a `model`, `endpoint`, `token_count`, and `cost` field.  
2. **Database Persistence** – The middleware inserts a document into the `metrics` collection with the following keys: `model`, `endpoint`, `token_count`, `cost`, `user_id`, `team_id`, `timestamp`.  
3. **No SQL Injection** – The `record_usage` function uses parameterized queries or an ODM that escapes inputs; no raw string interpolation is allowed.  
4. **Background Aggregation** – A scheduled worker runs daily, aggregates `token_count` and `cost` per `model` and `team_id`, and stores the result in a `daily_totals` collection with a `date` field.  
5. **REST Exposure** – A GET endpoint `/api/usage/daily?date=YYYY-MM-DD` returns the aggregated totals for the requested date in JSON.  
6. **Error Handling & Logging** – Any exception during recording or aggregation logs an error and does not crash the worker or API.  
7. **Performance** – Recording a metric does not add more than 5 ms latency to the LLM call under normal load (verified via load test).
