Below is a single, self‑contained Flask blueprint that

1. **Loads** the `docs.md` file on every request (so the UI always shows the latest content).  
2. **Renders** the Markdown to HTML using the `markdown` package (with a graceful fallback).  
3. **Provides** a lightweight server‑side keyword search that returns line numbers and snippets.  
4. **Serves** a minimal, self‑contained HTML page that contains the search UI and the rendered docs.  
5. **Keeps** the code simple, well‑documented and ready for unit‑testing.

> **Why this design?**  
> * The blueprint keeps the rest of the application untouched.  
> * Loading the file on every request guarantees “auto‑update” without a file‑watcher.  
> * The search logic is intentionally simple – it scans the raw Markdown, so it works even if the Markdown parser is missing.  
> * All static assets (CSS, JS) live inside the template – no external files are required.  
> * The code is fully type‑annotated, has docstrings, and follows PEP‑8.
