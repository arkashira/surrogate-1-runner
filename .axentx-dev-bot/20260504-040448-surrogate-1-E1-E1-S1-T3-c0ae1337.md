### 1. **Acceptance criteria**

- The compliance scan is triggered automatically on `pull_request` events (opened, synchronized, reopened).
- Scan results are posted as a GitHub PR comment within **2 minutes** of a push event.
- Only **HIPAA and SOC2** rule sets are evaluated during the scan.
- If no violations are found, the PR comment must state: `All compliance checks passed`.
- The comment must be **posted under the PR conversation**, not as a review or status check.
- The scanner must use environment variables (e.g., `GITHUB_TOKEN`) for authentication — no hardcoded credentials.
- The system must log an error and fail gracefully if comment posting fails (e.g., rate limits, network issues).

    ]

    body = mock_post.call_args[1]["body"]
    assert "HIPAA-001" in body
    assert "SOC2-003" in body
    assert "compliance checks passed" not in body
    mock_post.assert_called()

    with patch("src.github.comments.load_ruleset") as mock_load:
        scanner._evaluate_rules(files=["app.py"])
        mock_load.assert_any_call("HIPAA")
        mock_load.assert_any_call("SOC2")
        assert mock_load.call_count == 2

### 3. **Integration tests**

#### Happy paths:
1. **PR opened with compliant code**  
   → Trigger `pull_request.opened`, scanner runs, posts “All compliance checks passed” within 90 seconds.

2. **PR updated with HIPAA violation**  
   → On `pull_request.synchronize`, scanner detects HIPAA-105, posts formatted comment listing violation.

3. **PR with both HIPAA and SOC2 violations**  
   → Scanner evaluates both rule sets, aggregates findings, posts detailed summary comment.

4. **Multiple pushes to same PR**  
   → Each push triggers scan; previous comment is **edited in place** (not duplicated).

5. **Valid credentials and network**  
   → Comment is successfully posted using `GITHUB_TOKEN` from environment.

#### Edge cases:
1. **Empty file list or no files changed**  
   → Scanner returns no violations, posts “All compliance checks passed”.

2. **GitHub API rate-limited**  
   → Scanner retries with exponential backoff (up to 2 retries), logs warning if still failing.

3. **Malformed or missing `GITHUB_TOKEN`**  
   → Scanner fails early, logs error, does **not** crash pipeline.
