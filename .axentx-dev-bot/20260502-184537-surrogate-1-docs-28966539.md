## Connectors

Surrogate-1 provides pluggable connectors for risk and execution with strict contracts and safe error handling.

- All connectors inherit from `BaseConnector` and enforce `init()` / `call()` / `health()`.
- `BaseConnector` wraps raw exceptions into `ConnectorError` (never leaks stack traces) and `ConnectorError.to_dict()` returns a stable schema: `error_type`, `connector_type`, `connector_name`, `message`, `details`.
- `call()` returns `{"success": bool, "output": Any, "metadata": Dict[str, Any]}` on success and raises **only** `ConnectorError` on failure.
- `health()` returns one of `healthy` | `unhealthy` | `uninitialized` | `disabled` and never raises.
- Config requires `name` (str) and optional `enabled` (bool); subclasses validate via `_do_init` and raise `ConnectorError` for invalid configs.

### Risk connector
- `MaxNotionalConnector` rejects calls exceeding `max_notional` (numeric). On rejection, returns `success=False` and metadata containing the limit and attempted notional.

### Execution connector
- `SimulatedExecutionConnector` accepts valid orders and returns deterministic fills with `execution_id`, `filled_qty`, `avg_price`, `fees`, and metadata containing `simulated_latency_ms`.
