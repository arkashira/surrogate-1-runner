## 1. Dockerfile (Python 3.11 slim + Poetry)

```dockerfile
# Dockerfile
# -------------------------------------------------------------
# Build image for the surrogate‑1 English module
# -------------------------------------------------------------
FROM python:3.11-slim

# Avoid creating .pyc files and keep logs to stdout
ENV PYTHONDONTWRITEBYTECODE 1
ENV PYTHONUNBUFFERED 1

# Install system deps needed for building wheels
RUN apt-get update && apt-get install -y --no-install-recommends \
        build-essential \
        libpq-dev \
    && rm -rf /var/lib/apt/lists/*

# Install Poetry (pin to a stable version)
RUN pip install --no-cache-dir poetry==1.8.3

# Work‑dir
WORKDIR /app

# Copy Poetry lockfiles first – Docker cache hit if deps unchanged
COPY pyproject.toml poetry.lock ./

# Install Python deps (no‑dev for production image)
RUN poetry config virtualenvs.create false && \
    poetry install --no-dev

# Copy the rest of the code
COPY . .

# Expose the default port (adjust if your app uses a different one)
EXPOSE 8000

# Default command – change to your entry‑point
CMD ["python", "-m", "surrogate1.runner"]
```

> **Why this Dockerfile?**  
> * Uses the official slim base for minimal size.  
> * Poetry guarantees deterministic installs.  
> * No virtualenv inside the container – keeps the image lean.  
> * All build‑time deps are cached by Docker, so subsequent builds are fast.
