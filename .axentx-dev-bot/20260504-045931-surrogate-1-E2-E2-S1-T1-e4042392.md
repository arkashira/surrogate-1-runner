## 1. Acceptance Criteria
- **AC1** – When the integration tool is invoked on a system without a compatible Java runtime, it downloads the exact Java version specified in `config/java_version` and installs it to `$HOME/.surrogate/java/<version>`.
- **AC2** – After installation, the tool sets `JAVA_HOME` and updates the system `PATH` (within the tool’s process environment) so that `java -version` reports the installed version.
- **AC3** – If a compatible Java version already exists in the expected location, the tool skips download and uses the existing installation.
- **AC4** – If a conflicting Java version (different major version) is detected in the system `PATH`, the tool logs a warning, isolates its own Java runtime, and ensures subsequent commands use the managed version.
- **AC5** – All download, install, and configuration steps are idempotent; repeated runs produce no side‑effects and return a success status.
- **AC6** – Failure to download (e.g., network error, 404) aborts the process with a clear error message and does not leave partial files on disk.
- **AC7** – The tool provides a CLI flag `--java-status` that prints the currently managed Java version and its installation path.
