//
// `client` – Kubernetes client used to list/watch pods.
// `alertCh` – Channel where alert strings are sent.

// `silentFailureThreshold` – Time without a pod status update before reporting a silent failure.

   - Setup: Create a pod with status "Running" and a last heartbeat time older than the silent failure threshold.
   - Action: Run the `handlePod` function.
   - Expected Result: Silent failure detected.

2. **Pod Status Updated but Not Running:**
   - Setup: Create a pod with status "Terminated".
   - Action: Run the `handlePod` function.
   - Expected Result: Pod is not running.
