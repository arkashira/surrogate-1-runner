- **Happy Case 1:** Set a profit threshold of 100. Current profit is 150. An email alert is sent.
- **Happy Case 2:** Set a loss threshold of -50. Current loss is -75. An email alert is sent.
- **Happy Case 3:** Set a profit threshold of 100. Current profit is 90. No email alert is sent.
- **Edge Case 1:** Set a profit threshold of 100. Current profit is 100. No email alert is sent due to the threshold being exactly met.

  - **Detection:** User reports receiving duplicate email alerts, or unit tests fail.
  - **Mitigation:** Implement a mechanism to track recent threshold breaches and suppress duplicate alerts within the allowed time frame.

- **Risk:** Incorrect email address is used for sending alerts.
  - **Detection:** Email alerts are not received by the user, or the wrong email address is used in the logs.
