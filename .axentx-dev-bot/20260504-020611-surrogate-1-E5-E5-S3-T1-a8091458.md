### 2. Pattern Engine

```python
from dataclasses import dataclass
from typing import List, Tuple

@dataclass
class Pattern:
    regex: str
    name: str
    severity: str          # Critical / High / Medium / Low
    confidence: str        # high / medium / low
    flags: int = 0         # re.IGNORECASE, re.MULTILINE, etc.

# Example list – extend as needed
PATTERNS: List[Pattern] = [
    # PRIVATE KEYS – Critical
    Pattern(r'-----BEGIN\s+(?:RSA|EC|OPENSSH)?\s*PRIVATE\s+KEY-----',
            'pem_private_key', 'Critical', 'high', re.IGNORECASE),
    Pattern(r'\b[0-9a-fA-F]{64}\b',
            'hex_private_key', 'Critical', 'high'),

    # API KEYS – High
    Pattern(r'(?:api[_-]?key|apikey)\s*[=:]\s*["\']([0-9a-zA-Z\-_]{16,})["\']',
            'generic_api_key', 'High', 'high', re.IGNORECASE),
    Pattern(r'AKIA[0-9A-Z]{16}',
            'aws_access_key', 'Critical', 'high', re.IGNORECASE),
    Pattern(r'(?<![A-Za-z0-9])sk-[0-9a-zA-Z]{32,}',
            'openai_api_key', 'Critical', 'high', re.IGNORECASE),

    # PASSWORDS – High
    Pattern(r'(?:password|passwd|pwd|secret)\s*[=:]\s*["\']([^"\'\s]{8,})["\']',
            'password', 'High', 'medium', re.IGNORECASE),

    # CONNECTION STRINGS – High
    Pattern(r'(?:mongodb|postgres|mysql|redis):\/\/[^:]+:[^@]+@',
            'connection_string', 'High', 'medium', re.IGNORECASE),

    # GENERIC SECRETS – Medium
    Pattern(r'(?:secret|key|token)\s*[=:]\s*["\']([A-Za-z0-9+/=]{32,})["\']',
            'base64_secret', 'Medium', 'low', re.IGNORECASE),
    Pattern(r'0x[0-9a-fA-F]{32,}',
            'hex_secret', 'Medium', 'low', re.IGNORECASE),
]
```

**Why this list?**  
* Covers the most common secret types.  
* Uses `re.IGNORECASE` where the format is case‑insensitive.  
* Keeps regexes short enough to avoid catastrophic backtracking.  
* Allows easy addition of new patterns.

### 3. False‑Positive Filters

```python
def is_false_positive(line: str) -> bool:
    # 1. Skip comment lines
    if line.lstrip().startswith(('#', '//', '/*', '*')):
        return True

    # 2. Skip obvious non‑secret strings
    if any(tok in line for tok in ('example', 'demo', 'test', 'sample')):
        return True

    # 3. Skip lines that look like environment variable names
    if re.match(r'^[A-Z0-9_]+\s*[=:]\s*["\'].*["\']$', line):
        return False  # could still be a secret – let patterns decide

    return False
```

*The filter is intentionally conservative; the pattern engine does the heavy lifting.*

---

### 4. Scanner Logic

```python
import pathlib
import re

def scan_file(path: pathlib.Path) -> List[SecretFinding]:
    findings = []
    content = path.read_text(errors='ignore')
    lines = content.splitlines()

    for ln, line in enumerate(lines, start=1):
        if is_false_positive(line):
            continue

        for pat in PATTERNS:
            for m in re.finditer(pat.regex, line, pat.flags):
                findings.append(SecretFinding(
                    type=pat.name,
                    pattern=pat.regex,
                    line_number=ln,
                    line_content=line.strip(),
                    severity=pat.severity,
                    confidence=pat.confidence,
                    metadata={'file': str(path)}
                ))
    return findings
```

# TDD Test Plan: Hardcoded Secret Detection Rule

## 1. Acceptance Criteria

- **[AC-1]** Scanner detects Ethereum-style private keys (64 hex chars, optionally with `0x` prefix) in source files with ≥95% recall
- **[AC-2]** Scanner detects API key patterns (AWS, generic 20+ char alphanumeric, base64-encoded) with ≥90% recall
- **[AC-3]** Scanner detects password assignments (`password=`, `pwd:`, `secret=`) in string literals with ≥90% recall
- **[AC-4]** Private key detections are assigned severity level "Critical"; API keys and passwords are "High"
- **[AC-5]** False-positive rate ≤5% on non-secret code (tested against 1000+ open-source contract samples)
- **[AC-6]** Output format includes: `file`, `line`, `column`, `pattern_type`, `severity`, `matched_text` (redacted to first 8 chars)
- **[AC-7]** Scanner exits with code 1 when secrets found, 0 when clean (CI-compatible)
