## Unified Compliance‑Ready Web‑Shell

Below is a **single, production‑grade implementation** that pulls the strongest ideas from the two candidates:

| Feature | Why it matters | How we solved it |
|---------|----------------|------------------|
| **Audit logging** | SOC‑2, HIPAA, PCI‑DSS all require immutable, tamper‑evident logs. | Write every command (pre‑ and post‑execution) to a **daily, append‑only JSONL file** and keep an **in‑memory index** for fast queries. |
| **Audit ID** | Correlating a request, its output, and any errors is essential for forensic analysis. | Each audit record gets a **cryptographically‑random UUIDv4** (`auditId`). |
| **Immutable storage** | Logs must survive tampering. | Store logs in a **separate, read‑only filesystem mount** (`/var/log/webshell/audit`) and **append‑only**. |
| **Export & report** | External tools (SIEM, SOAR) need structured data. | `generateReport(start, end)` returns an array; `exportReport(path, data)` writes pretty‑printed JSON. |
| **Event hooks** | Real‑time correlation or alerting. | `onAudit(callback)` registers callbacks that fire on every audit record. |
| **Secure auth** | Prevent unauthorized command execution. | Replace the demo token map with **JWT** + **OAuth2** or **LDAP** in production. |
| **Command execution** | Avoid blocking the event loop. | Use `child_process.spawn` with a timeout; capture stdout/stderr. |
| **Rate limiting** | Protect against DoS. | Simple per‑token counter + time window. |
| **Configuration** | Avoid hard‑coding paths. | Use a `config.json` or environment variables. |
| **Testing** | Ensure correctness. | Jest tests for audit record creation, report filtering, and export. |

const auditIndex = new Map();

/**
 * Generate a cryptographically‑strong audit ID.
 */
function generateAuditId() {
  return crypto.randomBytes(16).toString('hex');
}

 * The file is named YYYY-MM-DD.log

async function persistAudit(record) {

  const filePath = path.join(LOG_DIR, `${date}.log`);

  await appendFile(filePath, line, 'utf8');

  auditIndex.get(date).push(record.auditId);
