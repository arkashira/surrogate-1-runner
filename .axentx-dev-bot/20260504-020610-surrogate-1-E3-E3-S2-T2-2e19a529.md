Below is a **single, production‑ready configuration subsystem** that pulls the best ideas from the two candidates:

* A **central registry** (`src/config/rules.py`) that holds every rule and its default/recommended severity.  
* A **loader** (`src/config/loader.py`) that:
  * Builds a sane default configuration when no file is supplied.  
  * Parses a YAML file, validates its shape, checks that every rule is known, and that the severity is one of the allowed values.  
  * Raises clear, actionable errors when something is wrong.  
* A **test‑suite** (`tests/config/test_loader.py`) that exercises the loader in isolation by monkey‑patching the registry.

The two modules are intentionally lightweight, fully typed, and fully documented.  
All contradictions have been resolved in favour of correctness and clarity:

| Issue | Resolution |
|-------|------------|
| `recommended` vs `default_severity` naming | The registry now uses `default_severity`. The loader reads that key. |
| Missing registry fallback | The loader still imports the registry but falls back to an empty dict if the import fails, allowing unit‑tests to monkey‑patch it. |
| Allowed severities | A single `SEVERITY_LEVELS` constant is exported from the registry and reused by the loader. |
| Validation error messages | All `ValueError`s contain the offending rule name and the set of allowed values. |

    "no-tabs": {"default_severity": "warning"},
    "indentation": {"default_severity": "error"},
    "unused-import": {"default_severity": "warning"},
    # Add more rules here as the linter evolves.
}

# Allowed severity levels

# --------------------------------------------------------------------------- #
# Convenience helpers (optional, but handy for callers)
# --------------------------------------------------------------------------- #

def is_valid_severity(sev: str) -> bool:
    """Return ``True`` if *sev* is one of the allowed severity levels."""
    return sev in SEVERITY_LEVELS
