   - Metrics are emitted for each send attempt: success/failure, latency, retry count, and channel (slack/email).
   - Configuration is team-scoped (configurable Slack webhook URL and digest recipients) and validated at startup; invalid config prevents job scheduling and logs an error.

     - `sendWithRetry(providerSend, maxRetries, backoffBase)` retries up to N times on provider failure; succeeds on first success; respects exponential backoff schedule.
     - `sendWithRetry` stops retrying and throws after max retries; logs each attempt.
   - DailyDigestJob

   - SlackNotifier

   - UnsubscribeHandler
     - `verify(token)` returns { valid, requestId, userId, type } for valid tokens; returns invalid for tampered/expired tokens.
     - `optOut(requestId?, userId, type)` persists opt-out record; subsequent digest builds exclude the user.

3. **Integration tests**
   - Happy paths
     - Status change triggers Slack webhook and email to watchers; both contain correct request ID and status; Slack receives 200 OK; email provider receives message.
     - Daily digest at 09:00 UTC sends one email per watcher per team with correct active/overdue counts; no duplicate emails for same watcher.
     - Request becomes overdue; immediate overdue alert is sent to watchers and Slack; next 09:00 UTC sends one daily overdue summary (no duplicate per-request alert that day).
   - Edge cases
     - Slack webhook returns 500; retry logic runs 3 times with backoff; failure is logged and does not block email send.

     - User opts out globally; subsequent status changes and daily digests exclude that user; unsubscribe link revokes correctly and is idempotent.
