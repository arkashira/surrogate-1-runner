### 2.1 `ShellResult` – the data that travels back to the client

```java
package com.axentx.surrogate.service;

import com.fasterxml.jackson.annotation.JsonInclude;

/**
 * Immutable DTO that holds everything a caller needs to know about a
 * shell execution.
 */
@JsonInclude(JsonInclude.Include.NON_NULL)
public final class ShellResult {

    private final String command;
    private final int exitCode;
    private final String stdout;
    private final String stderr;

    public ShellResult(String command, int exitCode, String stdout, String stderr) {
        this.command = command;
        this.exitCode = exitCode;
        this.stdout = stdout;
        this.stderr = stderr;
    }

    public String getCommand() { return command; }
    public int getExitCode() { return exitCode; }
    public String getStdout() { return stdout; }
    public String getStderr() { return stderr; }
}
```

## 3. Validation – “whitelist‑only” approach

> **Why?** Allowing *any* string to be passed to `/bin/bash -c` is a massive injection surface.  
> **What we do** – keep a small, configurable list of commands that are allowed, and reject everything else.

```java
package com.axentx.surrogate.validation;

import org.springframework.stereotype.Component;

import java.util.Collections;
import java.util.HashSet;
import java.util.Set;

/**
 * Very simple whitelist validator. In a real product you could load the list
 * from a DB, a YAML file, or an external policy service.
 */
@Component
public class CommandValidator {

    /** Example whitelist – extend as needed */
    private static final Set<String> ALLOWED = Collections.unmodifiableSet(
            new HashSet<>(Set.of(
                    "ls", "pwd", "whoami", "date", "uptime",
                    "df -h", "free -m", "cat /etc/hostname"
            ))
    );

    /**
     * Returns {@code true} if the supplied raw command matches an entry in the whitelist.
     * The check is *exact* – no extra arguments, no pipes, no redirections.
     */
    public boolean isAllowed(String rawCommand) {
        if (rawCommand == null) return false;
        String trimmed = rawCommand.trim();
        return ALLOWED.contains(trimmed);
    }
}
```

- [ ] [ ] **AC-3**: Every shell command execution (success or failure) generates a log entry with command, user, timestamp, and outcome
- [ ] **AC-4**: Invalid commands return HTTP 400 with descriptive error; unauthorized commands return HTTP 403; system failures return HTTP 500
- [ ] **AC-5**: UI error page displays user-friendly messages without exposing stack traces or system paths
- [ ] **AC-6**: All error responses follow consistent JSON schema across all failure scenarios
