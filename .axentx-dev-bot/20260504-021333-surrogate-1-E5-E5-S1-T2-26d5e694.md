# Synthesized Surrogate-1 API Reference

## Overview

Surrogate-1 offers a robust RESTful API enabling users to submit datasets for ingestion, monitor job statuses, and download processed results. The API is designed to be intuitive and efficient, supporting various operations through well-defined endpoints. All endpoints are versioned under `/v1` and require authentication via a bearer token.

## Authentication

To authenticate, obtain an access token by sending a POST request to the authentication endpoint:

```bash
curl -X POST https://api.surrogate-1.example.com/v1/auth/token \
     -H "Content-Type: application/json" \
     -d '{"username":"<YOUR_USER>","password":"<YOUR_PASS>"}'
```

The response will contain a JSON object with the access token and its expiration time:

```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expires_in": 3600
}
```

Use this token in the `Authorization` header for all subsequent requests:

```bash
Authorization: Bearer YOUR_ACCESS_TOKEN
```

## Endpoints

### 1. Submit Ingestion Job

Submit a new ingestion job with a list of dataset URLs:

```bash
POST /v1/jobs
```

**Request Body**

```json
{
  "dataset_urls": [
    "https://huggingface.co/datasets/axentx/surrogate-1-training-pairs",
    "https://example.com/other-dataset"
  ],
  "options": {
    "dedupe": true,
    "normalize_schema": "v1"
  }
}
```

**Response**

```json
{
  "job_id": "abc123",
  "status": "queued",
  "submitted_at": "2026-05-12T10:00:00Z"
}
```

### 2. Check Job Status

Retrieve the current status of a submitted job:

```bash
GET /v1/jobs/{job_id}
```

**Response**

```json
{
  "job_id": "abc123",
  "status": "processing",
  "progress": 45,
  "started_at": "2026-05-12T10:01:00Z",
  "estimated_completion": "2026-05-12T10:15:00Z"
}
```

### 3. Retrieve Processed Data

Download the processed dataset once the job is complete:

```bash
GET /v1/jobs/{job_id}/results
```

**Response**: Binary ZIP file containing the normalized and deduplicated dataset.

### 4. List All Jobs

Retrieve a paginated list of all jobs for the authenticated user:

```bash
GET /v1/jobs
```

**Query Parameters**

| Parameter | Type | Description |
|-----------|------|-------------|
| `page` | integer | Page number (default 1) |
| `per_page` | integer | Items per page (default 20) |

**Response**

```json
{
  "jobs": [
    {
      "job_id": "abc123",
      "status": "completed",
      "submitted_at": "2026-05-12T10:00:00Z",
      "completed_at": "2026-05-12T10:20:00Z"
    },
    ...
  ],
  "page": 1,
  "per_page": 20,
  "total": 42
}
```

## Error Handling

All error responses follow the standard HTTP status codes and include a JSON body:

```json
{
  "error": {
    "code": "INVALID_REQUEST",
    "message": "The request payload is malformed."
  }
}
```

## Rate Limiting

Clients are limited to **60 requests per minute**. Exceeding this limit will result in a `429 Too Many Requests` response.

## SDKs

Surrogate-1 provides official SDKs for Python and JavaScript. Refer to the [SDK documentation](https://docs.surrogate-1.example.com/sdk) for usage examples.

## Usage Examples

### Submit a Job with cURL

```bash
