# Java 8 Collections: Default Method Overrides for Synchronized/Concurrent Collections

## Overview

Java 8 introduced default methods in interfaces, allowing backward-compatible additions to interfaces. This is particularly valuable for `java.util.concurrent` and `java.util.Collections` classes where synchronized/concurrent collection implementations benefit from shared utility methods.

## Default Method Overrides in Concurrent Collections

### `Collections.synchronizedList()` and `Collections.synchronizedMap()`

These factory methods return synchronized wrappers around collections:

```java
import java.util.*;
import java.util.concurrent.ConcurrentHashMap;

// Traditional synchronized wrapper
List<String> syncList = Collections.synchronizedList(new ArrayList<>());
Map<String, Integer> syncMap = Collections.synchronizedMap(new HashMap<>());

// Thread-safe iteration
synchronized(syncList) {
    for (String item : syncList) {
        // process item
    }
}

// Better: use ConcurrentHashMap for concurrent access
ConcurrentHashMap<String, Integer> concurrentMap = new ConcurrentHashMap<>();
```

### `Collections.unmodifiableCollection()` and `Collections.unmodifiableMap()`

Default methods provide read-only views:

```java
import java.util.*;

List<String> original = new ArrayList<>();
original.add("item1");

// Unmodifiable view (default method override)
List<String> unmodifiable = Collections.unmodifiableList(original);

// Attempting modification throws UnsupportedOperationException
// unmodifiable.add("item2"); // Exception thrown
```

### `Collections.emptyMap()` and `Collections.emptySet()`

Static factory methods with default implementations:

```java
import java.util.*;

// Empty collections (default method overrides)
Map<String, Integer> emptyMap = Collections.emptyMap();
Set<String> emptySet = Collections.emptySet();
List<String> emptyList = Collections.emptyList();

// These are immutable and thread-safe
emptyMap.put("key", "value"); // Throws UnsupportedOperationException
```

## Concurrent Collection Utilities

### `ConcurrentHashMap` Default Methods

```java
import java.util.concurrent.ConcurrentHashMap;

ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();

// Default method: computeIfAbsent (Java 8+)
map.computeIfAbsent("key", k -> 0);

// Default method: merge (Java 8+)
map.merge("key", 1, Integer::sum);

// Default method: putIfAbsent (Java 8+)
map.putIfAbsent("key", 1);
```

### `ConcurrentLinkedQueue` and `ConcurrentLinkedDeque`

```java
import java.util.concurrent.*;

ConcurrentLinkedQueue<String> queue = new ConcurrentLinkedQueue<>();

// Default method: peekFirst / peekLast
String first = queue.peekFirst(); // Returns null if empty
String last = queue.peekLast();

// Default method: pollFirst / pollLast
String item = queue.pollFirst(); // Returns null if empty
```

## Thread-Safe Iteration Patterns

### Using `Collections.synchronizedCollection()` with Iterator

```java
import java.util.*;
import java.util.concurrent.*;

List<String> list = Collections.synchronizedList(new ArrayList<>());

// Safe iteration pattern
Iterator<String> iterator = list.iterator();
while (iterator.hasNext()) {
    String item = iterator.next();
    // process item
}
```

### Using `ConcurrentHashMap` with `compute`

```java
ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();

// Default method: compute
map.compute("key", (k, v) -> {
    if (v == null) return 0;
    return v + 1;
});

// Default method: computeIfPresent
map.computeIfPresent("key", (k, v) -> v + 1);
```

## Best Practices

1. **Prefer `ConcurrentHashMap` over `Collections.synchronizedMap()`** for concurrent access
2. **Use `computeIfAbsent` and `computeIfPresent`** for atomic updates
3. **Use `Collections.unmodifiable*`** for read-only views
4. **Always check for null** when using `peek*` methods on concurrent collections
5. **Consider `CopyOnWriteArrayList`** for read-heavy workloads
