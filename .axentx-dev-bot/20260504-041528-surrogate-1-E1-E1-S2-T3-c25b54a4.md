### 3. **Integration Tests**

**Happy Paths:**
1. Service starts successfully, loads all built-in rules from `config/rules/`, and `/api/v1/rules` returns 200 with full rule list.
2. GET request to `/api/v1/rules` returns only valid schema-compliant rules (filtered if any invalid ones were skipped).
3. Rules from multiple providers (AWS, GCP, Azure) are present and correctly categorized in the response.
4. Custom rule files in the rules directory are also loaded and included in the response.
5. API responds within 500ms under normal load with 15+ rules loaded.

**Edge Cases:**
1. One rule file contains malformed JSON → service starts, logs error, skips invalid file, returns only valid rules.
2. Rules directory is empty → endpoint returns 200 with empty array.
3. Rules directory does not exist → service fails to start or `/api/v1/rules` returns 500 with error message.
