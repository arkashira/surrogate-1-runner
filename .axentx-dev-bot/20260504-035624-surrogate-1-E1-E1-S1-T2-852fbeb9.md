## 1. Acceptance Criteria  
- **AC1**: `POST /v1/synthetic/metrics` returns HTTP **202** and a JSON body containing a non‑empty `job_id`.  
- **AC2**: The request body is validated against the Datadog metric schema:  
  - `name` (string, required) matches regex `^[a-zA-Z0-9_.]+$`.  
  - `type` (string, required) is one of `gauge`, `counter`, `rate`.  
  - `timestamps` (array of integers) and `values` (array of numbers) are present and of equal length.  
  - `tags` (array of strings) are optional but if present each tag must match `^[a-zA-Z0-9_:]+$`.  
- **AC3**: On validation failure the API returns **400** with a JSON body detailing the error(s).  
- **AC4**: Successful payloads are forwarded to the configured Datadog endpoint (or mock URL).  
- **AC5**: `GET /v1/synthetic/jobs/{id}` returns a JSON body with `status` set to one of `queued`, `running`, `completed`, `failed`.  
- **AC6**: Rate limiting of **10 requests per minute per API key** is enforced; exceeding this returns **429**.
