   - Single API call (post-rate-limit window) to `list_repo_tree` for the target date folder; save manifest JSON.

   - Per-file CDN download via `https://huggingface.co/datasets/.../resolve/main/...` (no auth, no API rate limit).
   - Stream-parse each file into `{prompt, response}` only; drop all other columns to avoid mixed-schema `CastError`.

1. **Acceptance criteria**
- Single `list_repo_tree` API call is made per run for the target DATE folder; zero additional recursive/tree calls.
- Only rows whose `hash(slug) % 16 == SHARD_ID` are processed; all others are skipped.
- Files are fetched via CDN URL (`https://huggingface.co/datasets/.../resolve/main/...`) with no Authorization header on the CDN request.
- Each row is stream-parsed to exactly `{prompt, response}`; any row missing either field is dropped; no other columns are retained.
- Per-row md5 is computed over canonical JSON; duplicates are skipped via `dedup.seen/add` and not written to output.
- Output is newline JSON (`.jsonl`) written to `batches/public-merged/{DATE}/shard{N}-{HHMMSS}.jsonl` with valid JSON lines and non-zero size only when rows exist.
- Script exits 0 on success and non-zero on required-env missing or unrecoverable API error; stderr contains actionable messages.
