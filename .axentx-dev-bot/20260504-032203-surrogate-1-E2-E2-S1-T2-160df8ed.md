## Unified Synthetic‑Metric Generator & Validator  

Below is a single, self‑contained solution that merges the strongest ideas from the two proposals:

* **Generator (`generateMetrics`)**
  * Guarantees **≥ 100** metrics per call.  
  * Randomised **name**, **type** (`gauge|count|distribution`) and **value** (numeric or bucketed).  
  * Produces **strictly increasing timestamps** that all lie inside a **60 000 ms (1 min) window**.  
  * Uses an even spread across the window plus a tiny jitter, then fixes any duplicate timestamps.

* **Validator (`validateSequentialTimestamps`)**
  * Works on any array of metric objects.  
  * Throws a clear `Error` if the array is empty, if the total span exceeds one minute, or if any two consecutive timestamps are not strictly increasing.  
  * Sorts the input first, so callers can pass an unsorted list safely.

* **Tests**
  * One test file that (a) checks the shape of the generated objects, (b) asserts the count, (c) runs the validator, and (d) double‑checks the window and monotonicity directly.  
  * Uses **Jest** (the most common modern test runner) but can be run with Mocha/Chai by swapping the `expect` API – the core logic is identical.
