## Unified Headline Generator

The two proposals share the same goal – produce a single‑sentence headline that summarizes the status of a daily digest – but they differ in the **input format** and in the **detail level** of the headline.

* **Candidate 1** works on a list of raw items (each item has a `content` field) and decides whether an item is an incident or a deployment by regex matching.
* **Candidate 2** works on a pre‑aggregated `stats` dictionary that already contains the counts of incidents, deployments, alerts, and a downtime flag.

Both implementations are correct; the only real contradiction is that they accept different inputs.  
The most useful solution is a **single class** that

1. **Accepts either** a list of items *or* a stats dictionary.
2. Uses the regex logic from Candidate 1 to **count** incidents/deployments when only items are supplied.
3. Falls back to the stats dictionary when it is supplied.
4. Builds the headline using the plural‑aware logic from Candidate 2.
5. Keeps the public API tiny – one `generate` method – and is fully testable.
