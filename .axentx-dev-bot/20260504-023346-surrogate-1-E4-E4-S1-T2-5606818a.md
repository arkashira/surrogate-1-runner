## 1. Acceptance Criteria  

- **Trigger Condition**  
  - The `AlertRuleEngine` must invoke the `AlertService` when the daily spend exceeds **120 %** of the allocated budget.  
- **Notification Payload**  
  - The `AlertService` must send **both** an email and an SMS.  
  - Each message must contain:  
    - `model` name  
    - `endpoint` URL  
    - `suggested_actions` string  
- **Recipient Separation**  
  - Email and SMS recipients are supplied as **distinct** strings (e.g., `email_recipient="ops@example.com"`, `sms_recipient="+15551234567"`).  
- **Idempotency**  
  - Re‑triggering the same alert (same model, endpoint, and date) must **not** result in duplicate notifications.  
  - The system must record each alert in an `audit_trail` table with a unique `alert_id`.  
- **Audit Logging**  
  - Every notification sent must be logged with: `alert_id`, `timestamp`, `recipient_type` (`email`/`sms`), and `status` (`sent`/`failed`).
