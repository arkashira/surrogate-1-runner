## 1. What we are building  

A **self‑contained “API‑Signature‑Change” monitoring stack** that  

| Goal | How we achieve it |
|------|-------------------|
| **Detect** any drift in the *shape* of API requests (method, path, query‑keys, header‑keys, request‑body schema) during a deployment **or** in steady‑state. | A lightweight **collector** records a deterministic “signature hash” for every request and pushes the sample to a time‑series store (Prometheus). |
| **Compare** the current signature distribution against a recent baseline (default = last 60 min). | A **detector** runs every 5 min, computes per‑endpoint change‑percent, and evaluates the three rule‑sets defined in the YAML (critical drift, gradual drift, new endpoint). |
| **Alert** the right people with the right severity and context. | An **alert dispatcher** reads the same YAML and fires Slack, PagerDuty or generic webhook actions. |
| **Persist** raw samples for later forensic analysis. | Prometheus retention settings (hot / warm / cold) are defined in the config. |
| **Stay simple** for “quick‑start” use‑cases. | The `APISignatureTracker` from Candidate 1 is kept as a minimal‑dependency fallback that can be used when a full TSDB is not available. |

The final artefacts are:

1. **`api_signature_change.yaml`** – a single source‑of‑truth configuration (merged view of both candidates).  
2. **`collector.py`** – a WSGI/ASGI middleware that extracts the signature and writes a Prometheus metric.  
3. **`detector.py`** – a scheduled job that queries Prometheus, evaluates the rules, and calls the dispatcher.  
4. **`dispatcher.py`** – generic alert‑sending helpers (Slack, PagerDuty, webhook).  
5. **`fallback_tracker.py`** – the simple hash‑based tracker from Candidate 1 for environments without Prometheus.

All files live under **`/opt/axentx/surrogate-1/`** so they can be packaged as a single Python wheel or Docker image.
