### Final Synthesis (single, correct, actionable)

**Diagnosis (resolved)**
- The training pipeline hits Hugging Face API rate limits because it repeatedly lists files or uses `load_dataset(..., streaming=True)` over authenticated `/api/` endpoints instead of the public CDN.
- Lightning Studio instances are not reused; idle timeouts silently stop long runs and quota is wasted on churn.
- No persisted file manifest exists, so workers cannot do zero-API, CDN-only data loading.

**Single proposed change**
Add a small orchestrator that (1) produces a one-time `file-list.json` via a single non-recursive `list_repo_tree` call, (2) reuses or restarts a single Lightning Studio, and (3) launches a worker (`train.py`) that loads data exclusively from CDN URLs. This removes authenticated API calls during training and prevents rate-limit blocks while cutting studio churn.

**Implementation (concrete, copy-ready)**

1) Orchestrator: `/opt/axentx/surrogate-1/train_orchestrator.py`

#!/usr/bin/env bash
set -euo pipefail
export SHELL=/bin/bash

REPO="datasets/axentx/surrogate-1"
DATE="${1:-$(date +%Y-%m-%d)}"
OUT_DIR="batches/mirror-merged/${DATE}"
MANIFEST="file-list-${DATE}.json"

python3 - "$REPO" "$OUT_DIR" "$MANIFEST" <<'PY'
import json, sys
from huggingface_hub import list_repo_tree
from lightning import Teamspace, Studio, Machine

_, REPO, OUT_DIR, MANIFEST = sys.argv

# One non-recursive API call per date folder
entries = list_repo_tree(REPO, path=OUT_DIR, repo_type="dataset", recursive=False)
files = [e.path for e in entries if e.path.endswith(".parquet")]
with open(MANIFEST, "w") as f:
    json.dump(files, f)

# Reuse running studio; restart only if stopped
team = Teamspace()
studio = next((s for s in team.studios if s.name == "surrogate-1-train"), None)

if studio is None:
    studio = Studio(
        name="surrogate-1-train",
        machine=Machine.L40S,
        cloud="lightning-public-prod",
        create_ok=True,
    )

if studio.status != "Running":
    studio.start(machine=Machine.L40S)

# Launch zero-API data-load training
studio.run(
    run_name="train-cdn",
    entry_point="python train.py",
    arguments=["--manifest", MANIFEST, "--repo", REPO, "--date", sys.argv[3]],
    enable_app_logs=True,
)
print("Studio launched:", studio.name, studio.status)
PY
