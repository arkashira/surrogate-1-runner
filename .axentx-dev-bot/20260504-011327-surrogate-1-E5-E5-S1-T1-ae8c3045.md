**Onboarding Analytics Dashboard Proposal**

**Overview**
The Onboarding Analytics Dashboard provides product managers with a quick view of how new users progress through the onboarding sequence. It aggregates key performance indicators (KPIs) to help identify friction points and opportunities for improvement.

**Key Metrics**

| Metric | Description | Calculation |
|--------|-------------|-------------|
| **Completion Rate** | Percentage of users who finish the entire onboarding flow. | `completed_users / total_started_users * 100` |
| **User Engagement** | Average number of onboarding steps interacted with per user. | `total_step_interactions / total_started_users` |
| **Feedback Score** | Average rating collected from post‑onboarding surveys (scale 1‑5). | `sum(feedback_ratings) / count(feedback_responses)` |

**Data Sources**

- **User Events**: Recorded in the `onboarding_events` table (step start, step complete, etc.).
- **Survey Responses**: Stored in the `onboarding_feedback` collection.
- **User Profiles**: Used to filter by cohort (e.g., region, acquisition channel).

**Dashboard Layout**

1. **Header** – Dashboard title and date range selector.
2. **Metric Cards** – Large cards showing the three KPIs with trend arrows.
3. **Completion Funnel** – Visual funnel chart breaking down drop‑off at each step.
4. **Engagement Heatmap** – Heatmap of step interactions over time.
5. **Feedback Distribution** – Bar chart of rating frequencies.

**Access Control**

- Only users with the `product_manager` role can view the dashboard.
- Authentication is handled via the existing SSO integration; the dashboard checks the role claim before rendering.

**Implementation Notes**

- The dashboard will be built with **React** and **Chart.js**.
- Backend endpoints will be exposed under `/api/analytics/onboarding/*`.
- Queries will be optimized with materialized views refreshed every hour.
- The dashboard will be placed in the `/docs` directory as a markdown file named `analytics_dashboard.md`.

**Tech Stack**

- Frontend: React
- Backend: Existing API infrastructure (e.g., Node.js, Express.js)
- Data Storage: Existing database (e.g., MongoDB, PostgreSQL)
- Charting Library: Chart.js

**Future Enhancements**

- Cohort comparison (e.g., A/B test groups).
- Real-time updates via WebSocket for live monitoring.
- Export to CSV / PDF.

**Project Structure**

- `docs/analytics_dashboard.md`: Onboarding Analytics Dashboard documentation.
- `src/components/OnboardingAnalyticsDashboard`: React component for the dashboard.
- `src/api/analytics/onboarding`: Backend endpoints for onboarding analytics data.

**Existing Backend APIs and Data Sources**

- `onboarding_events` table: User events (step start, step complete, etc.).
- `onboarding_feedback` collection: Survey responses.
- `user_profiles` collection: User profiles for filtering by cohort.
