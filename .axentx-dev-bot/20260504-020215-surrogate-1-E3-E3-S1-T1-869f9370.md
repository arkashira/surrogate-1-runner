## 1.  Design Overview

| Layer | Responsibility | Why it matters |
|-------|----------------|----------------|
| **Configuration** | Holds thresholds, notifier URLs, SMTP creds, etc. | Keeps secrets out of code, allows per‑environment overrides. |
| **Metrics** | Snapshot of a single run (speed, accuracy, sample count, timestamp). | Enables a clean, type‑safe comparison. |
| **AlertPayload** | Normalised data that every notifier consumes. | One source of truth for the message. |
| **Notifier Interface** | `send(payload: AlertPayload) -> bool` | Allows any channel (Slack, Email, PagerDuty, …) to plug in. |
| **SlackNotifier** | Sends a JSON payload to a webhook. | Uses `requests` for simplicity, retries, and a clear failure path. |
| **EmailNotifier** | Sends a MIME‑encoded email. | Uses `email.message.EmailMessage` (Python 3.6+) for readability. |
| **AlertEngine** | 1️⃣ Pulls baseline & current metrics.<br>2️⃣ Computes diff.<br>3️⃣ Decides if a regression exists.<br>4️⃣ Broadcasts via all configured notifiers.<br>5️⃣ Logs everything. | Centralised logic → no duplication, easier testing. |

### Contradiction resolution

| Issue | Candidate 1 | Fix |
|-------|-------------|-----|
| **Diff direction** | `diff_percent = ((current - baseline) / baseline) * 100` – no sign handling. | Treat *any* negative diff as a regression.  `is_regression = diff_percent < -threshold`. |
| **Slack color** | `danger` if `abs(diff) > threshold`. | `danger` only when `is_regression`; otherwise `warning`. |
| **Threshold per metric** | Hard‑coded `speed` vs `accuracy`. | Use a mapping (`{"speed": speed_threshold, "accuracy": accuracy_threshold}`) so new metrics can be added. |
| **Missing baseline** | No guard. | Return `None` and log a warning. |
| **Email body** | Plain string with manual formatting. | Use `EmailMessage` and `textwrap.dedent` for clean multiline strings. |
| **Error handling** | `except Exception` in email, `except URLError` in Slack. | Wrap each notifier in a retry loop (3×) and log all failures. |
| **Configuration** | Hard‑coded defaults. | Load from environment variables or a YAML/JSON file with `pydantic` validation. |

# alert_engine/config.py
from __future__ import annotations

import os
from pathlib import Path
from typing import Dict, List, Optional

from pydantic import BaseModel, Field, validator


class ThresholdConfig(BaseModel):
    speed_threshold_percent: float = Field(10.0, gt=0)
    accuracy_threshold_percent: float = Field(5.0, gt=0)
    min_samples: int = Field(100, gt=0)


class SlackConfig(BaseModel):
    webhook_url: Optional[str] = None


class SmtpConfig(BaseModel):
    host: str = Field("smtp.gmail.com")
    port: int = Field(587, ge=1, le=65535)
    username: str
    password: str
    from_addr: str
    to_addrs: List[str] = Field(default_factory=list)


class AlertEngineConfig(BaseModel):
    thresholds: ThresholdConfig = ThresholdConfig()
    slack: SlackConfig = SlackConfig()
    smtp: Optional[SmtpConfig] = None

    @validator("slack", always=True)
    def validate_slack(cls, v):
        if not v.webhook_url:
            v.webhook_url = os.getenv("SLACK_WEBHOOK_URL")
        return v

    @validator("smtp", always=True)
    def validate_smtp(cls, v):
        if v is None:
            # Try to load from env
            if os.getenv("SMTP_HOST"):
                v = SmtpConfig(
                    host=os.getenv("SMTP_HOST"),
                    port=int(os.getenv("SMTP_PORT", 587)),
                    username=os.getenv("SMTP_USER"),
                    password=os.getenv("SMTP_PASS"),
                    from_addr=os.getenv("SMTP_FROM"),
                    to_addrs=os.getenv("SMTP_TO", "").split(","),
                )
        return v
