## 1. Acceptance Criteria  

- **AC1 – Data ingestion**: `PeerMatcher` accepts a `pandas.DataFrame` or a CSV path, validates that the required columns (`org_id`, `industry`, `stack_size`, `optimization_patterns`) exist, and normalizes the input to a DataFrame preserving column order.  
- **AC2 – Encoding pipeline**: On `fit()`, a `ColumnTransformer` is built that one‑hot encodes `industry` and `stack_size` and leaves `optimization_patterns` untouched (list of strings). The transformer must be stored for reuse.  
- **AC3 – Similarity matrix**: After fitting, an internal matrix of shape `(n_orgs, n_features)` is created and stored. Cosine similarity between any two rows can be queried in O(1) time.  
- **AC4 – Peer retrieval**: `find_peers(target_id, top_n=5)` returns a list of `org_id`s sorted by descending cosine similarity to the target, excluding the target itself.  
- **AC5 – Filtered search**:  
  - `find_peers_by_industry(target_id, industry, top_n=5)` returns peers that share the same `industry`.  
  - `find_peers_by_stack_size(target_id, stack_size, top_n=5)` returns peers that share the same `stack_size`.  
  Both methods must respect `top_n`.  
- **AC6 – Pattern aggregation**: `top_patterns(peers: List[str], top_n=5)` counts occurrences of `optimization_patterns` across the supplied peer list and returns the top `top_n` patterns with their counts.  
- **AC7 – CLI demo**: The `__main__` block creates synthetic data, fits the matcher, and prints:  
  1. Top 5 peers for a sample `org_id`.  
  2. Top 5 peers filtered by industry.  
  3. Top 5 patterns among those peers.
