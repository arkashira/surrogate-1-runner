## Final Synthesis (Best of Both Proposals)

**Goal**: Eliminate HF API rate-limit failures and HF Space OOM within ≤2 hours by replacing recursive `list_repo_files` and per-file API calls with **one per-folder `list_repo_tree` + CDN-only fetches**, projecting to `{prompt,response}` at parse time.

**Why this is highest value**:
- Removes recursive `list_repo_files` (100-pagination, rate-limited) → single `list_repo_tree(path, recursive=False)` per folder.
- Uses CDN URLs (`/resolve/main/...`) which bypass `/api/` auth checks and rate limits.
- Projects to `{prompt,response}` only at parse time (avoids pyarrow CastError on mixed schemas).
- Keeps 16-shard parallelism intact; each shard still gets its own runner with 7 GB RAM.
