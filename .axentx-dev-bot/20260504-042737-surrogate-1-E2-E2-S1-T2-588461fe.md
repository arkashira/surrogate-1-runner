## 1. Acceptance Criteria
- **AC1**: A GET request to `/api/vocab/daily` returns **exactly 10** term objects.  
- **AC2**: Each term object contains the fields `term`, `definition`, `example`, and `category`.  
- **AC3**: The `category` field value must be one of the predefined categories (`Cloud`, `DevOps`, `Security`, `Networking`, `Programming`).  
- **AC4**: A POST request to `/api/vocab/learned` with a JSON body `{ "term": "<term>" }` marks the term as learned for the authenticated user.  
- **AC5**: After marking a term as learned, a subsequent GET to `/api/vocab/daily` **does not** include that term.  
- **AC6**: Learned terms are persisted in the user profile; a GET to `/api/user/profile` includes a list `learned_terms`.  
- **AC7**: All endpoints return appropriate HTTP status codes (`200 OK`, `201 Created`, `400 Bad Request`, `401 Unauthorized`, `404 Not Found`).
