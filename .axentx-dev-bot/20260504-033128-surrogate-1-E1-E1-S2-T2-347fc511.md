## 1. Acceptance Criteria
- **Batch Timing**: Events are aggregated and sent **every 5 seconds** or when the batch size reaches **50 events**, whichever occurs first.
- **Payload Structure**: The HTTP POST body is JSON containing:
  - `events`: array of event objects
  - `device_id`: string
  - `app_version`: string
- **Endpoint**: POST to `/api/v1/wholphin/events`.
- **Response Handling**: The worker must treat an HTTP **200 OK** as success and log the event batch; non‑200 responses trigger a retry mechanism.
- **Logging**: Each successful batch transmission is persisted to a log file with timestamp, batch size, and response status.
- **Error Logging**: Failed transmissions are logged with error details and a retry counter.
- **Configuration**: Batch size, interval, and endpoint URL are configurable via `AppState` and environment variables.
- **Performance**: Sending a batch of 50 events should complete within **200 ms** under normal load.
