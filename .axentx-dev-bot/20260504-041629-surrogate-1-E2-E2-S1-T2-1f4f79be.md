# Synthesized Workflow Documentation: Dataset Ingestion, Validation, and Model Training

## Overview

This document consolidates the strongest elements from multiple proposals into a unified, actionable workflow for the surrogate-1 project. It addresses dataset ingestion, validation, quality assurance, and model training as interconnected pipeline components.

## Task 1: Dataset Ingestion Workflow

### Purpose
Ingest public datasets from the HuggingFace Hub, normalize them according to schema, deduplicate records, and upload processed shards to the `axentx/surrogate-1-training-pairs` repository.

### Trigger
- **Scheduled**: Every 30 minutes via GitHub Actions `cron`
- **Manual**: `workflow_dispatch` from the Actions UI

### Workflow Steps

#### 1. Initialize Parallel Runners
- Spin up **16 parallel runners** for concurrent processing
- Each runner receives a unique `SHARD_ID` (0-15) derived deterministically from the repository slug

#### 2. Compute Shard Assignment
```bash
# bin/dataset-enrich.sh
SLICE=$(python -c "import hashlib, sys; print(int(hashlib.sha1('$GITHUB_REPOSITORY'.encode()).hexdigest(),16) % 16)")
export SHARD_ID=$SLICE
```
- **Why SHA-1**: Provides deterministic, non-overlapping distribution across runners
- Each runner processes a consistent slice across workflow runs

#### 3. Fetch and Filter Dataset List
- Read the master list of public dataset URLs
- Filter entries belonging to the current `SHARD_ID` (bucket = `hash % 16`)

#### 4. Stream and Normalize Data
```bash
curl -sL "$URL" | python -m surrogate1.normalize --schema schema.yaml
```
- **Schema Enforcement**: Strict normalization per project-wide JSON schema
- **Streaming**: Process data as it arrives to minimize memory footprint

#### 5. Deduplicate Records
- Compute **MD5 hash** of each normalized record
- Check against central hash store (`/opt/axentx/surrogate-1/hash_store/`)
- Skip records with existing hashes to ensure uniqueness

#### 6. Upload Processed Shard
```bash
huggingface-cli upload axentx/surrogate-1-training-pairs shard-${SHARD_ID}.jsonl --path data/shard-${SHARD_ID}
```
- Write processed records to `shard-${SHARD_ID}.jsonl`
- Push to HuggingFace dataset repository with version tracking

#### 7. Cleanup and Logging
- Remove temporary files
- Log success/failure metrics to `logs/ingest-${SHARD_ID}.log`

### Best Practices (from Candidate 2)
- **Automated Retry**: Implement automatic retry on transient failures (network timeouts, rate limits)
- **Versioning**: Automatic versioning based on commit timestamps
- **Runtime Metrics**: Collect metrics every 5 minutes for pipeline health monitoring

### Monitoring & Alerts
- **GitHub Actions UI**: Per-runner status visibility
- **CloudWatch/Equivalent**: Aggregated logs for error detection
- **Slack Integration**: `actions-slack` notifies `#data-ops` on runner failure
