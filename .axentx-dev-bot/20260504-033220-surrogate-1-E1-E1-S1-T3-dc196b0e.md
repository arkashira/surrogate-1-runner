## 1. `MediaCache`

```java
package com.axentx.surrogate.media;

import java.util.LinkedHashMap;
import java.util.Map;

/**
 * Thread‑safe LRU cache for media chunks.
 */
public class MediaCache {

    private final int maxSize;
    private final Map<String, byte[]> cache;

    public MediaCache(int maxSize) {
        if (maxSize <= 0) {
            throw new IllegalArgumentException("maxSize must be > 0");
        }
        this.maxSize = maxSize;
        this.cache = new LinkedHashMap<>(maxSize, 0.75f, true) {
            @Override
            protected boolean removeEldestEntry(Map.Entry<String, byte[]> eldest) {
                return size() > MediaCache.this.maxSize;
            }
        };
    }

    public synchronized void put(String key, byte[] value) {
        cache.put(key, value);
    }

    public synchronized byte[] get(String key) {
        return cache.get(key);
    }

    public synchronized boolean contains(String key) {
        return cache.containsKey(key);
    }

    public synchronized int size() {
        return cache.size();
```
    }
}

## 1. Acceptance Criteria
1. **Cache eviction** – When `MediaCache.size()` exceeds the configured `maxSize`, the eldest entry (by insertion order) is automatically removed.
2. **Cache hit** – A second call to `MediaStreamer.loadMediaChunk(mediaUrl, offset, chunkSize)` returns the exact same `byte[]` instance (or an equal copy) that was cached on the first call for the identical `mediaUrl` and `offset`.
3. **Range header correctness** – `MediaStreamer` sends an HTTP `Range: bytes=start-end` header where `start = offset` and `end = offset + chunkSize - 1` (or the file length‑1 for the final chunk). The returned byte array length is ≤ `chunkSize`.
4. **End‑of‑stream handling** – When `offset` is beyond the media length, `loadMediaChunk` returns an empty byte array and does **not** throw an exception.
5. **Player state machine** – `MediaPlayer.play()` sets `isPlaying` to `true`; `pause()` sets it to `false` without terminating the executor; `stop()` sets `isPlaying` to `false` and shuts down the internal `ExecutorService` cleanly.
6. **Thread‑safety** – Concurrent calls to `MediaCache.put/get` from multiple threads never corrupt internal state (no lost entries, no `ConcurrentModificationException`).
7. **Performance baseline** – Loading a media chunk that is cached must be at least 2× faster than loading the same chunk from the network (measured with a mock HTTP server).
