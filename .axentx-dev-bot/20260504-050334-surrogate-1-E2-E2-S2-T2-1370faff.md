# /opt/axentx/surrogate-1/docs/metrics.md
## Marketing Effectiveness Tracking – Unified Reference

This document consolidates the most actionable insights from the two candidate drafts.  
It covers:

* Core concepts & terminology  
* Data flow & architecture  
* User‑centric interaction flow  
* REST API surface  
* Practical usage examples  
* Immediate next‑steps for implementation  

---

### 1. Core Concepts

| Term | Definition |
|------|------------|
| **Marketing Advice** | AI‑generated recommendations (content topics, posting times, ad creatives). |
| **Effectiveness Metric** | Quantitative score that reflects how well a piece of advice performed (e.g., CTR, conversion rate, ROAS, engagement). |
| **Strategy Adjustment** | User‑initiated change to tactics based on metric insights (e.g., “increase posting frequency”). |
| **Feedback Loop** | Explicit user rating of advice usefulness that feeds back into the ranking engine. |

---

### 2. Data Flow & Architecture

```
+----------------+      +----------------+      +----------------+      +----------------+
|  Advice Engine | ---> |  Advice Table  | ---> |  Performance   | ---> |  Metrics Table |
| (AI model)     |      |  (advice_id)   |      |  (raw data)    |      |  (aggregated)  |
+----------------+      +----------------+      +----------------+      +----------------+
                                            ^                          |
                                            |                          |
                                            +--------------------------+
                                            |  External Analytics APIs |
                                            +--------------------------+
```

1. **Advice Generation** – AI model writes advice → stored in `advice` table.  
2. **Performance Capture** – External tools (GA, Facebook, etc.) push raw events to `performance` table.  
3. **Metric Calculation** – Scheduled job (e.g., nightly cron) aggregates raw data → `metrics` table.  
4. **User Review** – Dashboard displays advice + metrics.  
5. **Feedback Submission** – User rates advice → `feedback` table → triggers re‑ranking.  

---

### 3. User Interaction Flow

| Step | Action | UI Element | Backend |
|------|--------|------------|---------|
| 1 | View advice & metrics | Dashboard list | `GET /api/metrics` |
| 2 | Adjust strategy | Toggle/checkbox | `POST /api/strategy` |
| 3 | Rate advice | Rating widget | `POST /api/feedback` |
| 4 | System re‑evaluates | Background job | Re‑rank advice |

**Key UX Principles**

* **One‑click adjustments** – Strategy toggles should persist instantly.  
* **Contextual metrics** – Show metric trend (sparkline) next to each advice item.  
* **Transparent feedback** – Display how user ratings influence future recommendations.

---

### 4. API Surface

| Endpoint | Method | Payload | Description |
|----------|--------|---------|-------------|
| `/api/metrics` | GET | `?advice_id=…` | Retrieve current effectiveness metrics for one or many advice items. |
| `/api/strategy` | POST | `{ "advice_id": 42, "strategy": "increase_frequency" }` | Persist a user‑chosen strategy adjustment. |
| `/api/feedback` | POST | `{ "advice_id": 42, "feedback": "very_effective" }` | Record user rating; triggers re‑ranking. |

**Auth** – All endpoints require a bearer token (`Authorization: Bearer <token>`).  
**Rate‑limit** – 100 req/min per user to protect the ranking engine.

---

### 5. Example Usage

```bash
# 1. Get metrics for advice 42
curl -H "Authorization: Bearer <token>" \
     "https://api.axentx.com/metrics?advice_id=42"

# 2. Submit a strategy change
curl -X POST -H "Content-Type: application/json" \
     -d '{"advice_id":42,"strategy":"increase_frequency"}' \
     https://api.axentx.com/strategy

# 3. Submit feedback
curl -X POST -H "Content-Type: application/json" \
     -d '{"advice_id":42,"feedback":"very_effective"}' \
     https://api.axentx.com/feedback
```
