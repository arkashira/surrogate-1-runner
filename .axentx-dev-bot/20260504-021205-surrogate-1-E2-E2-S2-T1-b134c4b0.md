## Unified, production‑ready SAML‑based Web‑Shell

2. **Verifies the XML signature** against a trusted IdP certificate  
3. **Normalises the assertion into a plain JS object** (`req.saml`)  
4. **Authorises the user** by checking membership in a configurable list of groups  
5. **Stores the assertion in the session** for subsequent requests  
6. **Logs all critical events** (parsing, validation, auth decisions)  
7. **Handles errors gracefully** – no stack traces leaked to the client

* **Signature validation** – `xml-crypto` + `xmldom` (Candidate 1)  
* **Attribute extraction** – `xml2js` (Candidate 1)  
* **Middleware pattern** – lightweight, no network calls (Candidate 2)  
* **Configurable allowed groups** – `config.js` (both)  
* **Session‑backed state** – `express-session` (both)  
* **Logging** – `pino` (added for production readiness)
