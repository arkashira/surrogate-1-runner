## Unified `cron.yaml` – the “one‑stop‑shop” for the AWS Cost Explorer pipeline  

Below is the **single, production‑ready** scheduler definition that merges the strongest bits of both proposals:

| Feature | Why it’s kept | How it’s expressed |
|---------|---------------|--------------------|
| **Two jobs** – a data‑pull and a health‑check | Guarantees we both fetch the data **and** verify it arrived | `aws-cost-daily-pull` + `aws-cost-health-check` |
| **Run after 02:00 UTC** | Gives the Cost Explorer service a few minutes to settle after the daily roll‑up | Pull at **02:30 UTC**; health‑check at **06:00 UTC** |
| **POST request** (not GET) | Most internal “trigger” endpoints expect a POST payload (even if empty) and it matches the first proposal’s explicit `method: POST` | `method: POST` |
| **Absolute URL with env var** | Works even if the scheduler runs on a different host; keeps the config portable | `url: "https://{{ .Env.APP_HOST }}/api/v1/cost/aws/pull"` |
| **Retry + back‑off** | Guarantees resilience against transient network/API hiccups | `max_attempts: 3`, `backoff_seconds: 60` |
| **Slack + log alerting** | Gives immediate human visibility **and** a persistent audit trail | Two alert blocks under `on_failure` |
| **Storage metadata** | Makes downstream consumers (e.g., BI pipelines) know where raw JSON lands | `table: "cost_raw"` |
| **Health‑check that depends on the pull** | Prevents false‑positives – the check only runs if the pull job exists | `depends_on: "aws-cost-daily-pull"` |
| **Clear description & enable flags** | Improves readability for ops & future maintainers | `description` & `enabled: true` |

### Complete `cron.yaml`

```yaml
# /opt/axentx/surrogate-1/scheduler/cron.yaml
---
version: "1.0"

schedules:
  # ----------------------------------------------------------------------
  # 1️⃣ Daily AWS Cost Explorer pull – runs after the 02:00 UTC roll‑up
  # ----------------------------------------------------------------------
  - name: aws-cost-daily-pull
    description: "Daily AWS Cost Explorer data pull – runs at 02:30 UTC"
    cron: "30 2 * * *"                     # 02:30 UTC every day
    timezone: "UTC"
    enabled: true

    job:
      type: http
      method: POST
      # The host is injected from the runtime environment (e.g. Docker/K8s secret)
      url: "https://{{ .Env.APP_HOST }}/api/v1/cost/aws/pull"
      timeout: 300                         # 5 minutes – generous for large accounts
      headers:
        - key: "Content-Type"
          value: "application/json"

    retry:
      enabled: true
      max_attempts: 3
      backoff_seconds: 60                  # 1 minute exponential back‑off

    alerting:
      on_failure:
        - type: slack
          channel: "{{ .Env.SLACK_COST_ALERT_CHANNEL }}"
          message: |
            :warning: *AWS Cost Explorer pull failed*  
            Schedule: {{ .Schedule.name }}  
            Error: {{ .Error }}
        - type: log
          level: "error"
          message: "AWS cost pull failed – {{ .Error }}"

    storage:
      table: "cost_raw"
      format: "json"

  # ----------------------------------------------------------------------
  # 2️⃣ Health‑check – verifies that the pull succeeded within the last 4 h
  # ----------------------------------------------------------------------
  - name: aws-cost-health-check
    description: "Verify that a successful cost pull arrived in the last 4 h"
    cron: "0 6 * * *"                      # 06:00 UTC every day
    timezone: "UTC"
    enabled: true

    job:
      type: health-check
      depends_on: "aws-cost-daily-pull"
      max_age_minutes: 240                 # Fail if last successful run > 4 h ago

    alerting:
      on_failure:
        - type: slack
          channel: "{{ .Env.SLACK_COST_ALERT_CHANNEL }}"
          message: |
            :red_circle: *AWS Cost data missing*  
            Health‑check for `aws-cost-daily-pull` failed.  
            No successful run in the past 4 h.
