## 1. Acceptance Criteria
1. **Integration** – The `JavaFreeRouter` class is discoverable by Surrogate‑1 at runtime and can be instantiated via the existing router factory without requiring any Java runtime dependencies.  
2. **Performance** – Routing a benchmark PCB (≈ 100 nets, ≈ 500 components) completes in ≤ 50 % of the time taken by the legacy FreeRouting implementation on identical hardware.  
3. **KiCAD 9 Compatibility** – The router correctly parses KiCAD 9 `.kicad_pcb` files, respects layer stack‑up definitions, and produces a routed PCB that KiCAD 9 can open without errors.  
4. **Result Fidelity** – The generated netlist and layout match the input design constraints (no DRC violations, all nets connected, clearance rules obeyed).  
5. **Zero Java Dependency** – The compiled Surrogate‑1 binary does not embed any Java runtime libraries; `ldd` (or equivalent) shows no `libjvm` linkage.  
6. **CLI/GUI Consistency** – The router can be invoked both from the command‑line interface and the GUI with identical outcomes.  
7. **Rollback** – If the router fails (e.g., throws an exception), Surrogate‑1 falls back to the legacy FreeRouting engine without crashing.
