### Final Implementation Plan (≤2h)

Replace `bin/dataset-enrich.sh` with a **manifest-driven, CDN-bypass ingestion worker** (`bin/dataset-enrich.py`) that:

- Accepts `SHARD_ID`, `SHARD_TOTAL=16`, `DATE`, `HF_TOKEN`
- Uses **one** `list_repo_tree` call (per date folder) to enumerate candidate files, then deterministically assigns each file to a shard via `hash(slug) % SHARD_TOTAL`
- Downloads assigned files via **HF CDN bypass** (`https://huggingface.co/datasets/.../resolve/main/...`) with no Authorization header (avoids 429 on `/api/`)
- Projects heterogeneous schemas to `{prompt, response}` only at parse time (avoids PyArrow CastError)
- Deduplicates via central `lib/dedup.py` md5 store (same as Space)
- Writes normalized pairs to `batches/public-merged/{DATE}/shard{N}-{HHMMSS}.jsonl`
- Returns exit code 0 on success; logs counts and skipped duplicates

---

### Why this is the highest-value incremental improvement
- Fixes the known `load_dataset(streaming=True)` mixed-schema CastError pattern
- Eliminates HF API rate-limit risk during data load by using CDN-only fetches
- Keeps parallel 16-shard architecture intact while making each worker robust and observable
- Reuses existing dedup logic and filename convention — no downstream changes required
