## surrogate‑1 – Unified Documentation

> **What is surrogate‑1?**  
> A lightweight, open‑source toolkit that lets you  
> * ingest public datasets in parallel,  
> * run a quick benchmark on a pre‑trained model,  
> * extract embeddings, and  
> * apply a simple (or custom) watermark.  

> **Why use it?**  
> * **Fast, reproducible** – all you need is a fresh Python 3.8+ environment.  
> * **Parallel ingestion** – 16 workers automatically split the HuggingFace dataset.  
> * **Zero‑config notebook** – a ready‑to‑run Jupyter file demonstrates the full workflow.  
> * **Extensible** – plug‑in your own watermarking logic or downstream pipeline.

The command automatically:

1. Loads `axentx/surrogate-1-base` from HuggingFace.

3. Runs a forward pass on 10 samples.  
4. Prints the resulting CLS‑like embedding shape.

## 4. Parallel Dataset Ingestion

The **surrogate‑1-runner** can ingest the public dataset in 16 parallel shards.

```bash
# 1️⃣  Enrich the dataset list (creates `bin/dataset-enrich.sh` output)
./bin/dataset-enrich.sh

# 2️⃣  Launch the 16 workers (GitHub Actions does this automatically)
python -m surrogate_runner.main
```

Each worker:

* Reads its deterministic 1/16 slice (`SHARD_ID`).  
* Normalises per‑schema, deduplicates via a central MD5 store, and uploads to the dataset repo.  
* Runs every 30 minutes or on demand (`workflow_dispatch`).

> **Why 16 shards?**  
> Empirically balances throughput and memory usage on a standard CI runner.

> Replace the constant vector with any deterministic function (e.g., a hash‑based perturbation) and call `surrogate_1.apply_watermark(embeddings, key)`.
