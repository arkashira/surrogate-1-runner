| **Typed return objects** – DAO methods return **domain objects** (`SignatureDTO`) with the **decrypted** hash already populated. | Callers never see encrypted blobs, reducing the chance of misuse. | `SignatureDTO` dataclass. |
| **Comprehensive error handling & logging** – all DB interactions are wrapped in a context manager that logs failures with stack traces. | Simplifies debugging in production. | `_get_session()` context manager. |
| **Unit‑testable design** – DAO receives a `session_factory` argument, enabling in‑memory SQLite tests. | Guarantees behaviour before shipping. | `tests/test_signature_dao.py`. |
| **Configuration via environment variables** – `DATABASE_URL`, `SIGNATURE_KEY_ID`, `RETENTION_DAYS`. | Keeps secrets out of code and works with 12‑factor apps. | `settings.py`. |
