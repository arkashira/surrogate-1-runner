**Synthesized Answer: TokenGuard Documentation**

TokenGuard is a security tool designed to validate tokens in CI/CD pipelines and development workflows, ensuring that all tokens, secrets, and credentials are properly resolved before deployment.

### Installation

TokenGuard can be installed via:

* **Binary Release**: Download the latest release for your platform from the [releases page](https://github.com/axentx/surrogate-1/releases).
* **Homebrew (macOS/Linux)**: `brew install axentx/tokenguard/tokenguard`
* **Build from Source**: `git clone https://github.com/axentx/surrogate-1.git` and `go install ./cmd/tokenguard`

### CLI Usage

TokenGuard provides a simple CLI for validating tokens:

* **Basic Validation**: `tokenguard validate --file .env` or `tokenguard validate --dir ./config/secrets`
* **Check Specific Token**: `tokenguard check "your-token-here"`
* **Output Formats**: `tokenguard validate --file .env --output json` or `tokenguard validate --file .env --output sarif --output-file results.sarif`

### GitHub Actions Integration

Add TokenGuard to your CI/CD pipeline to automatically validate tokens on every push and pull request:

```yaml
name: TokenGuard Validation

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  validate-tokens:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Run TokenGuard
        uses: axentx/tokenguard-action@v1
        with:
          files: '.env,config/secrets.yaml'
          output-format: sarif
          token: ${{ secrets.TOKENGUARD_TOKEN }}

      - name: Upload results to GitHub Security tab
        uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: tokenguard-results.sarif
```

### Configuration

TokenGuard can be configured via a `tokenguard.yaml` file in your project root:

```yaml
# tokenguard.yaml
validation:
  strict: false
  timeout: 30s
  exclude:
    - "*.test.yaml"
    - "**/node_modules/**"

tokens:
  required_env_vars:
    - DATABASE_URL
    - API_SECRET
  allowed_patterns:
    - "^sk-[a-zA-Z0-9]+$"
    - "^ghp_[a-zA-Z0-9]+$"

output:
  format: text
  file: tokenguard-results.json
```

### Troubleshooting

Common issues and solutions:

* **"Token reference not resolved"**: Ensure the environment variable is set or replace it with the actual value.
* **"Invalid token format"**: Check your configuration or update the token to match expected formats.
* **"Timeout exceeded"**: Increase the timeout with `--timeout` flag or check for large files.
