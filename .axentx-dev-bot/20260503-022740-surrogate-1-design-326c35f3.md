## Final Implementation Plan (≤2h)

Replace `bin/dataset-enrich.sh` with a **manifest-driven, CDN-bypass ingestion worker** (`bin/dataset-enrich.py`) that:

- Accepts `SHARD_ID`, `SHARD_TOTAL=16`, `DATE`, `HF_TOKEN`, `REPO_ID` (default: `axentx/surrogate-1-training-pairs`)
- Single API call from runner (after rate-limit window) → `list_repo_tree(path=f"batches/public-merged/{DATE}", recursive=False)` → save `file-list.json`
- Deterministic shard assignment: `hash(slug) % SHARD_TOTAL == SHARD_ID`
- Per-file CDN download only: `https://huggingface.co/datasets/{REPO_ID}/resolve/main/{path}` (no Authorization header → bypasses API rate limit)
- Stream-parse each file (parquet/jsonl) → project to `{prompt, response}` only at parse time (avoid mixed-schema `pyarrow.CastError`)
- Central dedup via `lib/dedup.py` (md5 store) — same interface used by HF Space
- Upload shard output to `batches/public-merged/{DATE}/shard{SHARD_ID}-{HHMMSS}.jsonl`
- Retry/backoff on 429: wait 360s; on HF commit cap: deterministic sibling repo fallback (hash slug → pick repo)
- Keep Bash wrapper for cron compatibility: `#!/usr/bin/env bash`, `chmod +x`, invoke via `bash bin/dataset-enrich.py "$@"`, set `SHELL=/bin/bash` in crontab
