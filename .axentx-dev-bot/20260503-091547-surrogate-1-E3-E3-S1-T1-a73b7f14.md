### Diagnosis

The current implementation lacks a concrete plan for implementing a notification job queue and worker, rate-limited notifications, notification content, and configured channels. To address these blockers, we propose the following changes:

### Proposed Change

1.  **Notification job queue and worker**: We will use Redis as the job queue and implement the worker using Node.js. The worker will consume notification jobs from the queue and process them accordingly.
2.  **Rate-limited notifications**: We will use Redis's built-in features to implement rate limiting, ensuring that only one notification is sent per request per 5 minutes.
3.  **Notification content**: We will include the request title, new status, public URL, and actor in the notification content.
4.  **Configured channels**: We will store the configured channels in the database and retrieve them when sending notifications.

### Implementation

#### notificationQueue.ts

```typescript
import { createClient } from 'redis';
import { config } from 'dotenv';
import { RequestStatus } from '../models/requestStatus';

config();

const redisClient = createClient({
  host: process.env.REDIS_HOST,
  port: process.env.REDIS_PORT,
});

const notificationQueue = 'notificationQueue';
const rateLimitKey = 'rateLimit';

const notificationContent = (request: any, newStatus: RequestStatus, actor: string) => {
  return {
    title: request.title,
    newStatus: newStatus,
    publicUrl: request.publicUrl,
    actor: actor,
  };
};

async function sendNotification(request: any, newStatus: RequestStatus, actor: string) {
  const channels = await getChannels();
  const content = notificationContent(request, newStatus, actor);
  redisClient.lpush(notificationQueue, JSON.stringify(content));
  redisClient.set(rateLimitKey, new Date().getTime() / 1000 + 300);
  return true;
}

async function getChannels() {
  const channels = await db.query('SELECT * FROM channels');
  return channels;
}

async function enqueueNotificationJob(request: any, newStatus: RequestStatus, actor: string) {
  const rateLimit = await redisClient.get(rateLimitKey);
  if (rateLimit && rateLimit < new Date().getTime() / 1000) {
    return false;
  }
  await sendNotification(request, newStatus, actor);
  return true;
}

export { enqueueNotificationJob };
```
