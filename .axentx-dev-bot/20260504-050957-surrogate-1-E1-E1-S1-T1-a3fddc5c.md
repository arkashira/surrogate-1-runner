## 1. Acceptance Criteria
- **AC1 – Display Total P&L**: The dashboard shows a single numeric value representing the net profit/loss for the last 30 days, formatted as currency (e.g., “$12,345.67”).  
- **AC2 – Color Coding**: The displayed value is rendered **green** when the net amount is ≥ 0 (profit) and **red** when < 0 (loss).  
- **AC3 – 30‑Day Window**: The P&L is computed using all financial transactions whose `transactionDate` falls within the most recent 30‑day period relative to the current system time.  
- **AC4 – Daily Refresh**: The cached P&L value is automatically recomputed once every 24 hours (±5 min) even if no user interaction occurs.  
- **AC5 – Detail Link**: A clickable link labeled “View detailed P&L” is present next to the summary and navigates to `/reports/pl-detail`.  
- **AC6 – Real‑time Update on Import**: When a new transaction is imported (via the import service), the in‑memory P&L cache is invalidated and the dashboard updates within 2 seconds.  
- **AC7 – Precision**: Calculations use `BigDecimal` with a scale of 2 and rounding mode `HALF_EVEN`.
