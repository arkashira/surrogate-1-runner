- **AC1 – Create Workflow:** A user can create a new workflow via the API and receive a unique workflow ID; the workflow is persisted in the database.
- **AC2 – Retrieve Workflow:** Given a valid workflow ID, the system returns the exact workflow definition (name, steps, metadata) within 200 ms.
- **AC3 – Update Workflow:** Modifications to an existing workflow (e.g., adding/removing steps) are saved, and a new version number is generated while preserving prior versions.
- **AC4 – Delete Workflow:** A workflow can be soft‑deleted; the record remains in the audit log but is excluded from active queries.
- **AC5 – Version History:** The API can list all versions of a workflow, showing version number, timestamp, and change summary.
- **AC6 – Documentation Linkage:** Each workflow record contains a non‑null `documentation_url` that points to the associated process documentation; the URL must be a valid HTTP/HTTPS URI.
- **AC7 – Concurrency Control:** Simultaneous updates to the same workflow result in a conflict error (HTTP 409) with a clear message, preventing lost updates.
