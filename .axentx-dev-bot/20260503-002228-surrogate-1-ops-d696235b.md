### Final Implementation Plan (CDN-first ingestion, ≤2h)

**Highest-value change**: Add `bin/snapshot.sh` that produces a deterministic file manifest per date folder and update ingestion/training to use CDN URLs exclusively when a snapshot is provided. This eliminates Hugging Face API rate-limit risk during training and ingestion.

---

### Concrete steps (order of execution)

1. Add `bin/snapshot.sh`  
   - Accepts `REPO`, `DATE` (YYYY-MM-DD), optional `OUT` (default `snapshot-<DATE>.json`).  
   - Uses `huggingface_hub.list_repo_tree` to list files in `DATE` folder.  
   - Emits JSON array of `{ "path": "...", "size": ..., "sha256": ..., "cdn_url": "https://huggingface.co/datasets/REPO/resolve/main/..." }`.  
   - Retries once after 60s on 429/5xx; exits 0 on success.

2. Add `lib/cdn_download.py`  
   - `download_cdn(url, dest, retries=3, backoff=2, timeout=300)` with streaming, HTTP validation, progress to stderr, and idempotent writes.

3. Update `bin/dataset-enrich.sh`  
   - Add optional `SNAPSHOT_FILE` env var.  
   - If `SNAPSHOT_FILE` exists, read CDN URLs from it and skip `list_repo_tree`/API calls.  
   - Use `lib/cdn_download.py` (or `curl -fsSL` as fallback) to download each file.  
   - Keep existing schema projection and shard logic unchanged.

4. Add CDN-first training option  
   - New script or flag (e.g., `train-cdn.py` or `--snapshot snapshot-<DATE>.json`) that accepts a snapshot file.  
   - Builds an `IterableDataset` that yields local file paths or downloads on-the-fly from CDN URLs.  
   - No `load_dataset` or `list_repo_files` calls during training.  
   - Preserve deterministic shard selection via `slug-hash % 16 == SHARD_ID`.

5. CI / cron hygiene  
   - Orchestrator runs `snapshot.sh` once per date folder after rate-limit window clears; commits or stores snapshot JSON as an artifact/cloud object.  
   - GitHub Actions matrix injects `SNAPSHOT_FILE` artifact so ingestion runs use CDN-only paths.  
   - Training job mounts snapshot and uses CDN-only loader.

### Code snippets

#### bin/snapshot.sh
```bash
#!/usr/bin/env bash
set -euo pipefail

REPO="${1:-axentx/surrogate-1-training-pairs}"
DATE="${2:-$(date +%Y-%m-%d)}"
OUT="${3:-snapshot-${DATE}.json}"

echo "[$(date -u +%Y-%m-%dT%H:%M:%SZ)] snapshot: repo=${REPO} date=${DATE} -> ${OUT}" >&2

python3 - "$REPO" "$DATE" "$OUT" <<'PY'
import json, sys, time
from huggingface_hub import list_repo_tree

repo = sys.argv[1]
date = sys.argv[2]
out = sys.argv[3]

retry = 0
while True:
    try:
        entries = list_repo_tree(repo, path=date, repo_type="dataset", recursive=False)
        break
    except Exception as e:
        retry += 1
        if retry > 1:
            raise
        print(f"list_repo_tree failed (attempt {retry}): {e}", file=sys.stderr)
        time.sleep(60)

items = []
for e in entries:
    if getattr(e, "type", None) != "file":
        continue
    path = f"{date}/{e.path}" if not e.path.startswith(date) else e.path
    items.append({
        "path": path,
        "size": getattr(e, "size", None),
        "sha256": getattr(e, "sha256", None),
        "cdn_url": f"https://huggingface.co/datasets/{repo}/resolve/main/{path}"
    })

with open(out, "w") as f:
    json.dump(items, f, indent=2)
print(f"Wrote {len(items)} items to {out}", file=sys.stderr)
PY

echo "[$(date -u +%Y-%m-%dT%H:%M:%SZ)] done" >&2
```
