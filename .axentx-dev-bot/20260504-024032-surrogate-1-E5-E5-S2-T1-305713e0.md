# surrogate-1-runner

Parallel public-dataset ingest workers for the
[axentx/surrogate-1-training-pairs](https://huggingface.co/datasets/axentx/surrogate-1-training-pairs)
HuggingFace dataset.

## What this does

Every 30 minutes (or on `workflow_dispatch`), GitHub Actions launches **16 parallel runners**.
Each runner takes a deterministic 1/16 slice (`slug-hash bucket = SHARD_ID`)
of the public dataset list defined in `bin/dataset-enrich.sh`, streams,
normalizes per-schema, dedups via the central md5 hash store, and uploads
its output to a unique path on the dataset repo:

## Installation

To install the necessary dependencies and set up the environment, follow these steps:

1. Clone the repository:
   ```bash
   git clone https://github.com/your-repo/surrogate-1.git
   cd surrogate-1
   ```

2. Install Python dependencies:
   ```bash
   pip install -r requirements.txt
   ```

3. Set up the configuration file:
   Copy the sample `config.yaml` to `config.yaml` and modify it according to your needs.

## Configuration

The configuration file `config.yaml` is used to define various settings for the application. Below is an example of the schema and how to configure it:

### Example `config.yaml`
```yaml
dataset_path: "/path/to/dataset"
output_path: "/path/to/output"
shard_count: 16
interval_minutes: 30
```

- `dataset_path`: The path to the dataset directory.
- `output_path`: The path where the processed data will be stored.
- `shard_count`: The number of shards to divide the dataset into.
- `interval_minutes`: The interval in minutes at which the runners should be launched.

## Usage

To run the application, execute the following command:

```bash
python main.py
```

This will start the runners according to the configuration specified in `config.yaml`.

For more detailed information about the configuration options and advanced usage, refer to the [configuration documentation](docs/config.md).
