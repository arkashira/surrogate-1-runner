## 2.  The unified, “best‑of‑both” design

Below is a single, self‑contained module that:

* Works on **Linux, macOS, and Windows** (uses `notify2`, `pync`, or `win10toast`).
* Reads a tiny **YAML/JSON config file** (`~/.axentx/battery_monitor.yaml`) so the user can change thresholds, intervals, and notification style without touching code.
* Uses **asyncio** so the daemon can be cancelled cleanly.
* Logs to `~/.axentx/battery_alerts.log` and to the console.
* Provides a **CLI** that can run a single check or start the daemon.
* Comes with a **complete test‑suite** that mocks all external dependencies.

> **Contradiction resolution**  
> Candidate 1 uses `notify-send` (Linux only).  
> We replace it with a *strategy* pattern that picks the best notifier for the platform.  
> The “correctness” rule says: “use the most reliable, cross‑platform solution.”  
> Hence we keep the Linux path but add fallbacks for macOS and Windows.
