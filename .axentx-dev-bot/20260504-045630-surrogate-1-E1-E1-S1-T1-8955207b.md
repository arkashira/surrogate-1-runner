# Surrogate‑1 On‑boarding Flow
**Target:** First processed output in ≤ 10 minutes.

## ✅ Prerequisites (≈ 1 min)
- Docker + Docker‑Compose installed.  
- GitHub account (for optional workflow trigger).  
- Hugging Face read‑only token for `axentx/surrogate-1-training-pairs`.  

> **Tip:** The wizard will run a quick system‑check and tell you exactly what is missing.

## 1️⃣ Welcome & System Check (≈ 1 min)
- Run the wizard: `python -m src.onboarding.onboarding_flow`  
- The wizard shows a short intro and automatically validates:
  - `docker` command is available.
  - `docker‑compose` version ≥ 1.29.
  - `git` is installed.
  - Network connectivity to `huggingface.co` and `github.com`.

> If any check fails, the wizard prints the exact install command (e.g. `brew install docker`) and aborts until fixed.

## 2️⃣ Clone & Configure (≈ 2 min)
1. **Clone** (wizard does it for you if you run from any directory):
   ```bash
   git clone https://github.com/axentx/surrogate-1-runner.git
   cd surrogate-1-runner
   ```
2. **Copy & edit** the env file (wizard opens your editor):
   ```bash
   cp .env.example .env
   # wizard opens $EDITOR so you paste your HF_TOKEN (and optional GITHUB_TOKEN)
   ```

## 3️⃣ Start the Local Worker (≈ 2 min)
```bash
docker-compose up -d
docker-compose ps   # → surrogate-1-worker should be “Up”
```
The wizard tails the container logs for the first 30 seconds so you can see the worker boot.

## 4️⃣ Trigger the First Action (≈ 2 min)
- **Option A – Local:**  
  ```bash
  ./bin/dataset-enrich.sh   # a thin wrapper that pulls the HF dataset & writes ./output
  ```
- **Option B – GitHub Action:**  
  ```bash
  gh workflow run process_dataset.yml   # requires GITHUB_TOKEN in .env
  ```
The wizard monitors the logs (Docker or GitHub) and prints a **✅ success badge** when the first output file appears.

## 5️⃣ Verify Output & Completion (≈ 1 min)
```bash
ls -lh output/
cat output/sample.json   # should match schema described in bin/dataset-enrich.sh
```
The wizard shows a short “What’s next?” panel with links to:
- The Hugging Face dataset page.  
- Contribution guide (`CONTRIBUTING.md`).  
- Full documentation (`docs/`).

- **AC‑3:** The final screen of the wizard displays a single, visually distinct call‑to‑action button/link labeled “Start Building” (or equivalent) that is clickable/activatable.  
- **AC‑4:** All required system checks (Python ≥ 3.9, Docker, network connectivity) are performed and any missing prerequisite is reported with a clear, actionable error message.  
- **AC‑5:** The wizard can be invoked via both the CLI entry point (`surrogate onboarding`) and the programmatic API (`surrogate.wizard.start()`) with identical behavior.  
- **AC‑6:** All wizard steps are logged to a JSON file (`~/.surrogate/onboarding.log`) with timestamps and step outcomes for auditability.

// onboardingWizard.test.js
import { Wizard } from '../src/onboarding/wizard';
import { execSync } from 'child_process';
import fs from 'fs';
import path from 'path';

describe('Wizard Core', () => {
  test('initializes with default steps', () => {
    const w = new Wizard();
    expect(w.steps).toHaveLength(5);
    expect(w.currentStep).toBe(0);
  });

  test('progresses to next step on success', () => {
    const w = new Wizard();
    w.completeCurrentStep(true);
    expect(w.currentStep).toBe(1);
  });

  test('fails fast on prerequisite missing', () => {
    jest.spyOn(wizard, 'checkDocker').mockReturnValue(false);
    const w = new Wizard();
    expect(() => w.run()).toThrow('Docker not found');
  });

  test('writes log entry for each step', () => {
    const logPath = path.join(os.homedir(), '.surrogate', 'onboarding.log');
    if (fs.existsSync(logPath)) fs.unlinkSync(logPath);
    const w = new Wizard();
    w.run(); // runs all steps (mocked)
    const log = JSON.parse(fs.readFileSync(logPath, 'utf8'));
    expect(log).toHaveLength(5);
    expect(log[0]).toMatchObject({ step: expect.any(String), status: 'success' });
  });

  test('exposes CLI entry point', () => {
    const out = execSync('node ./bin/surrogate onboarding --dry-run').toString();
    expect(out).toContain('Surrogate Onboarding Wizard');
  });

  test('programmatic API mirrors CLI behavior', async () => {
    const result = await Wizard.start({ dryRun: true });
    expect(result.success).toBe(true);
    expect(result.stepsCompleted).toBe(5);
  });
});
