# surrogate‑1‑runner

Parallel public‑dataset ingest workers for the
[axentx/surrogate‑1‑training‑pairs](https://huggingface.co/datasets/axentx/surrogate-1-training-pairs)
HuggingFace dataset.

---

## What this does

* **Every 30 minutes** (or on `workflow_dispatch`) GitHub Actions launches **16 parallel runners**.  
* Each runner processes a deterministic 1/16 slice of the public‑dataset list defined in `bin/dataset‑enrich.sh`.  
* The slice is identified by `slug‑hash bucket = SHARD_ID`.  
* For each item the runner:
  1. Streams the raw data.  
  2. Normalises it to the schema required by the training‑pairs dataset.  
  3. Deduplicates via a central MD5‑hash store.  
  4. Uploads the cleaned shard to a unique path in the dataset repository.

The result is a fully‑deduplicated, schema‑consistent training‑pairs dataset that can be consumed by downstream training pipelines.

---

## Benchmarking Watermark Performance

The repository ships a lightweight benchmark that measures the latency of inserting and verifying a watermark in the embedding layer of a transformer model.

### Prerequisites

```bash
# Create a virtual environment (recommended)
python -m venv .venv
source .venv/bin/activate

# Install required packages
pip install -r requirements.txt
```

`requirements.txt` contains:

```
transformers>=4.30
torch>=1.12
datasets
```

### Running the benchmark

```bash
python benchmark_watermark_latency.py --model-name <model_name>
```

* `<model_name>` – HuggingFace model identifier (e.g. `bert-base-uncased`, `gpt2`, etc.).  
* The script will:
  1. Load the specified model.  
  2. Insert a deterministic watermark into the embedding layer.  
  3. Measure insertion time (ms).  
  4. Verify the watermark.  
  5. Measure verification time (ms).  
  6. Print a comparison table that includes the current model and all other models in the catalog.

### Example

```bash
python benchmark_watermark_latency.py --model-name bert-base-uncased
```

Output (example):

```
+----------------------+-------------------+---------------------+
| Model                | Insertion (ms)    | Verification (ms)   |
+----------------------+-------------------+---------------------+
| bert-base-uncased    | 12.3              | 8.7                 |
| gpt2                 | 18.9              | 12.4                |
| roberta-base         | 15.2              | 9.9                 |
+----------------------+-------------------+---------------------+
```

|---------|--------------|-----|

| Deduplication misses | MD5 store out of sync | Re‑run `scripts/refresh_md5_store.py` |
