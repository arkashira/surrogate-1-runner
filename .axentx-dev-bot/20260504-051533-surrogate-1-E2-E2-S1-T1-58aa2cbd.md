## 1.  What we’re building

A **single‑source‑of‑truth** monitoring stack that

| Layer | Responsibility |
|-------|----------------|
| **Metrics** | Collect CPU / memory / disk / network stats and expose them on `/metrics` (Prometheus format). |
| **Alerting** | Pull firing alerts from a Prometheus Alertmanager instance and forward them to an e‑mail address. |
| **Dashboard** | A tiny Flask app that renders a static page and lets the browser pull `/metrics` for live charts. |
| **Tests** | Unit‑tests that mock external dependencies (`psutil`, `smtplib`, `requests`, `generate_latest`) so the logic can be verified without side‑effects. |
| **Deployment** | A Docker image that runs all three components in one container (for simplicity). |

> **Why this design?**  
> *Prometheus* is the de‑facto standard for metrics, *Alertmanager* is the canonical alert source, and *Flask* gives a quick, lightweight UI.  
> All three pieces are small, testable, and can be swapped out if you ever want to move to a different stack.

from unittest.mock import patch, MagicMock
from monitoring import PerformanceMonitor

def test_performance_monitor_initialization():
    monitor = PerformanceMonitor()

    assert monitor._thread is None

def test_collect_once_updates_metrics():
    monitor = PerformanceMonitor()
    with patch('monitoring.requests.get') as mock_get:
        mock_response = MagicMock()
        mock_response.json.return_value = {'metric1': 100, 'metric2': 200}
        mock_get.return_value = mock_response

        monitor._collect_once()

        assert monitor._metrics['metric1'] == 100
        assert monitor._metrics['metric2'] == 200

def test_start_and_stop_monitor():
    monitor = PerformanceMonitor()
    monitor.start()
    assert monitor._server is not None
    assert monitor._thread is not None
    monitor.stop()
    assert monitor._server is None
    assert monitor._thread is None
