# Surrogate‑1 Formatting Style Manager – User Guide

## Overview
The **Formatting Style Manager** offers a web-based interface enabling users to configure, save, and apply code-formatting styles across their repositories. This tool ensures consistent code style without the need for developers to manually run local formatters.

## Accessing the Interface
1. Open your browser and go to the Surrogate‑1 dashboard URL:
   ```
   https://<your‑instance>.axentx.com/formatting
   ```
2. Log in using your AxentX credentials (SSO or personal access token).

## Configuring Formatting Styles
1. **Select a Language** – From the left-hand menu, choose the programming language you wish to configure (e.g., Python, JavaScript, Go).
2. **Edit Settings** – Modify the formatter options available (such as indent size, line length, quote style). Each option comes with a tooltip providing a brief description.
3. **Preview** – Insert a code snippet into the preview pane to instantly view the formatted output.

## Saving a Style
1. Click **Save Style** at the bottom of the configuration panel.
2. Enter a **Name** (e.g., `company‑python‑standard`) and an optional **Description**.
3. Set **Visibility**:
   * **Personal** – Only you can use this style.
   * **Team** – All members of the chosen team can utilize it.
4. Click **Confirm**. The style will then appear in the **Saved Styles** list.

## Applying a Predefined Style
1. Go to the **Saved Styles** tab.
2. Find the desired style and select **Apply**.
3. Pick the target repository (or several repositories) from the dropdown menu.
4. Confirm the action. Surrogate‑1 will execute the selected formatter on the repository's codebase and generate a pull request with the modifications.

## Managing Saved Styles
* **Edit** – Update the settings of an existing style.
* **Delete** – Remove a style you no longer require (confirmation needed).
* **Export** – Save the style definition as a JSON file for backup purposes or external sharing.

## Documentation & Help
* **Tooltips** – Hover over any option for a quick explanation.
* **Help Center** – Click the **?** icon in the top-right corner for in-depth articles, FAQs, and video guides.
* **Support** – Submit a ticket through the **Support** link if you face any problems or have suggestions for new features.

## Best Practices
* **Version Control** – Consider saved styles as version-controlled items. Include a changelog entry in the description whenever a style is updated.
* **CI Integration** – Set up your CI pipeline to automatically run the formatter during pull-request validation using the corresponding style ID.
* **Team Alignment** – Establish a default style for each language and designate it as the team-wide standard on the **Team Settings** page.
