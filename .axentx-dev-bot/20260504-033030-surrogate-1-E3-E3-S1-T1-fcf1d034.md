* **RTK Query** for automatic caching, loading & error handling  
* **React‑Query‑style debounced search** (no extra library)  
* **Memoised filtering** so the list only recomputes when the data or filters change  
* **Strict‑typing** with TypeScript interfaces that live in a single `types.ts` file  
* **Accessible & responsive UI** – the filter controls are labelled, the list is a semantic `<ul>` and the component is fully keyboard‑friendly  
* **Unit‑test skeleton** – shows how to test the async slice and the component in isolation  

> **Why these choices?**  
> *RTK Query* removes boilerplate (`createAsyncThunk`, `extraReducers`, etc.) while still giving you the same Redux store.  
> Debouncing the search input prevents a flood of re‑renders when the user types quickly.  
> Memoising the filtered list keeps the UI snappy even with hundreds of deals.  
> A single source of truth for the `Deal` type keeps the code DRY and type‑safe.
