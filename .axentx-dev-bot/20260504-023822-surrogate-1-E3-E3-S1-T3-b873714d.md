## 1. Acceptance Criteria
- **AC1** – The wizard UI allows a founder to input a *minimum* and *maximum* numeric threshold for every defined metric; both fields accept only numbers and are required.
- **AC2** – On form submission, the backend evaluates each submitted metric value against its configured min/max thresholds.
- **AC3** – If **all** metric values fall within their respective thresholds, the engine returns a recommendation of **“Continue”** with a rationale string (e.g., “All metrics meet target ranges.”).
- **AC4** – If **any** metric value is outside its configured range, the engine returns a recommendation of **“Pivot”** with a rationale that lists the offending metric(s) and the direction of violation (e.g., “Revenue below minimum threshold.”).
- **AC5** – When a metric value is missing (null/undefined) the engine treats it as a failure and returns **“Pivot”** with a rationale indicating the missing data.
- **AC6** – When a threshold definition is incomplete (missing min or max), the engine returns a validation error before evaluation, with a clear message (e.g., “Threshold for ‘Engagement’ missing min value.”).
- **AC7** – All responses are returned with HTTP 200 for successful evaluation and HTTP 400 for validation errors; payload conforms to the schema `{ recommendation: string, rationale: string[] }`.
