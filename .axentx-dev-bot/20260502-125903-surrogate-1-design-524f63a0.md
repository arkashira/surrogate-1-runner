## Final Synthesized Solution

**Core diagnosis (agreed by both candidates):**
- No date-scoped manifest → repeated HF repo enumeration → 429 rate-limit failures.
- Schema heterogeneity (mixed columns across repos) → pyarrow CastError during training loads.
- Training uses HF API calls (or `load_dataset`) instead of CDN URLs → burns quota.
- Lightning Studio reuse missing → quota waste and instability.
- Ingestion writes lack deterministic repo selection and clean projection → risks HF commit-cap blocks and training failures.

**Chosen approach (correctness + actionability):**
- Persist a **date-scoped manifest** with CDN URLs and deterministic sibling-repo index for commit-cap spreading.
- **Project to `{prompt,response}` at parse time**; store enriched outputs with slug-based filenames.
- **Training uses CDN-only fetches** (zero authenticated HF API calls).
- **Lightning Studio reuse + idle-stop guard** to preserve quota.
- **Schema-tolerant parsing** with explicit fallback heuristics to avoid CastError.

### 1. Manifest generator (persisted, date-scoped)

```bash
# /opt/axentx/surrogate-1/design/manifest.py
#!/usr/bin/env python3
"""
Generate and persist a date-scoped manifest for HF dataset repos.
Run once per date folder (or in CI) to avoid repeated repo enumeration.
"""
import json, os, hashlib, datetime
from huggingface_hub import HfApi

API = HfApi()
REPO = "datasets/axentx/surrogate-1"          # adjust if needed
DATE_FOLDER = os.getenv("DATE_FOLDER", "2026-04-29")
OUT_DIR = os.path.join(os.path.dirname(__file__), "..", "manifests")
os.makedirs(OUT_DIR, exist_ok=True)

def build_manifest() -> list[str]:
    # Non-recursive to minimize API use; caller can recurse if needed
    tree = API.list_repo_tree(repo_id=REPO, path=DATE_FOLDER, recursive=False)
    files = [
        entry.path for entry in tree
        if entry.type == "file" and entry.path.lower().endswith((".jsonl", ".parquet", ".json"))
    ]
    return sorted(set(files))

def pick_sibling_repo(slug: str, n_siblings: int = 5) -> int:
    """Deterministic sibling index to spread ingestion writes across repos."""
    digest = hashlib.sha256(slug.encode()).digest()
    return int.from_bytes(digest[:2], "big") % n_siblings

def main() -> None:
    files = build_manifest()
    out_path = os.path.join(OUT_DIR, f"manifest-{DATE_FOLDER}.json")
    meta = {
        "date": DATE_FOLDER,
        "generated_at": datetime.datetime.utcnow().isoformat() + "Z",
        "repo": REPO,
        "files": files,
        "cdn_base": f"https://huggingface.co/datasets/{REPO}/resolve/main",
        "sibling_repo_index": pick_sibling_repo(f"{REPO}/{DATE_FOLDER}"),
    }
    with open(out_path, "w") as f:
        json.dump(meta, f, indent=2)
    print(f"Manifest written: {out_path}  files={len(files)}")

if __name__ == "__main__":
    main()
```
