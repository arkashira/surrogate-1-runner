## 1. Acceptance Criteria
- **AC‑1**: The suite can load any open‑weight LLM defined in `ModelConfig` and execute both watermark insertion and verification without runtime errors.  
- **AC‑2**: Latency metrics (mean, median, 5th/95th percentiles, standard deviation) are reported for insertion **and** verification phases separately.  
- **AC‑3**: Throughput (tokens / second) and token‑rate (tokens / iteration) are calculated and displayed in the final JSON/CSV report.  
- **AC‑4**: Robustness is measured as the percentage of correctly verified watermarks after applying a configurable perturbation pipeline (e.g., temperature sampling, token dropout, paraphrase).  
- **AC‑5**: The runner accepts command‑line arguments **and** a programmatic API; both paths produce identical output files.  
- **AC‑6**: Warm‑up iterations, random seed, and perturbation settings are configurable via CLI flags or a `BenchmarkConfig` object, and the same seed yields identical metric distributions across runs.  
- **AC‑7**: Adding a new model requires only a one‑line entry in `models.py` (instantiating `ModelConfig`). The suite discovers it automatically without code changes.
