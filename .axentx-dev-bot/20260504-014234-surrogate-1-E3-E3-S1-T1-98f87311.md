### Final Answer: Real-Time Collaboration System for Workflow Execution

This solution provides a robust foundation for real-time collaboration within workflows using FastAPI, including secure invitation tokens, WebSocket-based communication, and scalable connection management — all designed for both development (CI) and production environments.

---

#### ✅ Key Features & Design Choices

1. **Secure Invitation Tokens**
   - Generates signed tokens with expiration times.
   - Uses a placeholder HMAC-like signing mechanism (replaceable with JWT or `itsdangerous` in production).
   - Token includes workflow ID, email, and expiration timestamp.

2. **WebSocket Communication**
   - Supports real-time messaging via WebSockets.
   - Each workflow has its own isolated channel.
   - Enforces maximum concurrent connections per workflow.
   - Broadcasts messages to all participants in a workflow.

3. **Configuration Management**
   - All settings are configurable via environment variables.
   - Default values are safe for local testing but require overrides in production.

4. **Error Handling & Security**
   - Validates tokens at connection time.
   - Rejects invalid or expired tokens.
   - Limits concurrent connections to prevent resource exhaustion.
   - Gracefully handles disconnections and malformed messages.

5. **Extensibility**
   - Modular structure allows swapping out components like storage backend or authentication layer.
   - Easy to integrate with existing systems through standard REST and WebSocket interfaces.
