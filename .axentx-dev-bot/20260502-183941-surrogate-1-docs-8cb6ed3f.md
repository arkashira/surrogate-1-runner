## Pipeline validation

Pipelines are validated on load. A valid pipeline must:

- Define at least one stage.
- Provide required fields (`name`, `type`, `config`) for every stage.
- Use stage names that are non-empty, alphanumeric + underscore only, and unique.
- Declare edges that reference existing stages, disallow self-loops, and form a directed acyclic graph (DAG).
- Set positive integer timeouts: `timeout_budget.per_stage_ms` and `timeout_budget.end_to_end_ms`. A per-stage `timeout_ms` (when provided) overrides the per-stage budget and must also be positive.
- Use a `retry_policy` with `attempts` between 1–10, non-negative `backoff_ms`, and `retry_on` values from the known set (defaults: `attempts=1`, `backoff_ms≥0`).

Validation is performed by `validate_pipeline(...)`, which accepts YAML/JSON strings, dicts, or `Path` objects and returns a `Pipeline` or raises `PipelineValidationError`. Use `Pipeline.stage_by_name(name)` to look up stages (raises `KeyError` if missing).
