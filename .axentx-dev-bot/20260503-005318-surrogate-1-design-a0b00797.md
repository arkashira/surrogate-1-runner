The training script is then updated to use CDN-only fetches, avoiding HF API rate limits during training.

**Implementation Plan**

1. Create `tools/snapshot_manifest.py`:
	* Inputs: `repo`, `date_partition` (e.g., `2026-04-29`), `out_path`
	* Single API call: `hf_api.list_repo_tree(repo, path=date_partition, recursive=False)`
	* For each file:
		+ Build CDN URL: `f"https://huggingface.co/datasets/{repo}/resolve/main/{date_partition}/{filename}"`
		+ Optional: store `size`, `etag`/`last_modified` if available for conditional requests
	* Output: `file_manifest.json`
	* Deterministic ordering (sorted by filename) for reproducibility
2. Add `tools/requirements.txt` (minimal):
	* `huggingface_hub`
3. Update training entrypoint (`train.py`):
	* Accept `--manifest file_manifest.json`
	* Read manifest; for each CDN URL:
		+ Stream download with `requests.get(url, stream=True, timeout=60)`
		+ If parquet: use `pyarrow.parquet.ParquetFile` + `read_columns(["prompt","response"])` to project
		+ If JSONL: stream line-by-line and project fields
	* Yield `{prompt, response}` pairs
	* No `datasets.load_dataset` on heterogeneous files
4. Add small utility: `tools/verify_manifest.py`:
	* Quick HEAD/conditional GET checks for CDN URLs (optional, for CI)
5. Update README section:
	* Document workflow: run snapshot → commit manifest → train with CDN-only
