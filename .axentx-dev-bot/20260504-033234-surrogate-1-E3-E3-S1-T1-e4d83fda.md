## 1.  Data Model (JSON‑style)

```json
{
  "investment": {
    "id": "inv001",
    "investor_id": "alice123",
    "investor_name": "Alice",
    "amount": 50000,
    "status": "In Progress",      // one of: In Progress, Completed, Flagged
    "last_updated": "2026-05-01T14:23:00Z"
  }
}
```

* **id** – unique identifier (string, UUID or prefixed like `inv001`).  
* **investor_id** – key used for messaging.  
* **investor_name** – display name.  
* **amount** – numeric, in the base currency (USD in the example).  
* **status** – enum; UI will colour‑code.  
* **last_updated** – ISO‑8601 timestamp.

Persist this in a relational table or a NoSQL collection; the API will simply read it.
