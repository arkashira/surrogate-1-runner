# docs/ci_cd_integration.md
# CI/CD Integration Guide for Mesh File Health Probe

This document provides integration instructions for the mesh file health probe across popular CI/CD systems.

## Overview

The mesh file health probe is designed to validate the integrity and readiness of mesh files within your CI/CD pipelines. It outputs structured JSON that can be consumed by monitoring systems and alerting tools.

## Jenkins Integration

### Prerequisites
- Jenkins 2.0+
- Node.js plugin installed (for JSON parsing)

### Configuration Steps

1. Add a build step to execute the health probe:
   ```bash
   ./mesh-health-probe --output-format json
   ```

2. Configure post-build actions to parse JSON output:
   - Use the "Publish JUnit test result report" plugin
   - Point to the output file generated by the health probe

3. Set up notifications using the "Email Extension Plugin" to trigger alerts based on probe results.

### Example Jenkinsfile Snippet
```groovy
pipeline {
    agent any
    stages {
        stage('Health Check') {
            steps {
                sh './mesh-health-probe --output-format json'
            }
            post {
                always {
                    publishTestResults 'health-report.json'
                }
            }
        }
    }
}
```

## GitHub Actions Integration

### Prerequisites
- GitHub repository with workflow permissions enabled

### Workflow Configuration

Create `.github/workflows/mesh-health-check.yml`:

```yaml
name: Mesh File Health Check

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  health-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Run Mesh Health Probe
        id: health_check
        run: |
          ./mesh-health-probe --output-format json > health_report.json
          
      - name: Upload Health Report
        uses: actions/upload-artifact@v3
        with:
          name: health-report
          path: health_report.json
          
      - name: Fail on Critical Issues
        if: contains(steps.health_check.outputs.status, 'CRITICAL')
        run: exit 1
```

### Output Format Compatibility

The health probe outputs JSON with the following structure:
```json
{
  "status": "HEALTHY|UNHEALTHY|CRITICAL",
  "timestamp": "2023-01-01T00:00:00Z",
  "checks": [
    {
      "name": "mesh-file-integrity",
      "status": "PASS|FAIL",
      "message": "Validation successful"
    }
  ]
}
```

## CircleCI Integration

### Prerequisites
- CircleCI project configured with appropriate permissions

### Configuration Steps

Add to `.circleci/config.yml`:

```yaml
version: 2.1

jobs:
  mesh-health-check:
    docker:
      - image: circleci/python:3.9
    steps:
      - checkout
      - run:
          name: Run Mesh Health Probe
          command: |
            ./mesh-health-probe --output-format json > health_report.json
      - store_artifacts:
          path: health_report.json
          destination: health_report.json

workflows:
  version: 2
  health-check-workflow:
    jobs:
      - mesh-health-check
```

### Monitoring Dashboard Integration

To integrate with monitoring dashboards like Datadog or Prometheus:

1. Parse the JSON output from the health probe
2. Extract key metrics:
   - Overall status (HEALTHY/UNHEALTHY/CRITICAL)
   - Number of checks passed/failed
   - Timestamp of last check
3. Send these metrics to your dashboard using appropriate APIs or integrations

### Example Alerting Rule (Prometheus)
```yaml
groups:
- name: mesh-health
  rules:
  - alert: MeshFileUnhealthy
    expr: mesh_health_status == 0
    for: 5m
    labels:
      severity: critical
    annotations:
      summary: "Mesh file health check failed"
```
