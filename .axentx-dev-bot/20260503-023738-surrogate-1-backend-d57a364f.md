## 1. Acceptance criteria
- For fixed (DATE, SHARD_ID, SHARD_TOTAL) the worker produces identical `batches/public-merged/{DATE}/shard{N}-{HHMMSS}.jsonl` byte-for-byte when re-run with same inputs and same manifest.
- Worker performs exactly one `list_repo_tree` (or equivalent manifest fetch) per DATE and writes a local manifest JSON; subsequent runs reuse that manifest when present and valid.
- All downloads use CDN URLs without an `Authorization` header; any request containing `Authorization` to HuggingFace datasets domains is treated as a failure in tests.
- Parquet projection maps rows to `{prompt: str, response: str}`; rows missing either field are logged and skipped, and schema mismatches do not crash the worker (exit code 0 for parse skips, non-zero only on fatal errors).
- Deduplication via `lib.dedup.DedupStore` removes exact `(prompt, response)` duplicates across the shard; output contains only unique entries and total output lines ≤ total input rows.
- Exit codes are deterministic: 0 = success (manifest + shard written), 1 = retryable (transient network/manifest fetch), 2 = fatal (bad inputs/internal error).
