# Surrogate API Drift Detection GitHub Action

[![GitHub Marketplace](https://img.shields.io/badge/Marketplace-Surrogate%20API%20Drift-blue)](https://github.com/marketplace/actions/surrogate-api-drift)
[![Action Status](https://img.shields.io/github/actions/workflow/status/axentx/surrogate-api-drift/drift.yml)](https://github.com/axentx/surrogate-api-drift/actions)

A lightweight GitHub Action for detecting drift between a deployed API and its expected schema. If drift is detected, the action fails the workflow with a concise diff summary.

## Key Features
- **Fast Execution**: Optimized to run in ≤ 30 seconds on standard GitHub runners.
- **Workflow Integration**: Exits with status code `1` on drift, failing the workflow.
- **Required Inputs**: Service name, version, and database connection string.

## Usage

```yaml
name: API Drift Check
on:
  push:
    branches: [ main ]
jobs:
  drift-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Check API Drift
        uses: axentx/surrogate-api-drift@v1
        with:
          service: "user-service"          # Name of the service to check
          version: "v2.3.1"                # Expected API version
          db-connection-string: ${{ secrets.DB_CONN_STRING }}  # Database connection
```

### Inputs
| Name                  | Description                                      | Required |
|-----------------------|--------------------------------------------------|----------|
| `service`             | Name of the service to check.                    | ✅ Yes   |
| `version`             | Expected API version (e.g., `v1.2.3`).           | ✅ Yes   |
| `db-connection-string`| Connection string to the drift database.        | ✅ Yes   |

### Behavior
1. Fetches the current API schema for the specified `service` and `version`.
2. Compares it against the stored baseline in the provided database.
3. On drift:
   - Prints a concise diff summary.
   - Exits with status code `1` (failing the workflow).
4. On no drift: Exits with status code `0`.
