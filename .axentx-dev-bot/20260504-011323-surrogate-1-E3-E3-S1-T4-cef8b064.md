| What we keep | Why it’s valuable |
|--------------|-------------------|
| **Plain‑function API** `rag_query(ingestion_id, question)` that returns `(answer, snippets)` | Simple, easy to call from any code base. |
| **Configuration loaded once, but mock‑able** (`_load_config`) | Keeps production code clean while allowing deterministic tests. |
| **Separate helpers** `_retrieve_snippets`, `_build_llm_client`, `_render_prompt` | Clear responsibilities, easy to patch in tests. |
| **Prompt templating with Jinja‑style `{{var}}` placeholders** | Gives flexibility without pulling in a heavy templating engine. |
| **LLM client abstraction with an async `generate(prompt, **kwargs)` method** | Works for both HTTP‑based (e.g., OpenAI, LLaMA) and gRPC‑based back‑ends; the test can supply a tiny dummy client. |
| **Pytest‑style, fully‑isolated unit test** that patches every external dependency | Guarantees deterministic, fast CI runs; no network or file‑system access required. |
| **Explicit error handling** (raise `RuntimeError` for unsupported endpoints, surface HTTP/gRPC errors) | Makes failures obvious and testable. |
| **Typed signatures & doc‑strings** | Improves readability and IDE support. |
