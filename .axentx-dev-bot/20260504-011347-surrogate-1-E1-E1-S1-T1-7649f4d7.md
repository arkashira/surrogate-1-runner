### 1. **Acceptance criteria**

- User can upload a `.tf` or `.yaml` configuration file via the UI; invalid extensions (e.g., `.txt`, `.exe`) are rejected.
- User can upload a `.json` audit log file; non-JSON files or unsupported formats are rejected.
- Upon successful upload of both required files, the "Generate Report" button becomes enabled.
- After clicking "Generate Report", the system initiates processing and displays a download link for the generated PDF.
- The downloaded PDF filename follows the format: `{config-filename}-{timestamp}.pdf` (e.g., `main.tf-20250405T123045Z.pdf`).

### 2. **Unit tests**

```javascript
// upload.test.js (Jest-style)

import { validateFile, isConfigFile, isAuditLogFile, enableGenerateButton } from './upload.js';

describe('File Upload Validation', () => {
  test('accepts .tf files as valid config files', () => {
    const file = new File([''], 'infra.tf', { type: 'application/tf' });
    expect(isConfigFile(file)).toBe(true);
  });

  test('accepts .yaml and .yml files as valid config files', () => {
    const yamlFile = new File([''], 'values.yaml', { type: 'application/yaml' });
    const ymlFile = new File([''], 'values.yml', { type: 'application/yaml' });
    expect(isConfigFile(yamlFile)).toBe(true);
    expect(isConfigFile(ymlFile)).toBe(true);
  });

  test('rejects non-config file types', () => {
    const txtFile = new File([''], 'notes.txt', { type: 'text/plain' });
    const jsFile = new File([''], 'script.js', { type: 'application/javascript' });
    expect(isConfigFile(txtFile)).toBe(false);
    expect(isConfigFile(jsFile)).toBe(false);
  });

  test('accepts .json files as valid audit logs', () => {
    const jsonFile = new File(['{}'], 'audit.json', { type: 'application/json' });
    expect(isAuditLogFile(jsonFile)).toBe(true);
  });

  test('rejects non-JSON files as audit logs', () => {
    const logFile = new File([''], 'audit.log', { type: 'text/plain' });
    const xmlFile = new File([''], 'data.xml', { type: 'application/xml' });
    expect(isAuditLogFile(logFile)).toBe(false);
    expect(isAuditLogFile(xmlFile)).toBe(false);
  });

  test('validateFile returns error for invalid extension', () => {
    const invalidFile = new File([''], 'malware.exe', { type: 'application/octet-stream' });
    expect(validateFile(invalidFile, ['tf'])).toEqual({ valid: false, error: 'Invalid file type' });
  });
});

describe('Generate Button State', () => {
  test('enableGenerateButton returns true when both files are uploaded', () => {
    expect(enableGenerateButton('main.tf', 'audit.json')).toBe(true);
  });

  test('enableGenerateButton returns false if either file is missing', () => {
    expect(enableGenerateButton('', 'audit.json')).toBe(false);
    expect(enableGenerateButton('main.tf', '')).toBe(false);
    expect(enableGenerateButton('', '')).toBe(false);
  });
});
```

#### Happy paths:

2. User uploads `deploy.yaml` and `activity-log.json` → button enables → PDF generated as `deploy.yaml-<timestamp>.pdf`.
3. User uploads `.yml` config (`config.yml`) + valid JSON log → report generates successfully.

#### Edge cases:
1. User uploads config file twice (same name) → second upload overwrites first; button remains enabled only if audit log also exists.
2. User uploads invalid file first, then valid one → error message clears and button enables when both valid files are present.
3. User uploads a `.json` file with invalid JSON content → client shows error or backend rejects during processing (client validation optional; backend must reject).
