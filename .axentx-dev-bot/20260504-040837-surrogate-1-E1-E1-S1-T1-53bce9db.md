### Final Proposal: Workflow Automator Terraform Provider

#### Overview

The Workflow Automator Terraform provider allows you to define and manage workflows within your infrastructure-as-code configurations. This provider enables you to create, update, and manage workflows using Terraform configurations.

#### Configuration

To use the Workflow Automator Terraform provider, you need to configure the provider with the following inputs:

- `api_key` (string, required): The API key for the Workflow Automator service.
- `base_url` (string, required): The base URL for the Workflow Automator API.

```terraform
// /opt/axentx/surrogate-1/providers/terraform/provider.tf
terraform {
  required_providers {
    workflow_automator = {
      source  = "axentx/workflow-automator"
      version = "~> 1.0"
    }
  }
}

provider "workflow_automator" {
  api_key = var.workflow_automator_api_key
  base_url = var.workflow_automator_base_url
}
```

#### Usage

To define a workflow using the Workflow Automator Terraform provider, you can use the following example:

```hcl
// /opt/axentx/surrogate-1/providers/terraform/docs.md
# Workflow Automator Terraform Provider

## Usage

```hcl
terraform {
  required_providers {
    workflow_automator = {
      source  = "axentx/workflow-automator"
      version = "~> 1.0"
    }
  }
}

provider "workflow_automator" {
  api_key = var.workflow_automator_api_key
  base_url = var.workflow_automator_base_url
}

resource "workflow_automator_workflow" "example" {
  name = "Example Workflow"

  # Define workflow steps here
}
```

#### Installation

To install the provider, execute the following command:

```bash
terraform providers install workflow_automator
```

#### Importing Existing Resources

You can import existing workflows using the `id`. For instance, the following will import the `example` workflow:

```bash
terraform import workflow_automator_workflow.example <workflow_id>
```
