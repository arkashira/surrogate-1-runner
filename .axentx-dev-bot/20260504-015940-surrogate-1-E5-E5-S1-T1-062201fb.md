## 1. Overview

`surrogateвҖ‘1` is an automated, parallel ingestion pipeline that turns public datasets into a single, deduplicated HuggingFace dataset (`axentx/surrogateвҖ‘1вҖ‘trainingвҖ‘pairs`).  
Key facts:

| Feature | Detail |
|---------|--------|
| **Parallelism** | 16 concurrent GitHub Actions runners (shardsвҖҜ0вҖ‘15) |
| **Schedule** | Runs every 30вҖҜmin (cron) or onвҖ‘demand via `workflow_dispatch` |
| **Deterministic sharding** | `SHARD_ID = hash(dataset_slug) % 16` вҖ“ guarantees each dataset is processed by the same runner every run |
| **Deduplication** | Central MD5 hash store (e.g., Redis or Postgres) prevents duplicate records |
| **Output format** | `shard-{SHARD_ID}/{timestamp}/{dataset_name}.jsonl` вҖ“ each line is a JSON object with `id`, `text`, `metadata` |
| **Target** | HuggingFace Hub repository `axentx/surrogate-1-training-pairs` |

---

## 2. Quick Start

### 2.1 Prerequisites

| Requirement | How to satisfy |
|-------------|----------------|
| **GitHub repo** | `axentx/surrogate-1` with Actions enabled |
| **HF write access** | HF token with `write` scope for `axentx/surrogate-1-training-pairs` |
| **GitHub Actions minutes** | Sufficient minutes on your plan (16 runners Г— 30вҖҜmin вүҲ 8вҖҜhrs per day) |
| **MD5 store** | Redis/PG instance reachable from the runners (see `MD5_STORE_URL` env var) |

### 2.2 Running the Pipeline

| Mode | Steps |
|------|-------|
| **Automatic** | The workflow `ingest.yml` is scheduled (`cron: '*/30 * * * *'`). No action required. |
| **Manual** | 1. Go to **Actions** вҶ’ **surrogateвҖ‘1** вҶ’ **Run workflow**. 2. Optionally set `shard_id` (0вҖ‘15) to test a single shard. |

### 2.3 Monitoring

```bash
# List recent runs
gh run list --workflow=surrogate-1

# View logs for a run
gh run view <run-id> --log

# Check last run status
gh run list --workflow=surrogate-1 --limit 1 --json status,conclusion
```
